# ERD (Entity Relationship Diagram)

> 숨고 클론 프로젝트 데이터베이스 설계  
> 이니셜 구분: `ec_` 이커머스/서비스 도메인 | `sy_` 시스템 공통

---

## 테이블 목록

| 이니셜 | 테이블명 | 설명 |
|--------|---------|------|
| ec_ | ec_member | 회원 (의뢰인 / 고수) |
| ec_ | ec_cate | 서비스 카테고리 |
| ec_ | ec_prod | 서비스 상품 (고수 등록 서비스) |
| ec_ | ec_order | 견적/주문 |
| ec_ | ec_order_item | 견적/주문 상세 항목 |
| sy_ | sy_code | 공통 코드 |
| sy_ | sy_user | 시스템 사용자 (관리자) |
| sy_ | sy_attach_grp | 첨부파일 그룹 |
| sy_ | sy_attach | 첨부파일 |
| sy_ | sy_login_hist | 로그인 이력 |

---

## ERD 다이어그램

```mermaid
erDiagram

    %% ─────────────── 시스템 공통 ───────────────

    sy_code {
        bigint      code_id         PK  "코드 ID"
        varchar(50) code_grp            "코드 그룹"
        varchar(100) code_label         "코드 라벨"
        varchar(100) code_value         "코드 값"
        int         sort_ord            "정렬 순서"
        char(1)     use_yn              "사용 여부"
        timestamp   created_at
        timestamp   updated_at
    }

    sy_user {
        bigint      user_id         PK  "사용자 ID"
        varchar(100) login_id           "로그인 ID"
        varchar(255) login_pw           "비밀번호 (BCrypt)"
        varchar(100) user_name          "사용자명"
        varchar(200) user_email         "이메일"
        varchar(50) role_cd             "역할 코드 (sy_code)"
        char(1)     use_yn              "사용 여부"
        timestamp   created_at
        timestamp   updated_at
    }

    sy_attach_grp {
        bigint      attach_grp_id   PK  "첨부 그룹 ID"
        varchar(100) grp_name           "그룹명"
        varchar(100) ref_table          "참조 테이블명"
        bigint      ref_id              "참조 레코드 ID"
        timestamp   created_at
    }

    sy_attach {
        bigint      attach_id       PK  "첨부파일 ID"
        bigint      attach_grp_id   FK  "첨부 그룹 ID"
        varchar(255) orig_file_name     "원본 파일명"
        varchar(500) file_path          "저장 경로"
        bigint      file_size           "파일 크기 (bytes)"
        varchar(100) file_type          "MIME 타입"
        int         sort_ord            "정렬 순서"
        timestamp   created_at
    }

    sy_login_hist {
        bigint      login_hist_id   PK  "로그인 이력 ID"
        bigint      user_id         FK  "시스템 사용자 ID"
        bigint      member_id       FK  "회원 ID"
        timestamp   login_dt            "로그인 일시"
        varchar(45) login_ip            "접속 IP"
        varchar(20) login_result_cd     "결과 코드 (sy_code)"
        varchar(500) user_agent         "브라우저 정보"
    }

    %% ─────────────── 이커머스/서비스 도메인 ───────────────

    ec_member {
        bigint      member_id       PK  "회원 ID"
        bigint      user_id         FK  "시스템 사용자 ID"
        varchar(10) member_type_cd      "회원 유형: CLIENT/PRO"
        varchar(100) member_name        "이름"
        varchar(20) phone               "연락처"
        varchar(200) email              "이메일"
        varchar(100) location           "활동 지역"
        varchar(500) bio                "자기 소개"
        bigint      profile_attach_id   FK "프로필 이미지 attach_id"
        decimal(3,1) avg_rating         "평균 평점"
        int         review_cnt          "리뷰 수"
        int         response_rate       "응답률 (%)"
        char(1)     del_yn              "탈퇴 여부"
        timestamp   created_at
        timestamp   updated_at
    }

    ec_cate {
        bigint      cate_id         PK  "카테고리 ID"
        bigint      parent_cate_id      "상위 카테고리 ID (자기참조)"
        varchar(100) cate_name          "카테고리명"
        varchar(10) cate_icon           "아이콘 (이모지)"
        varchar(500) cate_desc          "설명"
        int         sort_ord            "정렬 순서"
        char(1)     use_yn              "사용 여부"
        timestamp   created_at
        timestamp   updated_at
    }

    ec_prod {
        bigint      prod_id         PK  "상품(서비스) ID"
        bigint      cate_id         FK  "카테고리 ID"
        bigint      member_id       FK  "등록 고수 회원 ID"
        varchar(200) prod_name          "서비스명"
        text        prod_desc           "서비스 설명"
        decimal(12,2) price            "가격"
        varchar(50) price_unit          "가격 단위 (원/시간 등)"
        varchar(20) status_cd           "상태 코드 (sy_code)"
        bigint      attach_grp_id   FK  "포트폴리오 이미지 그룹"
        char(1)     del_yn              "삭제 여부"
        timestamp   created_at
        timestamp   updated_at
    }

    ec_order {
        bigint      order_id        PK  "주문(견적요청) ID"
        bigint      client_member_id FK "의뢰인 회원 ID"
        bigint      pro_member_id   FK  "고수 회원 ID"
        bigint      cate_id         FK  "요청 카테고리 ID"
        varchar(20) order_status_cd     "주문 상태 코드 (sy_code)"
        text        req_content         "요청 내용"
        date        req_date            "희망 날짜"
        varchar(200) req_location       "서비스 위치"
        decimal(12,2) total_amt        "총 금액"
        timestamp   created_at
        timestamp   updated_at
    }

    ec_order_item {
        bigint      order_item_id   PK  "주문 항목 ID"
        bigint      order_id        FK  "주문 ID"
        bigint      prod_id         FK  "상품(서비스) ID"
        int         qty                 "수량"
        decimal(12,2) unit_price       "단가"
        decimal(12,2) item_amt         "항목 금액 (qty * unit_price)"
        varchar(300) item_desc          "항목 메모"
        timestamp   created_at
    }

    %% ─────────────── 관계 정의 ───────────────

    sy_attach_grp ||--o{ sy_attach         : "1:N"
    sy_user        ||--o{ sy_login_hist    : "1:N"
    ec_member      ||--o{ sy_login_hist    : "1:N"
    sy_user        ||--o| ec_member        : "1:1"
    ec_member      }o--|| sy_attach        : "프로필이미지"
    ec_cate        ||--o{ ec_cate          : "자기참조(상하위)"
    ec_cate        ||--o{ ec_prod          : "1:N"
    ec_member      ||--o{ ec_prod          : "고수:서비스"
    ec_prod        }o--|| sy_attach_grp    : "포트폴리오"
    ec_member      ||--o{ ec_order         : "의뢰인"
    ec_member      ||--o{ ec_order         : "고수"
    ec_cate        ||--o{ ec_order         : "1:N"
    ec_order       ||--o{ ec_order_item    : "1:N"
    ec_prod        ||--o{ ec_order_item    : "1:N"
```

## 코드 그룹 정의 (`sy_code.code_grp`)

| code_grp | 설명 | code_value 예시 |
|----------|------|----------------|
| `MEMBER_TYPE` | 회원 유형 | `CLIENT`, `PRO` |
| `ORDER_STATUS` | 주문 상태 | `REQUESTED`, `IN_PROGRESS`, `COMPLETED`, `CANCELLED` |
| `PROD_STATUS` | 상품 상태 | `ACTIVE`, `INACTIVE`, `DELETED` |
| `USER_ROLE` | 사용자 역할 | `SUPER_ADMIN`, `ADMIN`, `OPERATOR` |
| `LOGIN_RESULT` | 로그인 결과 | `SUCCESS`, `FAIL_PW`, `FAIL_LOCK`, `FAIL_NOT_FOUND` |

## 관계 요약

```
sy_user ──────── ec_member      (1:1, 시스템 계정 ↔ 서비스 회원)
ec_member ─────── ec_prod       (1:N, 고수 한 명이 여러 서비스 등록)
ec_cate ──────── ec_prod        (1:N, 카테고리별 서비스)
ec_cate ──────── ec_cate        (자기참조, 대/중/소분류)
ec_member ─────── ec_order      (1:N, 의뢰인으로 여러 건 요청)
ec_member ─────── ec_order      (1:N, 고수로 여러 건 수락)
ec_order ──────── ec_order_item (1:N, 주문 내 여러 서비스 항목)
ec_prod ──────── ec_order_item  (1:N, 서비스 상품 → 주문 항목)
sy_attach_grp ── sy_attach      (1:N, 그룹 내 여러 파일)
sy_user ─────── sy_login_hist   (1:N, 로그인 이력)
```
