# 21. 구글 (Firebase + Play Console) 설정

ShopJoy 안드로이드 앱의 Firebase (FCM 푸시) + Google Play Console 설정.

## A. Firebase 프로젝트 (FCM 푸시)

### A.1 콘솔에서 프로젝트 생성

1. https://console.firebase.google.com → **프로젝트 추가**
2. 프로젝트 이름: `shopjoy-prod` (또는 `shopjoy-dev`)
3. Google Analytics: OFF (필요 시 추가)

### A.2 Android 앱 4개 등록

| 환경 | 패키지명 |
|---|---|
| FO Local | `com.shopjoy.fo.local` |
| FO Dev | `com.shopjoy.fo.dev` |
| FO Prod | `com.shopjoy.fo` |
| BO Local | `com.shopjoy.bo.local` |
| BO Dev | `com.shopjoy.bo.dev` |
| BO Prod | `com.shopjoy.bo` |

각각 등록 → **google-services.json** 다운로드.

### A.3 google-services.json 배치

```
_apps/android/ShopjoyFoApp/app/src/local/google-services.json
_apps/android/ShopjoyFoApp/app/src/dev/google-services.json
_apps/android/ShopjoyFoApp/app/src/prod/google-services.json
```

(또는 단일 파일로 prod 만 두고 빌드 시 분기)

### A.4 build.gradle 활성화

`app/build.gradle.kts` 의 주석 해제:

```kotlin
plugins {
    id("com.google.gms.google-services")  // ← 주석 해제
}

dependencies {
    implementation(platform("com.google.firebase:firebase-bom:33.4.0"))
    implementation("com.google.firebase:firebase-messaging-ktx")
}
```

### A.5 SHA-1 / SHA-256 인증서 등록 (선택)

카카오 SDK / Google 로그인 사용 시 필요:

```powershell
# Debug keystore SHA
keytool -list -v -keystore "$env:USERPROFILE\.android\debug.keystore" `
  -alias androiddebugkey -storepass android -keypass android

# Release keystore SHA
keytool -list -v -keystore shopjoy-fo-release.keystore -alias shopjoy-fo
```

출력된 SHA-1 / SHA-256 → Firebase Console → 프로젝트 설정 → Android 앱 → SHA 인증서 추가.

### A.6 테스트 발송

Firebase Console → Cloud Messaging → 새 캠페인 → 테스트 메시지:
- FCM 등록 토큰 입력 (앱 콘솔 로그에서 확인)

## B. Google Play Console (배포)

### B.1 계정 가입

- https://play.google.com/console
- 가입비 USD $25 (1회)
- 개인 / 사업자 선택

### B.2 앱 등록

**앱 만들기** → 정보 입력:

| 항목 | FO 앱 | BO 앱 |
|---|---|---|
| 앱 이름 | ShopJoy | ShopJoy 관리자 |
| 기본 언어 | 한국어 | 한국어 |
| 앱/게임 | 앱 | 앱 |
| 무료/유료 | 무료 | 무료 |

### B.3 필수 콘텐츠 작성

좌측 메뉴 **앱 콘텐츠**:

- [ ] 개인정보처리방침 URL — 자체 호스팅 필수
- [ ] 광고 포함 여부
- [ ] 콘텐츠 등급 설문
- [ ] 타겟 사용자층 (만 18세 이상 / 미성년 포함)
- [ ] 데이터 안전 (수집 데이터 명시)
- [ ] 정부 앱 여부
- [ ] COVID-19 추적 앱 여부
- [ ] 뉴스 앱 여부

### B.4 앱 정보

좌측 메뉴 **메인 스토어 등록정보**:

- 짧은 설명 (80자)
- 자세한 설명 (4000자)
- 앱 아이콘 512×512 PNG (32-bit + 알파)
- 피쳐 그래픽 1024×500
- 폰 스크린샷 2~8장
- 태블릿 스크린샷 (선택)

### B.5 출시 트랙

| 트랙 | 용도 | 검토 시간 |
|---|---|---|
| **Internal testing** | 내부 100명, Google 그룹 | 즉시 |
| **Closed testing** | 외부 비공개 1000명 | 1~2일 |
| **Open testing** | 누구나, Play Store에 베타 표시 | 1~3일 |
| **Production** | 일반 출시 | 1~7일 (드물게 더) |

권장 흐름: **Internal → Closed → Production**.

### B.6 서비스 계정 (CI/CD 자동 업로드)

1. Play Console → **API 액세스** → Google Cloud 프로젝트 연결
2. Service Account 생성 → JSON 키 다운로드
3. Play Console에서 해당 SA에 **Release Manager** 권한 부여
4. CI 환경변수 `GCLOUD_SERVICE_ACCOUNT_CREDENTIALS` 에 JSON 등록

상세 자동 업로드 설정 → [`32.배포하기.md`](32.배포하기.md) 참조.

## C. Google Sign-In (선택)

ShopJoy에 Google 로그인 추가 시:

```kotlin
implementation("com.google.android.gms:play-services-auth:21.2.0")
```

Firebase Console → Authentication → Sign-in method → Google 활성화 + SHA-1 등록.
