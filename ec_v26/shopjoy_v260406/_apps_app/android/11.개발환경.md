# 11. 안드로이드 네이티브 개발환경 구축

ShopJoy 순수 네이티브 안드로이드 앱 (`_apps/android/Shopjoy{Fo,Bo}App`) 개발을 위한 환경.

## 필수 도구

| 도구 | 권장 버전 | 다운로드 |
|---|---|---|
| **JDK** | OpenJDK 17 LTS | https://adoptium.net |
| **Android Studio** | Hedgehog (2023.1) 이상 | https://developer.android.com/studio |
| **Android SDK** | API 24~34 | Android Studio → SDK Manager |
| **Android SDK Build-Tools** | 34.0.0+ | SDK Manager |
| **Android SDK Platform-Tools** | 최신 | SDK Manager |
| **Android Emulator** | 최신 | SDK Manager |
| **Android Virtual Device (AVD)** | API 34 권장 | Device Manager |

## 설치 (Windows)

### 1. JDK 17 설치

```powershell
# Eclipse Temurin 17 설치 (winget)
winget install EclipseAdoptium.Temurin.17.JDK

# 환경변수 설정
[System.Environment]::SetEnvironmentVariable('JAVA_HOME', 'C:\Program Files\Eclipse Adoptium\jdk-17.0.13.11-hotspot', 'User')
```

### 2. Android Studio 설치

- https://developer.android.com/studio 다운로드 → 기본 설치
- 첫 실행 시 SDK 자동 다운로드

### 3. SDK 환경변수 설정

```powershell
[System.Environment]::SetEnvironmentVariable('ANDROID_HOME', "$env:LOCALAPPDATA\Android\Sdk", 'User')
[System.Environment]::SetEnvironmentVariable('ANDROID_SDK_ROOT', "$env:LOCALAPPDATA\Android\Sdk", 'User')

# PATH 추가
$path = [System.Environment]::GetEnvironmentVariable('Path', 'User')
$path += ";$env:LOCALAPPDATA\Android\Sdk\platform-tools"
$path += ";$env:LOCALAPPDATA\Android\Sdk\emulator"
$path += ";$env:LOCALAPPDATA\Android\Sdk\cmdline-tools\latest\bin"
[System.Environment]::SetEnvironmentVariable('Path', $path, 'User')
```

새 PowerShell 창 열고 검증:

```powershell
java --version          # openjdk 17
adb --version           # 1.0.41 이상
emulator -list-avds     # 에뮬레이터 목록
```

## 프로젝트 빌드 검증

```powershell
cd _apps\android\ShopjoyFoApp
.\gradlew assembleLocalDebug
```

성공 시 `app\build\outputs\apk\local\debug\app-local-debug.apk` 생성.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `JAVA_HOME is not set` | JDK 미설정 | 위 1번 단계 재확인 |
| `SDK location not found` | ANDROID_HOME 누락 | 위 3번 단계 재확인 + 셸 재시작 |
| `Gradle daemon ... 멈춤` | 캐시 손상 | `.\gradlew --stop` 후 재시도 |
| `Failed to create Jar file` | IntelliJ Gradle daemon 잠금 | IntelliJ 종료 후 재시도 |
