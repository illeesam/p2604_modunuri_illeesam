# 20. Apple ID 가입 + iOS 인증번호 받기 — 따라하기 (Android 폴더 — 참고)

> 이 문서는 **Android Native 앱** 빌드와는 직접 관련이 없습니다. Android 는 Apple 계정 / 인증서를 사용하지 않습니다.
>
> 다만, ShopJoy 가 **iOS + Android 양쪽 출시** 라면 별도 iOS 인증 절차가 필요하므로 cross-link 용으로 유지합니다.

## Android vs iOS 인증 비교

| 항목 | Android (이 폴더) | iOS |
|---|---|---|
| 앱 식별 | `applicationId` (예: `com.shopjoy.fo`) | **Bundle ID** |
| 팀/조직 | (개인 / Play Console 가입자) | **Team ID** (10자) |
| 푸시 발송 키 | `google-services.json` (FCM) | **APNs 인증 키 (.p8)** + Key ID |
| 코드 서명 | keystore (`.keystore` / `.jks`) | **Distribution Certificate** |
| 빌드 권한 | (Play App Signing 시 자동) | **Provisioning Profile** |
| 배포 콘솔 | Google Play Console | App Store Connect |
| 가입 비용 | Google: $25 (1회) | Apple: $99/년 |

## iOS 도 함께 빌드 / 출시할 경우

ShopJoy 가 **iOS + Android 양쪽 출시 (Capacitor / RN / Flutter / 또는 별도 네이티브 iOS)** 라면, 별도로:

### 1. Apple ID 가입

브라우저에서 https://appleid.apple.com → **Apple ID 생성** → 2단계 인증 활성화.

### 2. Apple Developer Program 가입

https://developer.apple.com/programs/enroll → **Individual** / **Organization** → $99 결제 → 1~2일 승인.

### 4. 양쪽 출시 시 공통 작업

iOS + Android 양쪽 출시 시 다음을 양쪽 모두 등록:

#### Firebase 콘솔
- Android 앱 (`com.shopjoy.fo`) — SHA-1 + SHA-256 등록
- iOS 앱 (`com.shopjoy.fo`) — APNs `.p8` 키 업로드

#### 카카오 Developers
- Android 플랫폼 — 패키지명 + 키 해시
- iOS 플랫폼 — Bundle ID 만 (키 해시 없음)

#### Naver Developers
- Android — 패키지명 + 마켓 URL
- iOS — Bundle ID + URL Scheme

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| Android 만 출시 — iOS 가입 불필요 | 이 문서 무시 OK |
| Cross-platform 출시 시 인증 혼동 | 위의 비교표로 차이 명확히 |
| Firebase 한 프로젝트에 양쪽 등록 시 | 각 플랫폼별로 별도 앱 등록 (같은 프로젝트 안에) |
