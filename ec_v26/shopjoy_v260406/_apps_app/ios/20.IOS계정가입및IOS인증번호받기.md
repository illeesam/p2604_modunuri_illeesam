# 20. Apple ID 가입 + iOS 인증번호 (Team ID / Bundle ID / APNs Key) 받기 — 따라하기

이 문서를 따라하면 Apple ID 가입과 iOS 빌드 / Firebase / 카카오 SDK 등에 등록할 **iOS 인증번호** 를 받을 수 있습니다.

> iOS 는 Android 의 SHA-1 대신 **Team ID + Bundle ID + APNs 인증 키 (.p8) + Distribution Certificate + Provisioning Profile** 을 사용합니다.

## iOS 인증번호 — 한눈에 비교

| 용도 | iOS 식별자 | 발급 위치 |
|---|---|---|
| 앱 식별 | **Bundle ID** (예: `com.shopjoy.fo`) | Apple Developer → Identifiers |
| 팀 식별 | **Team ID** (예: `ABCDE12345`) | Apple Developer → Membership |
| 푸시 발송 | **APNs 인증 키 (.p8)** + Key ID | Apple Developer → Keys |
| 코드 서명 | **Distribution Certificate** | Apple Developer → Certificates |
| 빌드 권한 | **Provisioning Profile** | Apple Developer → Profiles |
| 앱스토어 등록 | **App Store Connect Apple ID** | App Store Connect → 나의 앱 |

## 사전 준비

- [ ] **macOS 14+** (Apple Developer 가입은 어떤 OS 에서도 가능하지만 인증서 발급은 Mac 키체인 필요)
- [ ] Xcode 15.4+ 설치 ([`11.개발환경.md`](11.개발환경.md))
- [ ] 신용카드 (Apple Developer Program 가입 시 — $99/년)

---

## 1단계: Apple ID 가입

### 1-1. Apple ID 생성

브라우저에서 https://appleid.apple.com 접속 → **Apple ID 생성**.

### 1-2. 가입 정보 입력

| 항목 | 입력값 |
|---|---|
| 이메일 | `shopjoy.dev@icloud.com` 또는 본인 이메일 |
| 비밀번호 | 강력한 비밀번호 (12자+, 대문자/숫자/특수문자) |
| 이름 / 성 | 본인 이름 |
| 국가 | 대한민국 |
| 생년월일 | 만 13세 이상 |
| 보안 질문 | 3개 |

→ **계속**.

### 1-3. 이메일 인증

받은 6자리 코드 입력 → **확인** → 가입 완료.

### 1-4. 2단계 인증 활성화 (필수)

1. https://appleid.apple.com 로그인
2. **로그인 및 보안 → 2단계 인증** → **켜기**
3. 신뢰할 수 있는 전화번호 등록
4. 신뢰할 수 있는 기기 (Mac / iPhone) 등록

> ⚠️ Apple Developer / App Store Connect 사용 시 **2단계 인증 필수**.

### 1-5. 운영 / 개인 Apple ID 분리

| 용도 | 권장 |
|---|---|
| Apple Developer Program / App Store Connect | `shopjoy.ops@icloud.com` (운영 공유) |
| 본인 일상 (App Store / iCloud / 음악) | `myname@icloud.com` (개인) |

운영용은 팀에 공유하지 말고, **각자 본인 Apple ID 를 App Store Connect 의 사용자 + 권한 으로 추가** 하는 방식 표준.

## 2단계: Apple Developer Program 가입

App Store 배포가 필요하면 유료 Developer 가입 ($99/년).

### 2-1. 가입 페이지 접속

https://developer.apple.com/programs/enroll → **시작하기**.

### 2-2. Apple ID 로 로그인

위 1단계의 Apple ID 로 로그인.

### 2-3. 멤버십 유형 선택

| 유형 | 비용 | 용도 |
|---|---|---|
| **개인 (Individual)** | $99/년 | 본인 명의 |
| **회사 (Organization)** | $99/년 | 회사 명의 (D-U-N-S Number 필요, 7~14일 소요) |

### 2-4. 약관 동의 + 결제

신용카드 입력 → 가입 완료. **승인까지 1~2 영업일** 소요.

### 2-5. 가입 확인

https://developer.apple.com/account 접속 → 좌측에 메뉴 (Certificates / Identifiers / Profiles / Membership) 가 보이면 OK.

---

## 3단계: Team ID 확인

Team ID 는 Firebase / Codemagic / 외부 서비스에 자주 등록.

### 3-1. Apple Developer Portal

1. https://developer.apple.com/account 접속
2. 우상단 본인 이름 옆 또는 좌측 메뉴 **Membership** 클릭
3. **Team ID** 항목에 10자 영숫자 (예: `ABCDE12345`) 표시 → 복사

### 3-2. Xcode 에서 확인

Xcode → **Settings → Accounts** 탭 → 본인 Apple ID 선택 → 우측 Team 옆 괄호 안 ID.

---

## 4단계: Bundle ID (App Identifier) 등록

### 4-1. Apple Developer Portal → Identifiers

1. https://developer.apple.com/account → 좌측 **Certificates, Identifiers & Profiles** → **Identifiers**
2. 우측 **+** 버튼

### 4-2. App IDs 선택

**App IDs** → **Continue** → **App** → **Continue**.

### 4-3. Bundle ID 입력

| 항목 | 값 |
|---|---|
| Description | `ShopJoy FO Production` |
| Bundle ID | **Explicit** 선택 → `com.shopjoy.fo` |

### 4-4. Capabilities 체크

사용할 기능 모두 체크:
- ✅ **Push Notifications** — FCM 사용 시 필수
- ✅ **Sign in with Apple** — Apple 로그인 사용 시
- ✅ **Associated Domains** — Universal Link 사용 시

→ **Continue** → **Register**.

### 4-5. ShopJoy 환경별 6개 등록

| 환경 | Bundle ID |
|---|---|
| FO Local | `com.shopjoy.fo.local` |
| FO Dev | `com.shopjoy.fo.dev` |
| FO Prod | `com.shopjoy.fo` |
| BO Local | `com.shopjoy.bo.local` |
| BO Dev | `com.shopjoy.bo.dev` |
| BO Prod | `com.shopjoy.bo` |

각각 4-1 ~ 4-4 반복.

> 환경 분리 안 하면 prod 1개만 등록해도 됨.

---

## 5단계: APNs 인증 키 (.p8) 발급

푸시 알림 발송 시 핵심. **한 키로 여러 앱 / 환경 모두 사용 가능**.

### 5-1. Apple Developer Portal → Keys

좌측 **Keys** → **+** 버튼.

### 5-2. 키 정보 입력

| 항목 | 값 |
|---|---|
| Key Name | `ShopJoy APNs Key` |
| Apple Push Notifications service (APNs) | ✅ 체크 |

**Configure** 버튼 클릭:
- **Sandbox & Production** 선택 (개발 / 운영 모두)
- **Apps** → **All Apps** 또는 특정 Bundle ID 선택
- **Save**

→ **Continue** → **Register**.

### 5-3. .p8 파일 다운로드

**Download** 버튼 클릭 → `AuthKey_ABCD123456.p8` 다운로드.

> ⚠️ **이 .p8 파일은 한 번만 다운로드 가능**. 분실 시 키 재발급 필요. 안전한 위치에 보관 (1Password 등).

화면에 표시된 정보 메모:
- **Key ID** (10자 영숫자, 예: `ABCD123456`)
- **Team ID** (3단계에서 확인한 값)

---

## 6단계: Distribution Certificate 발급

App Store 배포 시 필요한 인증서.

### 6-1. 키체인에서 CSR 생성

Mac:
1. **Launchpad → 기타 → 키체인 접근** 실행
2. 메뉴바 → **키체인 접근 → 인증서 지원 → 인증 기관에서 인증서 요청**
3. 정보 입력:
   - 사용자 이메일: 본인 Apple ID 이메일
   - 일반 이름: `ShopJoy Distribution`
   - **요청 항목 → 디스크에 저장됨** 선택 + **본인 키 쌍 정보 지정** ✅
4. **계속** → 데스크톱에 `CertificateSigningRequest.certSigningRequest` 저장
5. 키 쌍 — **2048 비트 / RSA** → **계속** → 완료

### 6-2. Apple Developer Portal 에 업로드

1. **Certificates** → **+**
2. **Apple Distribution** 선택 → **Continue**
3. 방금 만든 `.certSigningRequest` 파일 업로드 → **Continue**
4. **Download** → `distribution.cer` 다운로드

### 6-3. 키체인에 등록

다운로드한 `.cer` 파일 더블클릭 → 키체인 접근이 자동 실행되어 **로그인** 키체인에 추가됨.

키체인 접근 → **로그인** 카테고리 → **내 인증서** → "Apple Distribution: 본인이름 (TEAM_ID)" 항목 표시 + 화살표 클릭 시 개인 키도 함께 보이면 OK.

---

## 7단계: Provisioning Profile 발급

각 환경 (Local/Dev/Prod) 별로 발급 가능 또는 Prod 만.

### 7-1. Profiles → +

Apple Developer Portal → **Profiles** → **+**.

### 7-2. 유형 선택

- **App Store** — App Store / TestFlight 배포용
- **Ad Hoc** — 100명 이내 사내 베타 배포
- **Development** — 개발자 본인 기기 디버그

→ 일반적으로 **App Store** 선택 → **Continue**.

### 7-3. App ID 선택

`com.shopjoy.fo` (또는 다른 Bundle ID) → **Continue**.

### 7-4. Certificate 선택

방금 만든 Distribution Certificate → **Continue**.

### 7-5. 이름 입력

Profile Name: `ShopJoy FO Prod App Store` → **Generate**.

### 7-6. 다운로드 + Xcode 등록

**Download** → `ShopJoy_FO_Prod_App_Store.mobileprovision` → 더블클릭 → Xcode 자동 등록.

---

## 8단계: App Store Connect 등록

### 8-1. App Store Connect 접속

https://appstoreconnect.apple.com → 위 1단계의 Apple ID 로 로그인.

### 8-2. 새 앱 만들기

**나의 앱 → +**:

| 항목 | 값 |
|---|---|
| 플랫폼 | iOS |
| 이름 | ShopJoy |
| 기본 언어 | 한국어 |
| Bundle ID | `com.shopjoy.fo` (4단계에서 등록한 ID) |
| SKU | `SHOPJOY-FO-001` (임의 고유) |
| 사용자 액세스 | 전체 |

→ **만들기**.

### 8-3. 앱 정보 작성

좌측 메뉴:
- **앱 정보**: 카테고리 / 콘텐츠 권한 / 저작권
- **가격 및 사용 가능 여부**: 무료
- **앱 개인정보 보호**: 데이터 수집 양식

---

## 9단계: App Store Connect API Key (CI/CD 자동화)

Codemagic / Fastlane 자동 빌드 시 필요.

### 9-1. API Key 생성

1. App Store Connect → 좌측 **사용자 및 액세스** → **통합** 탭 → **App Store Connect API**
2. **API 키 생성** 또는 **+**
3. 이름: `ShopJoy CI`, 액세스: **App Manager** (또는 Admin)
4. **생성** → `.p8` 파일 다운로드 (한 번만)
5. **Issuer ID** + **Key ID** 메모

### 9-2. CI 환경변수 등록

Codemagic 의 경우:
- 통합 → Apple Developer Portal → API Key 등록
- `.p8` 파일 + Issuer ID + Key ID 입력

상세 → [`6.iOS빌드및Codemagic.md`](../../_doc/정책서-Capacitor/6.iOS빌드및Codemagic.md)

## 10단계: 인증번호를 어디에 등록하는가?

### 10-1. Firebase Console (FCM 푸시 발송)

[`21.구글설정하기.md`](21.구글설정하기.md) 참조:

1. https://console.firebase.google.com → 프로젝트 → ⚙ 프로젝트 설정 → **클라우드 메시징** 탭
2. **Apple 앱 구성** 섹션 → **APNs 인증 키 업로드**
3. 입력:
   - `.p8` 파일 (5단계에서 발급)
   - **Key ID**
   - **Team ID** (3단계)

### 10-2. Codemagic / Fastlane (자동 빌드)

- **Team ID**, **Bundle ID**, **API Key** 등록
- 인증서 / 프로파일은 `app-store-connect fetch-signing-files` 자동 처리

### 10-3. Kakao Developers (카카오 SDK iOS)

1. https://developers.kakao.com → 내 애플리케이션
2. **플랫폼 → iOS 플랫폼 등록**
3. **Bundle ID** 입력 (각 환경별 / `com.shopjoy.fo`, `com.shopjoy.fo.dev` 등)
4. iOS 는 **키 해시 등록 없음** (Bundle ID 만 사용)

### 10-4. Naver Developers (네이버 로그인 iOS)

https://developers.naver.com → 애플리케이션 등록 → iOS 설정 → URL Scheme + Bundle ID.

### 10-5. Universal Link (관련 도메인 사용 시)

`https://shopjoy.com/.well-known/apple-app-site-association` 파일 업로드 + Apple Developer 의 App ID 에 **Associated Domains** Capability 활성.

## 11단계: 검증

### 11-1. Xcode 자동 서명 동작

```bash
cd _apps/ios/ShopjoyFoApp
xcodegen generate
open ShopjoyFoApp.xcodeproj
```

Xcode → 타겟 → **Signing & Capabilities** → 다음 모두 OK?

- [ ] ✅ Automatically manage signing
- [ ] Team: 본인 팀 선택됨
- [ ] Bundle Identifier: `com.shopjoy.fo` (또는 환경별)
- [ ] **Status**: "Provisioning Profile: ShopJoy FO Prod App Store" 자동 인식

### 11-2. 실기기 빌드

```bash
flutter run                # Flutter
npm run ios:local          # Capacitor / RN
```

또는 Xcode 에서 ▶ — 에러 없으면 인증 정보 정상.

### 11-3. APNs 토큰 발급

iPhone 실기기 (시뮬레이터 X) 에서 앱 실행 → 콘솔에 APNs / FCM 토큰 출력 확인.

### 11-4. Firebase 테스트 푸시

Firebase Console → Cloud Messaging → 새 캠페인 → 테스트 메시지 → 토큰 입력 → 전송 → iPhone 알림 도착.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `No matching profiles found` | Provisioning Profile 미발급 | Step 7 재확인 |
| `Provisioning profile doesn't include the device` | Personal Team (무료) 7일 만료 | 다시 ▶ — 자동 재서명 |
| `Distribution certificate already exists` | 인증서 중복 | 키체인에서 기존 삭제 후 재발급 |
| Firebase 테스트 푸시 미수신 | APNs 키 미업로드 / Bundle ID 불일치 | Step 10-1 재확인 |
| `.p8` 파일 분실 | 한 번만 다운로드 가능 | Apple Developer 에서 키 revoke + 재발급 → Firebase 도 재업로드 |
| App Store Connect 앱이 안 보임 | Bundle ID 가 Identifiers 에 미등록 | Step 4 먼저 |
