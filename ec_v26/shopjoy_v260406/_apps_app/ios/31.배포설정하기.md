## 1단계: 버전 / 빌드 번호 정책

### 1-1. project.yml 의 버전 확인

```yaml
settings:
  base:
    MARKETING_VERSION: "1.0.0"
    CURRENT_PROJECT_VERSION: "1"
```

| 항목 | 형식 | 변경 시점 |
|---|---|---|
| **MARKETING_VERSION** | `MAJOR.MINOR.PATCH` | 기능 추가/수정 |
| **CURRENT_PROJECT_VERSION** | 정수 또는 점표기 | 빌드마다 +1 |

### 1-2. 자동 증가 스크립트 (선택)

`_apps/ios/ShopjoyFoApp/scripts/bump-build.sh`:

```bash
#!/bin/bash
cd "$(dirname "$0")/.."
xcodegen generate
cd ShopjoyFoApp.xcodeproj/..
agvtool next-version -all
```

```bash
chmod +x scripts/bump-build.sh
./scripts/bump-build.sh
```

## 2단계: 앱 아이콘 생성

### 2-1. 1024×1024 PNG 준비

투명 배경 X / 모서리 둥글게 X / 알파 채널 없음.

본인 로고를 `icon-1024.png` 로 저장.

### 2-2. 자동 생성 도구 설치

```bash
brew install --cask appicon
```

### 2-3. 아이콘 생성

**App Icon Generator** 실행 → 1024×1024 PNG 드래그 → 자동으로 모든 크기 생성.

또는 명령행 (`mkicon` 등) 사용.

### 2-4. Xcode 에 등록

생성된 폴더를 `_apps/ios/ShopjoyFoApp/ShopjoyFoApp/Assets.xcassets/AppIcon.appiconset/` 에 복사.

또는 Xcode → **Assets.xcassets → AppIcon** 에 PNG 들 드래그.

## 4단계: Info.plist 권한 사유 작성

`ShopjoyFoApp/Info.plist` 의 사용 사유 문구 검토:

| 키 | 문구 |
|---|---|
| `NSCameraUsageDescription` | 상품 사진 촬영을 위해 카메라 접근 권한이 필요합니다. |
| `NSPhotoLibraryUsageDescription` | 갤러리에서 상품 이미지를 선택하기 위해 권한이 필요합니다. |
| `NSUserNotificationsUsageDescription` | 주문/배송 알림을 받기 위해 권한이 필요합니다. |
| `NSFaceIDUsageDescription` (BO) | 관리자 빠른 로그인을 위해 Face ID를 사용합니다. |

> ⚠️ **사용 사유 문구 누락 시 앱이 권한 요청 시점에 크래시**.

`project.yml` 에 이미 작성되어 있으므로 `xcodegen generate` 후 자동 반영.

## 5단계: ATS (App Transport Security) 설정

운영은 **HTTPS 강제**, 개발/로컬만 HTTP 허용.

`project.yml` 의 Info.plist 섹션:

```yaml
NSAppTransportSecurity:
  NSAllowsArbitraryLoads: false   # ← 운영에서는 false
  NSExceptionDomains:
    illeesam.netlify.app:
      NSIncludesSubdomains: true
      NSExceptionAllowsInsecureHTTPLoads: false
```

> **로컬 개발 시에만** `NSAllowsArbitraryLoads: true` (Debug Configuration 만).

## 6단계: Capabilities 검증

Xcode → 타겟 → **Signing & Capabilities** 탭에서 다음이 추가되어 있어야 함:

- ✅ **Push Notifications**
- ✅ **Background Modes** → ☑ Remote notifications

다른 기능 사용 시:
- HealthKit / HomeKit / iCloud / Sign in with Apple 등 → **+ Capability** 로 추가

## 8단계: App Store Connect 메타데이터 작성

### 8-1. 스크린샷 준비

| 디바이스 | 해상도 | 매수 |
|---|---|---|
| 6.7" iPhone (Pro Max) | 1290×2796 | 2~6 |
| 6.5" iPhone | 1242×2688 | (선택) |
| 5.5" iPhone | 1242×2208 | (선택) |
| 12.9" iPad Pro | 2048×2732 | (iPad 지원 시) |

시뮬레이터에서 캡처:
```bash
xcrun simctl io booted screenshot ~/Desktop/screenshot1.png
```

### 8-2. 짧은 설명 / 긴 설명

- 앱 이름: 30자 이내
- 부제: 30자 이내 (선택)
- 프로모션 텍스트: 170자 이내
- 설명: 4000자 이내
- 키워드: 100자 (쉼표 구분)
- 지원 URL / 마케팅 URL

### 8-3. 앱 미리보기 비디오 (선택)

15~30초, 시뮬레이터 화면 녹화 가능:

```bash
xcrun simctl io booted recordVideo ~/Desktop/preview.mov
# 녹화 끝낼 때 Ctrl+C
```

## 9단계: 앱 심사 정보 작성

App Store Connect → 앱 → 좌측 **App 심사 정보** 메뉴.

### 9-1. 연락처 정보

- 성 / 이름 / 전화번호 / 이메일

### 9-2. 데모 계정 (BO 앱 필수)

관리자 앱처럼 로그인이 필요한 앱은 Apple 심사관용 데모 계정 필수:

- 사용자명: `apple-reviewer@shopjoy.com`
- 비밀번호: `Test1234!`

### 9-3. 메모

특수 기능 (Face ID 사용, 외부 결제 등) 설명.

## 10단계: 데이터 수집 양식

App Store Connect → 앱 정보 → **앱 개인정보 보호** → **시작하기**.

수집하는 데이터 정직 신고:

- 이름 / 이메일 / 휴대폰 번호 → 회원 가입
- 결제 정보 → 주문 (제3자 PG 처리)
- 위치 (선택) → 배송지 검색
- 사용 데이터 (분석 / 제품 상호작용) → Firebase Analytics

> 거짓 신고 시 앱 거절.
