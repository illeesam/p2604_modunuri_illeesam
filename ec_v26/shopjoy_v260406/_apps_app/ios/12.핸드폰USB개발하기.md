# 12. iPhone USB 로 개발/디버그하기 — 따라하기

iPhone 실기기에 ShopJoy 앱을 설치하고 디버깅하는 절차.

## 사전 준비

- [ ] [`11.개발환경.md`](11.개발환경.md) 완료
- [ ] iPhone 또는 iPad 실기기
- [ ] Lightning 또는 USB-C 데이터 케이블 (충전 전용 X)
- [ ] Apple ID (Personal Team 또는 유료 Developer 계정)

---

## 1단계: iPhone 을 Mac 에 USB 연결

### 1-1. 케이블로 연결

iPhone — Lightning/USB-C 케이블 — Mac.

### 1-2. iPhone 화면 잠금 해제

### 1-3. 신뢰 다이얼로그

iPhone 화면에 **이 컴퓨터를 신뢰하시겠습니까?** 표시됨 → **신뢰** 탭.

iPhone 비밀번호 입력.

---

## 2단계: Xcode 에 Apple ID 등록

### 2-1. Xcode 실행

```bash
open -a Xcode
```

### 2-2. 계정 추가

Xcode 메뉴 → **Settings...** → **Accounts** 탭 → 좌하단 **+** 버튼.

### 2-3. Apple ID 선택

**Apple ID** → **Continue** → 본인 Apple ID + 비밀번호 입력 → **Sign In**.

### 2-4. Team 확인

좌측 목록에 본인 Apple ID 가 추가되고, 우측에 **Personal Team** (또는 회사 Team) 표시.

---

## 3단계: 프로젝트에 Team 설정

### 3-1. 프로젝트 열기

```bash
cd _apps/ios/ShopjoyFoApp
xcodegen generate
open ShopjoyFoApp.xcodeproj
```

### 3-2. 타겟 클릭

좌측 Project Navigator 에서 **ShopjoyFoApp** (파란 아이콘) 클릭 → 우측 패널에서 **TARGETS → ShopjoyFoApp** 선택.

### 3-3. Signing 설정

상단 탭 **Signing & Capabilities** 클릭:

- **Automatically manage signing** ✅ 체크
- **Team** 드롭다운 → 본인 팀 선택

오류가 사라지면 OK.

---

## 4단계: 디바이스 선택

### 4-1. Xcode 좌상단 디바이스 드롭다운

연결된 iPhone 이 목록에 표시됨 → 클릭.

처음이면 Xcode 가 **Preparing iPhone for Development...** 표시 → 1~3분 대기.

---

## 5단계: ▶ 빌드 + 설치

### 5-1. Scheme 선택

좌상단 Scheme 드롭다운 → **ShopjoyFoApp-Local** (또는 Dev/Prod) 선택.

### 5-2. ▶ 클릭

상단 ▶ 버튼 (Cmd+R) → 빌드 + 자동 설치 + 자동 실행.

### 5-3. 처음 실행 시 신뢰

iPhone 첫 실행 시 **신뢰할 수 없는 개발자** 표시되면:

1. iPhone → **설정 → 일반 → VPN 및 기기 관리**
2. 본인 Apple ID 항목 클릭
3. **신뢰** 버튼 탭
4. 다시 ShopJoy 앱 실행

---

## 6단계: WebView 디버깅 (Safari Web Inspector)

### 6-1. iPhone Safari 웹 속성 활성화

iPhone 설정 → **Safari → 고급 → 웹 속성** ON.

### 6-2. Mac Safari 개발자 메뉴 활성화

Mac Safari → **Safari → 환경설정 → 고급** → **메뉴 막대에 개발자용 메뉴 표시** 체크.

### 6-3. ShopJoy 앱 실행 중인 상태에서

Mac Safari 메뉴바 → **개발자용 → [iPhone 이름] → ShopJoy.app**

→ Web Inspector 창 열림. 데스크톱 DevTools 와 동일하게 사용.

---

## 7단계: 호스트 Mac 의 Live Server 접근 (local 환경)

`local` 스킴은 `http://127.0.0.1:5501` 인데, **iPhone 실기기는 Mac 의 127.0.0.1 에 접근할 수 없습니다**.

### 7-1. Mac LAN IP 확인

```bash
ipconfig getifaddr en0
```

예: `192.168.0.50`

### 7-2. project.yml 임시 수정

`_apps/ios/ShopjoyFoApp/project.yml` 의 `Debug-Local` 블록:

```yaml
Debug-Local:
  INFOPLIST_KEY_BASE_URL: "http://192.168.0.50:5501"   # ← 본인 Mac IP
```

### 7-3. 재생성 + 다시 ▶

```bash
xcodegen generate
```

Xcode 에서 다시 ▶ → iPhone 에서 ShopJoy 가 Mac 의 Live Server 와 통신.

### 7-4. Mac 방화벽 허용

시스템 설정 → 네트워크 → 방화벽 → 옵션 → VS Code 또는 Live Server 허용.

---

## 8단계: 무선 디버깅 (USB 분리)

### 8-1. USB 연결 상태에서

Xcode → **Window → Devices and Simulators** (Cmd+Shift+2)

좌측 iPhone 선택 → **Connect via network** ✅ 체크.

### 8-2. USB 분리

이제 USB 케이블을 빼도 Xcode 가 iPhone 인식 (같은 Wi-Fi 네트워크일 때).

---

## 9단계: 콘솔 로그

Xcode → **Window → Devices and Simulators** → iPhone 선택 → **Open Console** 클릭.

또는 macOS 의 **Console.app** 실행 → 좌측에서 iPhone 선택.

ShopJoy 의 `print(...)` 출력이 실시간 표시.

---

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| iPhone 이 Xcode 에 안 보임 | 케이블 / 신뢰 / Xcode 재시작 |
| `Could not locate device support files` | Xcode 업데이트 또는 iOS 버전 호환성 확인 |
| `Code signing required` | Step 3 의 Team 설정 재확인 |
| `Provisioning profile doesn't include the device` | Personal Team 은 7일마다 재인증 필요 → 다시 ▶ |
| 첫 실행 시 흰 화면 | local 환경인데 LAN IP 미수정 + Mac 방화벽 |
| 앱이 자동 종료됨 | Xcode → Stop 안 누르고 케이블 분리 → 정상 |
