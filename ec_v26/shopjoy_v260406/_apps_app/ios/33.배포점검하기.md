# 33. 배포 점검하기 (iOS) — 따라하기

iOS 앱 출시 전후 점검 체크리스트.

## A. 빌드 전 (출시 1일 전)

### A-1. 버전 확인

```bash
cd _apps/ios/ShopjoyFoApp
grep -E "MARKETING_VERSION|CURRENT_PROJECT_VERSION" project.yml
```

다음 모두 OK?

- [ ] `MARKETING_VERSION` 이 이전 버전보다 큼
- [ ] `CURRENT_PROJECT_VERSION` 이 이전보다 큼

### A-2. 운영 환경 가리키는지 확인

```bash
grep -A1 "Release-Prod" project.yml | grep INFOPLIST_KEY_BASE_URL
```

운영 URL (`https://illeesam.netlify.app/...`) 인지 확인.

### A-3. 디버그 코드 정리

```bash
grep -r "print(" ShopjoyFoApp/ --include="*.swift" | wc -l
```

너무 많으면 정리 (Release 빌드에서 자동 제거되는 도구 사용 권장).

### A-4. Info.plist 권한 사유 작성 확인

```bash
grep -E "NS\w+UsageDescription" project.yml
```

사용하는 모든 권한에 사유 문구 있는지 확인.

### A-5. APNs / Firebase 키 운영용 사용

- [ ] `GoogleService-Info.plist` 가 운영 Firebase 프로젝트 키
- [ ] APNs 키 운영용 등록 (Sandbox + Production)
- [ ] 카카오/토스 결제 키 운영 가맹점 ID

## B. 빌드 후 (Archive 직후)

### B-1. Archive 검증

Xcode → **Window → Organizer** → 방금 만든 Archive 선택.

오른쪽 패널 → **Validate App** 버튼 클릭.

자동 검증:
- [ ] 인증서 / Provisioning Profile 정합성
- [ ] Bundle ID 일치
- [ ] 권한 / Capability 일치
- [ ] Asset / 아이콘 누락 없음

### B-2. 실기기 수동 테스트

다음 플로우 모두 정상 동작?

- [ ] 앱 설치 → 첫 실행 → 권한 요청 정상
- [ ] 회원 가입 / 로그인 / 자동 로그인
- [ ] 상품 검색 / 조회 / 상세
- [ ] 장바구니 추가 / 수정 / 삭제
- [ ] 주문 → 결제 (카카오 / 토스)
- [ ] 결제 성공 시 딥링크 복귀 (`shopjoy-fo://pay/success`)
- [ ] 결제 실패 / 취소 처리
- [ ] 푸시 권한 다이얼로그 → 허용
- [ ] FCM 푸시 발송 → 포그라운드 / 백그라운드 모두 수신
- [ ] 푸시 탭 시 정확한 화면 이동
- [ ] 앱 종료 후 재실행 시 토큰 유지 (자동 로그인)

### B-3. 디바이스 / OS 매트릭스

- [ ] iPhone SE 3 (4.7") — 작은 화면
- [ ] iPhone 15 (6.1") — 표준
- [ ] iPhone 15 Pro Max (6.7") — 큰 화면
- [ ] iPad (선택, 지원 시)
- [ ] iOS 14 (minimum) / iOS 17 (latest)

### B-4. 다양한 조건

- [ ] 다크 모드
- [ ] 가로 회전
- [ ] 한국어 / 영어 (다국어 지원 시)
- [ ] 오프라인 → 온라인 복귀
- [ ] 저속 네트워크 (개발자 옵션 → Network Link Conditioner)
- [ ] 백그라운드 → 포그라운드 (앱 메모리 회수 시 복원)
- [ ] 1시간 이상 사용 후 메모리 누수 확인

## D. 출시 후 모니터링 (D+1, D+3, D+7)

### D-1. App Store Connect 통계

App Store Connect → **분석** 탭:

| 지표 | 안전 임계값 |
|---|---|
| 크래시 수 | < 1% (사용자당) |
| 평점 | > 4.0 |
| 1성 리뷰 | 즉시 응답 |
| 설치 → 활성 사용 비율 | > 70% |

### D-2. Crashlytics (Firebase)

Firebase Console → **Crashlytics** → 실시간 크래시 추적.

```swift
// 활성화: AppDelegate.swift
import FirebaseCrashlytics

Crashlytics.crashlytics().setCustomValue(BuildConfig.APP_ENV, forKey: "env")
```

### D-3. 사용자 리뷰

App Store Connect → **App Store → 평점 및 리뷰**:

- ☑ 1~3성 리뷰 → 24시간 내 답글
- 패턴 분석 → 공통 불만 사항 핫픽스

### D-4. 단계적 출시 일시정지 (사고 시)

크래시율 1% 초과 시:

1. App Store Connect → 출시 → **Phased Release** → **Pause** 클릭
2. 이미 받은 사용자에게는 그대로, 신규 다운로드 차단
3. 핫픽스 빌드 → TestFlight 검증 → 새 버전 출시

## E. 사고 대응 (긴급)

### E-1. 즉시 조치

크래시율 5% 초과 / 결제 안 됨 / 데이터 유실:

1. **단계적 출시 일시정지**
2. **TestFlight 베타** 의 안정 버전으로 사내 검증
3. 핫픽스 코드 작성

### E-2. 핫픽스 출시

```bash
git checkout -b hotfix/v1.0.2
# 코드 수정
git tag fo-v1.0.2
git push origin fo-v1.0.2
```

Codemagic 자동 빌드 → TestFlight 업로드 → 사내 30분 검증 → App Store 제출 → **Expedited Review 요청** (긴급 검토, 24시간 단축).

### E-3. 메타데이터만 수정 (코드 미변경)

App Store Connect → **수정 가능한 항목** 만 수정 (설명, 스크린샷 등) → **저장** → 즉시 반영.
