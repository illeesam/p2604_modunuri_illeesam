## A. Android Release AAB 빌드

### A-1. JS 번들 생성 (자동, 명시적으로 강제 가능)

```bash
cd _apps/reactNative/ShopjoyFoApp
ENVFILE=.env.prod npx react-native bundle \
  --platform android \
  --dev false \
  --entry-file index.js \
  --bundle-output android/app/src/main/assets/index.android.bundle \
  --assets-dest android/app/src/main/res
```

### A-2. AAB 빌드

```bash
cd android
./gradlew bundleProdRelease
```

결과: `android/app/build/outputs/bundle/prodRelease/app-prod-release.aab`

### A-3. APK 빌드 (사내 직접 배포용)

```bash
./gradlew assembleProdRelease
```

결과: `android/app/build/outputs/apk/prod/release/app-prod-release.apk`

## C. iOS Archive (macOS)

### C-1. 운영 환경 설정

```bash
cd _apps/reactNative/ShopjoyFoApp
ENVFILE=.env.prod ./node_modules/.bin/react-native bundle \
  --platform ios \
  --dev false \
  --entry-file index.js \
  --bundle-output ios/main.jsbundle \
  --assets-dest ios
```

### C-2. Xcode Archive

```bash
open ios/ShopjoyFoApp.xcworkspace
```

Xcode:
1. 좌상단 디바이스 → **Any iOS Device (arm64)**
2. Scheme → `ShopjoyFoApp-Prod`
3. **Product → Archive**

### C-3. Distribute App → App Store Connect → Upload

자동 서명 → 업로드 (5~15분).

## E. 자동 빌드 (Codemagic)

### E-1. codemagic.yaml 추가

저장소 루트의 `codemagic.yaml` 에 RN 워크플로우 추가:

```yaml
rn-fo-android-release:
  name: RN FO — Android Release
  instance_type: mac_mini_m2
  environment:
    vars:
      APP_DIR: "_apps/reactNative/ShopjoyFoApp"
    groups:
      - shopjoy_keystore
      - google_play
    node: 20.18.0
    java: 17
  triggering:
    events: [tag]
    tag_patterns:
      - pattern: 'rn-fo-v*'
  scripts:
    - cd $APP_DIR && npm ci
    - cd $APP_DIR && cd ios && pod install && cd ..
    - cd $APP_DIR/android && ENVFILE=.env.prod ./gradlew bundleProdRelease
  artifacts:
    - $APP_DIR/android/app/build/outputs/bundle/prodRelease/*.aab
  publishing:
    google_play:
      credentials: $GCLOUD_SERVICE_ACCOUNT_CREDENTIALS
      track: internal
```

### E-2. 자동 빌드

```bash
git tag rn-fo-v1.0.0
git push origin rn-fo-v1.0.0
```
