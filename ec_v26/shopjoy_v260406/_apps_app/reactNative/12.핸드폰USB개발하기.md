## A. Android 핸드폰

### A-1. USB 디버깅 활성화

핸드폰: 설정 → 휴대전화 정보 → 빌드번호 7번 탭 → 개발자 옵션 → **USB 디버깅** ON.

PC 연결 + 신뢰 다이얼로그 허용.

### A-2. 인식 확인

```bash
adb devices -l
```

`device` 상태 확인.

### A-3. local 환경 빌드 + 설치

별도 터미널에서 Metro 시작:
```bash
cd _apps/reactNative/ShopjoyFoApp
npm start
```

다른 터미널에서:
```bash
npm run android:local
```

자동 진행:
1. `.env.local` 환경변수 로드
2. Gradle 빌드
3. 핸드폰 자동 설치 + 실행

### A-4. PC LAN IP 로 변경 (실기기 전용)

`.env.local` 의 BASE_URL 은 `http://10.0.2.2:5501` (에뮬레이터용).

실기기는 PC LAN IP 사용:

```bash
ipconfig | Select-String "IPv4"
# 예: 192.168.0.100
```

`.env.local` 수정:
```
BASE_URL=http://192.168.0.100:5501
```

다시 `npm run android:local`.

PC 방화벽 → 5501 포트 허용.

## B. iOS iPhone (macOS)

### B-1. iPhone USB 연결 + 신뢰

### B-2. Metro 시작

```bash
cd _apps/reactNative/ShopjoyFoApp
npm start
```

### B-3. local 환경 빌드 + 설치

```bash
npm run ios:local
```

또는 Xcode 에서 직접:
```bash
open ios/ShopjoyFoApp.xcworkspace
```

좌상단 디바이스 → 본인 iPhone → Scheme `ShopjoyFoApp-Local` → ▶.

## C. WebView 디버깅

### C-1. Android (Chrome DevTools)

`chrome://inspect` → ShopjoyFoApp → **inspect**.

### C-2. iOS (Safari Web Inspector)

iPhone 설정 → Safari → 고급 → 웹 속성 ON.
Mac Safari → 개발자용 메뉴 → [iPhone 이름] → ShopjoyFoApp.

### C-3. RN 자체 Dev Menu

핸드폰에서 흔들기 (또는 `Ctrl+M` Android, `Cmd+D` iOS Sim) → Dev Menu:
- Reload
- Debug
- Element Inspector

---

## D. Hot Reload

Metro 가 실행 중이면, App.tsx 등 RN 코드 수정 시 → **Fast Refresh** 자동 동작.

WebView 내부 (웹 코드) 는 Live Server 가 별도로 처리.

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `adb devices` 빈 목록 | USB 디버깅 / 케이블 / 다이얼로그 |
| Metro 가 8081 포트 사용 중 | `npx react-native start --port 8082` |
| Android 흰 화면 | LAN IP 미수정 + 방화벽 |
| `Cannot find module 'react-native-config'` | `cd ios && pod install` |
| iOS 빌드 시 `Pods.xcodeproj missing` | `cd ios && pod install` |
