## A. VS Code (가장 추천)

### A-1. Flutter 확장 설치

VS Code → 확장 → `Flutter` (Dart-Code) 검색 → Install.

### A-2. 프로젝트 열기

```bash
cd _apps/flutter/ShopjoyFoApp
code .
```

### A-3. 디바이스 선택

VS Code 우하단 상태바 → 디바이스 선택 (예: `Pixel_7_API_34 (android)`).

### A-4. 환경 선택 (Launch Configuration)

**Run → Add Configuration → Flutter** 선택.

`.vscode/launch.json` 생성:

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Local",
      "request": "launch",
      "type": "dart",
      "program": "lib/main.dart",
      "args": ["--dart-define=APP_ENV=local"]
    },
    {
      "name": "Dev",
      "request": "launch",
      "type": "dart",
      "program": "lib/main.dart",
      "args": ["--dart-define=APP_ENV=dev"]
    },
    {
      "name": "Prod",
      "request": "launch",
      "type": "dart",
      "program": "lib/main.dart",
      "args": ["--dart-define=APP_ENV=prod"]
    }
  ]
}
```

### A-5. ▶ 실행

좌측 사이드바 ▶ 디버그 아이콘 → 상단 드롭다운 (`Local` / `Dev` / `Prod`) 선택 → ▶.

### A-6. Hot Reload

코드 수정 후 저장 → 자동 Hot Reload.

수동: 디버그 컨트롤 바의 ⚡ 아이콘.

## B. Android Studio

### B-1. 프로젝트 열기

Android Studio → Open → `_apps/flutter/ShopjoyFoApp` 폴더.

### B-2. Flutter 플러그인 설치

설정 → Plugins → `Flutter` 검색 → Install → Restart.

### B-3. 디바이스 + 환경 선택

좌상단 디바이스 드롭다운 → AVD 또는 실기기 선택.

좌상단 Configuration 드롭다운 → **Edit Configurations** → **+** → Flutter:
- Name: `Local`
- Build flavor: (필요 시)
- Additional run args: `--dart-define=APP_ENV=local`

### B-4. ▶

상단 ▶ 클릭 (Shift+F10).

## C. Xcode (iOS, macOS 만)

Flutter 는 Xcode 가 필요한 작업이 제한적 (보통 VS Code 만 사용):

- Pod 의존성: `cd ios && pod install`
- 인증서 / 프로파일 / Capability: Xcode → `ios/Runner.xcworkspace` 열기 → 타겟 → Signing
- Archive: `flutter build ipa` 명령으로 충분

```bash
open _apps/flutter/ShopjoyFoApp/ios/Runner.xcworkspace
```

## D. 환경 전환

### D-1. local → dev → prod

```bash
flutter run --dart-define=APP_ENV=local
flutter run --dart-define=APP_ENV=dev
flutter run --dart-define=APP_ENV=prod
```

또는 VS Code Launch Configuration 에서 선택.

### D-2. APP_ENV 값에 따라

`lib/env.dart` 가 자동으로 `.env.{local|dev|prod}` 로드.

```dart
static String get envFile {
  switch (_appEnv) {
    case 'prod': return '.env.prod';
    case 'dev':  return '.env.dev';
    default:     return '.env.local';
  }
}
```

## E. 자주 쓰는 명령

| 명령 | 동작 |
|---|---|
| `flutter pub get` | 의존성 설치 |
| `flutter pub upgrade` | 최신 버전 업데이트 |
| `flutter pub outdated` | 구버전 의존성 확인 |
| `flutter clean` | 빌드 산출물 삭제 |
| `flutter analyze` | 정적 분석 |
| `flutter test` | 단위 테스트 |
| `flutter build apk` | Android APK |
| `flutter build appbundle` | Android AAB (Play Store) |
| `flutter build ios` | iOS (macOS) |
| `flutter build ipa` | iOS IPA (App Store) |
| `flutter doctor` | 환경 진단 |
| `flutter devices` | 연결된 디바이스 |

## F. Flutter DevTools

### F-1. 활성화

```bash
flutter pub global activate devtools
```

### F-2. 사용

`flutter run` 실행 중 콘솔에 표시되는 DevTools URL 클릭.

탭:
- **Inspector** — Widget 트리 검사
- **Performance** — FPS / 프레임 분석
- **CPU Profiler**
- **Memory** — 메모리 사용량 + 누수
- **Network** — HTTP 요청
- **Logging** — print() 출력

## H. 트러블슈팅

| 증상 | 해결 |
|---|---|
| `Target of URI doesn't exist` | `flutter pub get` |
| `Multidex error` (Android) | `android/app/build.gradle` 의 `multiDexEnabled true` |
| iOS Pod 충돌 | `cd ios && pod deintegrate && pod install` |
| Hot Reload 가 일부 변경 미반영 | Hot Restart (`R`) 또는 Stop + Run |
| `flutter doctor` Android licenses | `flutter doctor --android-licenses` |
| Gradle 빌드 OOM | `android/gradle.properties` `org.gradle.jvmargs=-Xmx4096m` |
