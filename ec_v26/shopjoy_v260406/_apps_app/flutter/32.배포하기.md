## A. Android Release AAB

### A-1. AAB 빌드

```bash
cd _apps/flutter/ShopjoyFoApp
flutter clean
flutter pub get
flutter build appbundle --release --flavor prod --dart-define=APP_ENV=prod
```

결과: `build/app/outputs/bundle/prodRelease/app-prod-release.aab`

### A-2. APK 빌드 (사내용)

```bash
flutter build apk --release --flavor prod --dart-define=APP_ENV=prod
```

결과: `build/app/outputs/flutter-apk/app-prod-release.apk`

## B. Play Console 업로드

1. https://play.google.com/console → ShopJoy FO
2. **출시 → 테스트 → Internal testing → 새 버전 만들기**
3. AAB 업로드
4. 출시 노트
5. **저장 → 검토 → 내부 테스트 출시**

검토 즉시 통과.

## C. iOS Archive (macOS)

### C-1. IPA 빌드

```bash
cd _apps/flutter/ShopjoyFoApp
flutter clean
flutter pub get
flutter build ipa --release --flavor prod --dart-define=APP_ENV=prod
```

결과: `build/ios/ipa/Runner.ipa`

### C-2. Xcode Archive (수동 검증 시)

```bash
open ios/Runner.xcworkspace
```

Xcode:
1. 좌상단 디바이스 → **Any iOS Device (arm64)**
2. Scheme → `Runner-Prod` (또는 환경에 맞게)
3. **Product → Archive**

### C-3. App Store Connect 업로드

방법 A. **Transporter 앱**:
- App Store Connect → Apps → 앱 → TestFlight 빌드 추가
- macOS Transporter 앱 → IPA 드래그 → Deliver

방법 B. **xcrun altool**:
```bash
xcrun altool --upload-app -f build/ios/ipa/Runner.ipa \
  -t ios -u "$APPLE_ID" -p "$APP_SPECIFIC_PASSWORD"
```

방법 C. Xcode Organizer → Distribute App → App Store Connect.

## D. TestFlight + App Store

### D-1. TestFlight

App Store Connect → 앱 → TestFlight 탭 → 빌드 ❗ → 암호화 사용 입력 → 내부 테스터 추가.

iPhone TestFlight 앱 → 초대 메일 → 재구매.

### D-2. App Store 제출

App Store 탭 → **+ 새 버전 만들기** → 빌드 선택 → 메타데이터 → **검토 위해 제출** (1~3일).

## E. Codemagic 자동 빌드

`codemagic.yaml` 에 Flutter 워크플로우 추가:

```yaml
flutter-fo-android-release:
  name: Flutter FO — Android Release
  instance_type: mac_mini_m2
  environment:
    vars:
      APP_DIR: "_apps/flutter/ShopjoyFoApp"
    groups:
      - shopjoy_keystore
      - google_play
    flutter: 3.22.3
    java: 17
  triggering:
    events: [tag]
    tag_patterns:
      - pattern: 'flutter-fo-v*'
  scripts:
    - cd $APP_DIR && flutter pub get
    - cd $APP_DIR && flutter build appbundle --release --flavor prod --dart-define=APP_ENV=prod
  artifacts:
    - $APP_DIR/build/app/outputs/bundle/prodRelease/*.aab
  publishing:
    google_play:
      credentials: $GCLOUD_SERVICE_ACCOUNT_CREDENTIALS
      track: internal

flutter-fo-ios-release:
  name: Flutter FO — iOS Release
  instance_type: mac_mini_m2
  integrations:
    app_store_connect: shopjoy_app_store
  environment:
    vars:
      APP_DIR: "_apps/flutter/ShopjoyFoApp"
      BUNDLE_ID: "com.shopjoy.fo"
    flutter: 3.22.3
    xcode: 15.4
    cocoapods: default
  triggering:
    events: [tag]
    tag_patterns:
      - pattern: 'flutter-fo-v*'
  scripts:
    - cd $APP_DIR && flutter pub get
    - cd $APP_DIR/ios && pod install
    - keychain initialize
    - app-store-connect fetch-signing-files $BUNDLE_ID --type IOS_APP_STORE --create
    - keychain add-certificates
    - xcode-project use-profiles
    - cd $APP_DIR && flutter build ipa --release --flavor prod --dart-define=APP_ENV=prod --export-options-plist=$HOME/export_options.plist
  artifacts:
    - $APP_DIR/build/ios/ipa/*.ipa
  publishing:
    app_store_connect:
      auth: integration
      submit_to_testflight: true
```

자동 트리거:
```bash
git tag flutter-fo-v1.0.0
git push origin flutter-fo-v1.0.0
```
