# 33. 배포 점검하기 (Capacitor) — 따라하기

## A. 빌드 전 (Pre-build)

체크리스트를 모두 통과해야 빌드 시작:

### A-0. 환경 파일 점검 (.env)

- [ ] `.env.local` / `.env.dev` / `.env.prod` 6개 (FO+BO) 존재
- [ ] `.env.prod` 의 `BASE_URL` 이 운영 URL (`https://illeesam.netlify.app/...`)
- [ ] `.env.prod` 의 `CLEARTEXT=false`
- [ ] `.env.prod` 의 `WEB_CONTENTS_DEBUGGING=false`
- [ ] `.env.prod` 의 `ANDROID_SCHEME=https`
- [ ] `.env.prod` 의 `KAKAO_PAY_CID` 비어있고 → `.env.prod.local` 에 운영 키 입력
- [ ] `.env.prod` 의 `TOSS_CLIENT_KEY` 비어있고 → `.env.prod.local` 에 운영 키 입력
- [ ] `.gitignore` 에 `.env.*.local` 등록 확인 (`git status` 에 표시 안 됨)
- [ ] `npm run config:prod` 실행 → 출력에 `(loaded from .env.prod + .env.prod.local)` 확인

검증 명령:

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run config:prod
grep -A3 "server:" capacitor.config.ts
# 기대: url=운영 URL, cleartext=false, androidScheme=https
cat www/env.js | grep BASE_URL
# 기대: BASE_URL=운영 URL
```

### A-1. 버전 / 빌드 자원

- [ ] `package.json` 의 `version` 증가
- [ ] `android/app/build.gradle` 의 `versionCode` 증가
- [ ] `android/app/build.gradle` 의 `versionName` 증가
- [ ] iOS `Version` (`MARKETING_VERSION`) 증가
- [ ] iOS `Build` (`CURRENT_PROJECT_VERSION`) 증가
- [ ] FCM `google-services.json` / `GoogleService-Info.plist` 운영 키
- [ ] 카카오 / 토스 결제 키 운영 가맹점
- [ ] Android keystore release 설정
- [ ] iOS Distribution Certificate / Provisioning Profile 활성

## B. 빌드 후 (Post-build)

### B-1. 실기기 수동 테스트 — 핵심 플로우

```bash

adb install -r android/app/build/outputs/apk/debug/app-debug.apk

# iOS — Xcode 에서 ▶ Cmd+R
```

체크:

- [ ] 앱 첫 실행 → 흰 화면 X
- [ ] 회원가입 / 로그인
- [ ] 상품 검색 / 조회 / 장바구니
- [ ] 주문 → 결제 (카카오 / 토스)
- [ ] 결제 성공 시 딥링크 복귀 (`shopjoy-fo://pay/success`)
- [ ] 푸시 권한 요청 → 허용 → 토큰 발급
- [ ] FCM 발송 → 포그라운드 수신
- [ ] FCM 발송 → 백그라운드 수신 + 탭 시 화면 이동

#### iOS
- [ ] iPhone SE (4.7")
- [ ] iPhone 15 (6.1")
- [ ] iPhone 15 Pro Max (6.7")
- [ ] iOS 14 (minimum) / iOS 17 (latest)

### B-3. 네트워크 / 환경

- [ ] 다크 모드

- [ ] 백그라운드 → 포그라운드 복귀
