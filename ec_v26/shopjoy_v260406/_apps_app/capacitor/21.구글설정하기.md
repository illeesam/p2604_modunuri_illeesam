### 1-2. Analytics OFF → 만들기

## 2단계: Android 앱 등록 (FCM)

### 2-1. Android 추가

대시보드 → 우측 ⚙ → 프로젝트 설정 → **앱 추가** → **Android** 아이콘.

### 2-2. 패키지명 입력

| 환경 | 패키지명 |
|---|---|
| FO | `com.shopjoy.fo` |
| BO | `com.shopjoy.bo` |

→ **앱 등록** → **google-services.json 다운로드**.

### 2-3. 파일 배치 (환경별)

Capacitor 의 `appId` 는 환경에 관계없이 단일 (`com.shopjoy.fo`)로 사용하므로, 일반적으로 1개 `google-services.json` 만 사용해도 됩니다.

#### 옵션 A — 단일 파일 (간단, 권장)

```bash
mv ~/Downloads/google-services.json \
   _apps/capacitor/ShopjoyFoApp/android/app/google-services.json
```

#### 옵션 B — 환경별 분리 (Firebase 프로젝트가 dev/prod 별도일 때)

`build.gradle` 에 productFlavors 분기를 추가하고 각 환경별 디렉토리에 배치:

```
android/app/src/local/google-services.json     # Firebase shopjoy-dev 키
android/app/src/dev/google-services.json       # Firebase shopjoy-dev 키
android/app/src/prod/google-services.json      # Firebase shopjoy-prod 키
```

> ShopJoy Capacitor 앱은 기본적으로 **flavor 분기 없이 server.url 만 변경**하므로 옵션 A 권장. 옵션 B 는 운영 / 개발 Firebase 프로젝트가 분리된 경우만.

### 2-4. .gitignore 등록 (이미 적용됨)

`_apps/capacitor/ShopjoyFoApp/.gitignore` 에 이미 다음이 포함:

```
android/app/google-services.json
android/app/src/*/google-services.json
ios/App/App/GoogleService-Info.plist
```

→ 운영 키가 git 에 커밋되지 않습니다.

## 3단계: Android Gradle 설정

### 3-1. android/build.gradle 수정

`_apps/capacitor/ShopjoyFoApp/android/build.gradle` 의 `dependencies` 블록에 추가:

```gradle
buildscript {
    dependencies {
        classpath 'com.google.gms:google-services:4.4.2'
    }
}
```

### 3-2. android/app/build.gradle 수정

마지막 줄에 추가:
```gradle
apply plugin: 'com.google.gms.google-services'
```

`dependencies` 블록에 추가:
```gradle
dependencies {
    implementation platform('com.google.firebase:firebase-bom:33.4.0')
    implementation 'com.google.firebase:firebase-messaging'
}
```

### 3-3. 동기화

```bash
cd _apps/capacitor/ShopjoyFoApp
npx cap sync android
```

## 4단계: iOS 앱 등록 (macOS 만)

### 4-1. Firebase Console → iOS 추가

Bundle ID 입력 → `GoogleService-Info.plist` 다운로드.

### 4-2. 파일 배치

```bash
mv ~/Downloads/GoogleService-Info.plist \
   _apps/capacitor/ShopjoyFoApp/ios/App/App/GoogleService-Info.plist
```

### 4-3. Xcode 에 등록

```bash
npm run open:ios
```

Xcode → 좌측 우클릭 → **Add Files to "App"...** → `GoogleService-Info.plist` 선택 → **Copy items if needed** ✅ → **Add to targets: App** ✅ → **Add**.

### 4-4. APNs 인증 키

[`_apps/ios/22.IOS설정하기.md`](../ios/22.IOS설정하기.md) 의 **2단계** 따라서 `.p8` 발급 → Firebase Console → 클라우드 메시징 → APNs 인증 키 업로드.

### 4-5. AppDelegate.swift 수정

`ios/App/App/AppDelegate.swift` 에 Firebase 초기화 코드 추가:

```swift
import UIKit
import FirebaseCore
import FirebaseMessaging
import Capacitor

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {
    func application(_ application: UIApplication,
                     didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        FirebaseApp.configure()
        return true
    }

    func application(_ application: UIApplication,
                     didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
        Messaging.messaging().apnsToken = deviceToken
        NotificationCenter.default.post(name: .capacitorDidRegisterForRemoteNotifications,
                                        object: deviceToken)
    }
}
```

### 4-6. Pod 추가

`ios/App/Podfile`:
```ruby
target 'App' do
  pod 'FirebaseCore'
  pod 'FirebaseMessaging'
end
```

```bash
cd ios/App
pod install
cd ../..
```

## 5단계: 웹 코드에서 푸시 초기화

`utils/coNative.js` 가 이미 작성되어 있으므로 (이전 단계에서), 로그인 직후 호출:

`pages/Login.js` 또는 동등 위치:

```javascript
async function onLoginSuccess() {
  // ... 기존 로직 ...

  if (window.coNative) {
    await window.coNative.initPush({
      apiClient: window.foApi,
      apiPath: '/fo/my/device-token',
      uiNm: '로그인',
      cmdNm: '푸시토큰등록',
    });
  }
}
```

`index.html` 에 스크립트 추가:
```html
<script src="utils/coNative.js"></script>
```

## 6단계: 빌드 + 토큰 확인

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run android:dev
```

Chrome DevTools → Console 에서:

```javascript
await Capacitor.Plugins.PushNotifications.requestPermissions();
await Capacitor.Plugins.PushNotifications.register();
```

`registration` 리스너 콜백에서 토큰 출력 확인.

## 7단계: 테스트 메시지 발송

Firebase Console → **Cloud Messaging** → **새 캠페인** → **알림** → 테스트 메시지 보내기 → FCM 토큰 입력.

핸드폰에 알림 표시되면 성공!

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| Android: `Default FirebaseApp is not initialized` | google-services 플러그인 미적용 / google-services.json 경로 |
| iOS: 토큰 미수신 | 시뮬레이터 X (실기기 필수), APNs 키 등록 확인 |
| 권한 다이얼로그 안 뜸 | Info.plist 의 NSUserNotificationsUsageDescription 추가 |
| 푸시 도착 X | 백엔드 발송 시 FCM 토큰 정확? Firebase 프로젝트 일치? |
