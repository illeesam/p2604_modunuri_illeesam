## A. Android 핸드폰

### A-1. USB 디버깅 활성화

핸드폰에서:
1. 설정 → **휴대전화 정보** → **빌드번호** 7번 연속 탭
2. 설정 → 시스템 → 개발자 옵션 → **USB 디버깅** ON

### A-2. PC 연결 + 신뢰

USB 케이블로 PC 연결 → 핸드폰 화면 다이얼로그에서 **이 컴퓨터에서 항상 허용** 체크 + 확인.

### A-3. 인식 확인

```bash
adb devices -l
```

`device` 상태로 표시되면 OK.

### A-4. local 환경 빌드 + 설치

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run android:local
```

자동 진행:
1. `set-env.js local` 실행 → capacitor.config.ts 업데이트
2. `cap sync` → www 자산 + 플러그인 동기화
3. Gradle 빌드 → APK 생성
4. 핸드폰에 자동 설치 + 실행

### A-5. PC LAN IP 로 변경 (실기기 전용)

`local` 의 기본 URL은 `http://10.0.2.2:5501` 이지만, **이는 에뮬레이터 전용**.
실기기는 PC LAN IP 사용해야 합니다.

#### 1) PC IP 확인

```powershell
ipconfig | Select-String "IPv4"
# 예: 192.168.0.100
```

#### 2) 옵션 A — `.env.local` 직접 수정 (간단)

`_apps/capacitor/ShopjoyFoApp/.env.local` 의 `BASE_URL` 한 줄만 변경:

```env
APP_ENV=local
APP_NAME=ShopJoy FO Local
BASE_URL=http://192.168.0.100:5501       # ← 본인 PC LAN IP
CLEARTEXT=true
ANDROID_SCHEME=http
DEEP_LINK_SCHEME=shopjoy-fo
WEB_CONTENTS_DEBUGGING=true
ALLOW_MIXED_CONTENT=true
```

> ⚠️ 이 방식은 git 에 커밋되므로 동료의 IP 와 충돌 가능. 본인 환경만 임시로 쓰는 경우.

#### 3) 옵션 B — `.env.local.local` 오버라이드 (권장)

`.gitignore` 에 등록된 오버라이드 파일을 사용해 본인 PC IP 만 적용:

`_apps/capacitor/ShopjoyFoApp/.env.local.local` (신규 생성):

```env
BASE_URL=http://192.168.0.100:5501
```

set-env.js 가 자동으로 `.env.local` + `.env.local.local` 을 머지합니다.

#### 4) 다시 빌드

```bash
npm run android:local
```

#### 5) PC 방화벽

Windows 방화벽 → 인바운드 규칙 → 포트 **5501** 허용.

#### 6) BO 앱도 동일

BO 는 `.env.local` 의 BASE_URL 이 `http://192.168.0.100:5501/bo.html` (`/bo.html` 포함).

## B. iOS iPhone (macOS 만)

### B-1. iPhone 을 Mac 에 USB 연결

- 케이블 연결
- iPhone 화면 잠금 해제
- "이 컴퓨터를 신뢰" 다이얼로그 → 신뢰

### B-2. local 환경 빌드 + 설치

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run ios:local
```

자동 진행:
1. set-env.js local
2. cap sync ios
3. Xcode 자동 빌드 + 설치 + 실행

### B-3. 첫 실행 시 신뢰

iPhone 첫 실행 시 **신뢰할 수 없는 개발자** 메시지 → 설정 → 일반 → **VPN 및 기기 관리** → 본인 Apple ID → **신뢰**.

## C. WebView 디버깅

### C-1. Android (Chrome DevTools)

1. 핸드폰에서 ShopJoy 앱 실행 중
2. 데스크톱 Chrome → `chrome://inspect`
3. **ShopJoy** 항목 → **inspect** 클릭

### C-2. iOS (Safari Web Inspector)

1. iPhone 설정 → Safari → 고급 → **웹 속성** ON
2. Mac Safari → 환경설정 → 고급 → **메뉴 막대에 개발자용 메뉴 표시**
3. Mac Safari → 개발자용 → [iPhone 이름] → ShopJoy

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `adb devices` 빈 목록 | USB 디버깅 / 케이블 / 다이얼로그 허용 재확인 |
| 앱이 흰 화면 | local 환경에서 PC LAN IP 미수정 + 방화벽 |
| `unauthorized` | 핸드폰 다이얼로그 다시 허용 |
| iOS `Could not locate device` | 케이블 / Xcode → Devices and Simulators 재확인 |
