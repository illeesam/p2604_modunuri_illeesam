## 1단계: 앱 버전 정책

### 1-1. android/app/build.gradle 수정

```gradle
android {
    defaultConfig {
        versionCode 10000          // 빌드마다 +1
        versionName "1.0.0"        // 의미적 버전
    }
}
```

권장 versionCode 산정: `MAJOR * 10000000 + MINOR * 100000 + PATCH * 1000 + BUILD`

### 1-2. iOS 버전 (Xcode)

Xcode → 타겟 → General 탭:
- **Version** (`MARKETING_VERSION`): `1.0.0`
- **Build** (`CURRENT_PROJECT_VERSION`): `1`

### 1-3. package.json 동기화

```json
{
  "version": "1.0.0"
}
```

## 2단계: Android 키스토어 생성

```powershell
keytool -genkey -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo `
  -keyalg RSA -keysize 2048 -validity 10000
```

비밀번호 / 정보 입력. **분실 주의!** (분실 시 앱 업데이트 영구 불가)

### 2-1. 키스토어 등록

`_apps/capacitor/ShopjoyFoApp/android/key.properties` (gitignore):

```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=shopjoy-fo
storeFile=../shopjoy-fo-release.keystore
```

### 2-2. android/app/build.gradle 수정

```gradle
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

### 2-3. .gitignore

```

*.keystore
*.jks

ios/App/App/GoogleService-Info.plist
```

## 4단계: 운영 환경 (.env.prod) 설정

### 4-1. .env.prod 검토

`_apps/capacitor/ShopjoyFoApp/.env.prod` 의 모든 키가 운영 기준인지 확인:

```env
APP_ENV=prod
APP_NAME=ShopJoy
BASE_URL=https://illeesam.netlify.app/mainframe#sj_index
CLEARTEXT=false                  # ← 운영은 반드시 false (HTTPS 강제)
ANDROID_SCHEME=https
DEEP_LINK_SCHEME=shopjoy-fo
WEB_CONTENTS_DEBUGGING=false     # ← 운영은 false (앱스토어 정책)
ALLOW_MIXED_CONTENT=false
KAKAO_PAY_CID=                   # ← 비밀 — .env.prod.local 에서 오버라이드
TOSS_CLIENT_KEY=                 # ← 비밀 — .env.prod.local 에서 오버라이드
```

### 4-2. 운영 비밀 — `.env.prod.local` (git 무시)

운영 가맹점 ID / 토큰 등 **민감 정보는 절대 커밋 금지**. 별도 파일에 저장:

`_apps/capacitor/ShopjoyFoApp/.env.prod.local` (신규 생성):

```env
KAKAO_PAY_CID=CID_OPERATIONAL_REAL
TOSS_CLIENT_KEY=live_ck_REAL_OPERATIONAL_KEY
```

set-env.js 가 자동으로 `.env.prod` + `.env.prod.local` 을 머지하여 운영 키만 주입합니다.

`.gitignore` 에 이미 등록되어 있어 커밋되지 않습니다:

```gitignore
.env.*.local
```

### 4-3. CI 환경에서 비밀 주입 (Codemagic / GitHub Actions)

CI 빌드 시 `.env.prod.local` 을 환경변수로 동적 생성:

```yaml
# codemagic.yaml 예시 — fo-android-release 워크플로우
- name: Inject prod secrets into .env.prod.local
  script: |
    cd $APP_DIR
    cat > .env.prod.local <<EOF
    KAKAO_PAY_CID=$KAKAO_PAY_CID_PROD
    TOSS_CLIENT_KEY=$TOSS_CLIENT_KEY_PROD
    EOF
- name: Apply prod environment + build
  script: |
    cd $APP_DIR
    npm run sync:prod
    cd android && ./gradlew bundleRelease
```

Codemagic 콘솔의 환경변수 그룹 `shopjoy_secrets` 에 등록:
- `KAKAO_PAY_CID_PROD` (secure)
- `TOSS_CLIENT_KEY_PROD` (secure)
- `FIREBASE_KEY_PATH` (secure)

### 4-4. 운영 환경 검증

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run config:prod
```

기대 출력 (CLEARTEXT=false 확인):

```
[set-env] PROD 적용 완료
         APP_NAME       = ShopJoy
         BASE_URL       = https://illeesam.netlify.app/mainframe#sj_index
         CLEARTEXT      = false
         ANDROID_SCHEME = https
         (loaded from   .env.prod + .env.prod.local)   # ← .local 머지됨
         (www/env.js    10 keys)
```

생성된 `capacitor.config.ts` 확인:

```bash
grep -A3 "server:" capacitor.config.ts
# server: {
#     url: 'https://illeesam.netlify.app/mainframe#sj_index',
#     cleartext: false,
#     androidScheme: 'https',
```
