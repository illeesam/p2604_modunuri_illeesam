  SCH 패키지 — 스케줄러 구동 원리
═══════════════════════════════════════════════════════

▶ 핵심 흐름 (앱 시작 → 자동 실행)

  1. [앱 시작] Spring Boot Application 구동
         │
  2. [SchBatchConfig] @Configuration 로드
         │  ├─ @EnableScheduling           → Spring 스케줄링 활성화
         │  └─ ThreadPoolTaskScheduler 빈  → 실제 cron 실행 스레드풀 (5개)
         │
  3. [SchBatchRunner] ApplicationRunner.run() 호출
         │  ├─ app.scheduler.enabled=false 이면 전체 스킵
         │  ├─ sy_batch 테이블에서 batch_status_cd = 'ACTIVE' 행 조회
         │  └─ 각 배치 → SchBatchJobRegistry.register() 호출
         │
  4. [SchBatchJobRegistry.register()]
         │  ├─ 배치의 cron_expr 읽기  (예: "0 0 * * * *" = 매시 정각)
         │  ├─ ThreadPoolTaskScheduler.schedule(람다, CronTrigger) 호출
         │  │    └─ 람다 = SchBatchExecutor.execute(batch)
         │  └─ ScheduledFuture → ConcurrentHashMap에 저장
         │        (key: batchCode, value: ScheduledFuture)
         │
  5. [cron 시각 도달 시] ThreadPoolTaskScheduler가 자동 트리거
         │
  6. [SchBatchExecutor.execute()]
         │  ├─ batchCode로 SchBatchJobHandler 구현체 탐색
         │  │    (없으면 run_status = 'NO_HANDLER' 저장 후 종료)
         │  ├─ handler.execute(batch) 호출  ← 실제 비즈니스 로직
         │  └─ sy_batch 업데이트
         │        batch_run_status  = 'SUCCESS' or 'FAIL'
         │        batch_last_run    = 실행 시각
         │        batch_next_run    = 다음 예정 시각 (cron 계산)
         │        batch_run_count   += 1


▶ 파일별 역할

  config/SchBatchConfig.java
    - ThreadPoolTaskScheduler 빈 등록 (스레드풀 = 5)
    - @EnableScheduling 으로 스케줄링 활성화

  config/SchBatchProperties.java
    - application.yml 의 app.scheduler.* 값 바인딩
    - enabled / jenkins.enabled / allowedIps

  core/SchBatchRunner.java
    - 앱 시작 시 1회 실행 (ApplicationRunner)
    - ACTIVE 배치 목록 조회 → 전부 레지스트리에 등록

  core/SchBatchJobRegistry.java
    - batchCode ↔ ScheduledFuture 매핑 관리
    - register / unregister / unregisterAll (런타임 동적 제어)

  core/SchBatchExecutor.java
    - 핸들러 탐색 → 실행 → sy_batch 이력 업데이트

  handler/SchBatchJobHandler.java
    - 실제 배치 로직 구현 인터페이스
    - 새 배치 추가 시 이 인터페이스를 implements 하고 @Component 붙이면 자동 등록

  controller/SchBatchController.java
    - 관리자 REST API (BoOnly 권한)
    - GET  /sch/batch          : 전체 배치 목록 + 등록 여부
    - POST /sch/batch/{code}/run : 즉시 실행
    - POST /sch/batch/{code}/on  : 스케줄 등록
    - POST /sch/batch/{code}/off : 스케줄 해제
    - POST /sch/reload           : DB 재조회 후 전체 재등록


▶ 새 배치 작업 추가 방법

  1. sy_batch 테이블에 행 INSERT
       batch_code      = 'MY_JOB'
       cron_expr       = '0 30 2 * * *'   (매일 새벽 2:30)
       batch_status_cd = 'ACTIVE'

  2. SchBatchJobHandler 구현체 작성

       @Component
       public class MyJob implements SchBatchJobHandler {
           @Override
           public String batchCode() { return "MY_JOB"; }

           @Override
           public void execute(SyBatch batch) {
               // 실제 비즈니스 로직
           }
       }

  3. 앱 재시작 or POST /sch/reload 호출 → 자동 등록됨


▶ yml 설정 (application-{profile}.yml)

  app:
    scheduler:
      enabled: true          # false 이면 앱 시작 시 전체 스킵
      jenkins:
        enabled: false       # true 이면 Jenkins 연동 모드
      allowed-ips: ""        # 허용 IP (^구분, 빈값=전체허용)
