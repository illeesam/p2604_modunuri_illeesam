# 22. iOS (Apple Developer) 설정하기 — 따라하기

ShopJoy iOS 앱 배포에 필요한 Apple Developer 계정 + 인증서 + 프로파일 설정.

## 사전 준비

- [ ] [`11.개발환경.md`](11.개발환경.md) 완료
- [ ] Apple Developer Program 유료 가입 ($99/년)

---

## 1단계: App Identifier 등록

App Identifier 는 각 앱의 고유 ID 입니다.

### 1-1. Apple Developer Portal 접속

https://developer.apple.com/account → **Certificates, Identifiers & Profiles** 클릭.

### 1-2. Identifiers 메뉴

좌측 **Identifiers** 클릭 → 우측 **+** 버튼.

### 1-3. App IDs 선택

**App IDs** → **Continue** → **App** → **Continue**.

### 1-4. 정보 입력

| 항목 | 값 |
|---|---|
| Description | `ShopJoy FO Production` |
| Bundle ID | **Explicit** 선택 → `com.shopjoy.fo` |

**Capabilities** 섹션에서 사용 기능 체크:
- ✅ **Push Notifications**

→ **Continue** → **Register**.

### 1-5. 6개 모두 등록

| 환경 | Bundle ID |
|---|---|
| FO Local | `com.shopjoy.fo.local` |
| FO Dev | `com.shopjoy.fo.dev` |
| FO Prod | `com.shopjoy.fo` |
| BO Local | `com.shopjoy.bo.local` |
| BO Dev | `com.shopjoy.bo.dev` |
| BO Prod | `com.shopjoy.bo` |

각각 1-3 ~ 1-4 반복.

## 2단계: APNs 인증 키 생성

푸시 알림 발송에 필요한 키입니다.

### 2-1. Keys 메뉴

좌측 **Keys** 클릭 → 우측 **+** 버튼.

### 2-2. 키 정보 입력

- Key Name: `ShopJoy APNs Key`
- ✅ **Apple Push Notifications service (APNs)** 체크 → **Configure** 버튼:
  - **Sandbox & Production** 선택
  - 환경: **All Apps** 또는 특정 Bundle ID 선택
  - **Save**

→ **Continue** → **Register**.

### 2-3. 키 다운로드

**Download** 버튼 클릭 → `AuthKey_ABCD123456.p8` 파일 다운로드.

> ⚠️ **이 .p8 파일은 한 번만 다운로드 가능**. 분실 시 키 재발급 필요. 안전하게 보관.

화면에 표시된 **Key ID** (예: `ABCD123456`) 메모.

### 2-4. Team ID 확인

화면 우상단 본인 이름 옆 → 또는 좌측 **Membership** 메뉴 → **Team ID** (예: `1A2B3C4D5E`) 메모.

## 3단계: Distribution Certificate 생성

App Store 배포 시 필요한 인증서.

### 3-1. Certificate Signing Request (CSR) 생성

Mac 키체인에서:

1. **Launchpad → 기타 → 키체인 접근** 실행
2. 메뉴 → **키체인 접근 → 인증서 지원 → 인증 기관에서 인증서 요청**
3. 정보 입력:
   - 사용자 이메일: 본인 Apple ID 이메일
   - 일반 이름: `ShopJoy Distribution`
   - **요청 항목 → 디스크에 저장됨** 선택
4. **계속** → 데스크톱에 `CertificateSigningRequest.certSigningRequest` 저장.

### 3-2. Apple Developer Portal 에서 Certificate 생성

좌측 **Certificates** → **+** → **Apple Distribution** 선택 → **Continue**.

### 3-3. CSR 업로드

방금 만든 `.certSigningRequest` 파일 선택 → **Continue** → **Download** 버튼.

`distribution.cer` 파일이 다운로드됨.

### 3-4. 키체인에 추가

다운로드한 `.cer` 파일 더블클릭 → 키체인 접근 자동 실행 → **로그인** 키체인에 추가.

---

## 4단계: Provisioning Profile 생성

각 환경 (Local/Dev/Prod) 별로 1개씩, 또는 Prod 만.

### 4-1. Profiles 메뉴

좌측 **Profiles** → **+** → **App Store** (배포용) 또는 **Ad Hoc** (베타 배포) 선택 → **Continue**.

### 4-2. App ID 선택

`com.shopjoy.fo` (또는 다른 Bundle ID) → **Continue**.

### 4-3. Certificate 선택

방금 만든 Distribution Certificate 선택 → **Continue**.

### 4-4. 이름 입력

Provisioning Profile Name: `ShopJoy FO Prod App Store` → **Generate**.

### 4-5. 다운로드

`ShopJoy_FO_Prod_App_Store.mobileprovision` 다운로드.

### 4-6. Xcode 에 추가

다운로드한 파일을 더블클릭 → Xcode 자동 등록.

## 5단계: Xcode 자동 서명 설정

### 5-1. 프로젝트 열기

```bash
cd _apps/ios/ShopjoyFoApp
xcodegen generate
open ShopjoyFoApp.xcodeproj
```

### 5-2. 타겟 → Signing & Capabilities

좌측 Project Navigator → 타겟 ShopjoyFoApp → **Signing & Capabilities** 탭.

### 5-3. Team 선택

- ✅ **Automatically manage signing**
- **Team** 드롭다운 → 본인 회사 Team 선택

자동 서명 시 Xcode 가 적절한 Provisioning Profile 자동 다운로드.

### 5-4. Bundle Identifier 확인

각 Configuration 별로:
- Debug-Local: `com.shopjoy.fo.local`
- Debug-Dev: `com.shopjoy.fo.dev`
- Debug-Prod: `com.shopjoy.fo`
- Release-* 도 동일

`project.yml` 에 이미 설정되어 있으므로 `xcodegen generate` 후 자동 반영.

## 6단계: App Store Connect 앱 등록

### 6-1. App Store Connect 접속

https://appstoreconnect.apple.com → **나의 앱** → **+** → **새로운 앱**.

### 6-2. 정보 입력

| 항목 | 값 |
|---|---|
| 플랫폼 | iOS |
| 이름 | ShopJoy |
| 기본 언어 | 한국어 |
| Bundle ID | `com.shopjoy.fo` (드롭다운에서 선택) |
| SKU | `SHOPJOY-FO-001` (임의 고유 식별자) |
| 사용자 액세스 | 전체 액세스 |

→ **만들기**.

### 6-3. 앱 정보 작성

좌측 메뉴:

- **앱 정보**: 카테고리 (쇼핑) / 하위 카테고리 / 콘텐츠 권한 (4+) / 저작권
- **가격 및 사용 가능 여부**: 무료
- **앱 개인정보 보호**: 데이터 수집 양식 작성

## 7단계: TestFlight 베타 테스터 등록

### 7-1. App Store Connect → TestFlight 탭

빌드 업로드 완료 후 (Xcode → Archive → Distribute App → App Store Connect):

### 7-2. 내부 테스터 (즉시 배포)

**Apple ID 추가** → 본인 또는 사내 Apple ID 입력 → **추가**.

iPhone 에 **TestFlight** 앱 설치 → 초대 메일에서 **재구매** 또는 **테스트 시작**.

### 7-3. 외부 테스터 (Apple 검토 24시간)

10000명까지 가능. **외부 테스터 추가** → 초대 링크 공유.

## 8단계: 인증서 / 프로파일 자동 관리 (Fastlane Match — 선택)

여러 개발자가 같은 인증서를 공유하려면:

```bash
sudo gem install fastlane
cd _apps/ios/ShopjoyFoApp/ShopjoyFoApp
fastlane init
fastlane match init
```

GitHub private repo 또는 Google Cloud Storage 에 인증서 암호화 저장.

```bash
fastlane match appstore         # App Store 배포용
fastlane match development      # 개발용
```

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `No matching profiles found` | Step 4 의 Provisioning Profile 미발급 |
| `The provisioning profile doesn't include the device` | Personal Team 만 가능, 7일 만료 → 다시 ▶ |
| `Distribution certificate already exists` | 키체인 → 기존 인증서 삭제 후 재발급 |
| Xcode 가 Profile 인식 못함 | Xcode 재시작 또는 `~/Library/MobileDevice/Provisioning Profiles/` 정리 |
| `.p8` 파일 분실 | Apple Developer Portal 에서 키 revoke + 재발급 (Firebase 도 재업로드) |
