# 32. 배포하기 (iOS) — 따라하기

ShopJoy iOS 앱을 TestFlight + App Store 에 업로드하는 절차.

- [ ] App Store Connect 앱 등록 완료
- [ ] 인증서 / Provisioning Profile 등록 완료

## 1단계: 운영 환경 빌드 준비

### 1-1. 프로젝트 디렉토리로 이동

```bash
cd _apps/ios/ShopjoyFoApp
```

### 1-2. 버전 증가

`project.yml` 편집:

```yaml
settings:
  base:
    MARKETING_VERSION: "1.0.1"        # ← 증가
    CURRENT_PROJECT_VERSION: "2"      # ← 증가
```

### 1-3. xcodegen 재생성

```bash
xcodegen generate
```

---

## 2단계: Archive 빌드

### 2-1. Xcode 열기

```bash
open ShopjoyFoApp.xcworkspace
# (Pod 사용 시) 또는 .xcodeproj
```

### 2-2. 디바이스 선택

좌상단 디바이스 드롭다운 → **Any iOS Device (arm64)** 선택.

### 2-3. Scheme 선택

Scheme 드롭다운 → **ShopjoyFoApp-Prod** 선택.

### 2-4. Archive 실행

메뉴 → **Product → Archive** (또는 단축키 없음, 메뉴에서만).

10~30분 빌드 진행. 완료 후 **Organizer** 창 자동 오픈.

---

## 3단계: TestFlight 업로드

### 3-1. Organizer 에서 Distribute App

Organizer → 방금 만든 Archive 선택 → **Distribute App** 버튼.

### 3-2. App Store Connect 선택

**App Store Connect** → **Next**.

### 3-3. Upload 선택

**Upload** → **Next**.

### 3-4. 옵션 (기본값 그대로)

- ✅ Upload your app's symbols
- ✅ Manage Version and Build Number

→ **Next**.

### 3-5. 자동 서명

**Automatically manage signing** → **Next** → 자동으로 인증서 + 프로파일 사용.

### 3-6. Upload

**Upload** 버튼 → 5~15분 업로드.

성공 시 **App Store Connect uploads completed** 메시지.

---

## 4단계: TestFlight 검증

### 4-1. App Store Connect 접속

https://appstoreconnect.apple.com → 앱 → **TestFlight** 탭.

### 4-2. 빌드 처리 대기

업로드 후 5~30분 처리 시간. 상태가 **처리 중 → 테스트 준비 완료** 변경.

### 4-3. 컴플라이언스 입력

빌드 옆 ❗ → **암호화 사용 여부** 질문 → 일반적으로 **아니오** 선택 → 저장.

### 4-4. 내부 테스터 추가

좌측 **앱 내부 테스트 → 내부 테스터** → **+** → 본인 / 동료 Apple ID 추가.

### 4-5. iPhone TestFlight 앱에서 설치

iPhone 에서 **App Store** → `TestFlight` 검색 → 설치.

TestFlight 앱 → 초대 메일에서 **재구매** 또는 **테스트 시작**.

ShopJoy 베타 빌드가 설치됨.

---

## 5단계: 외부 테스터 (선택)

10000명까지 일반 사용자 베타 배포 가능. Apple 검토 24시간 소요.

### 5-1. 외부 테스터 그룹 생성

좌측 **앱 외부 테스트 → 그룹** → **+ 그룹 추가** → 그룹 이름 (예: `Beta Users`).

### 5-2. 빌드 추가

그룹 → **빌드** 탭 → **+** → 빌드 선택 → **추가**.

### 5-3. 베타 정보 입력

- 베타 앱 설명
- 피드백 이메일
- 베타 앱 검토 정보 (수출 규정 / IDFA)

→ **검토를 위해 제출**.

### 5-4. 검토 통과 후 초대 링크 발송

**공개 링크** 활성화 → URL 복사 → 사용자에게 공유.

---

## 6단계: App Store 제출

### 6-1. App Store 탭

App Store Connect → 앱 → **App Store** 탭.

### 6-2. 새 버전 만들기

좌측 버전 옆 **+** → 버전 번호 입력 (예: `1.0.1`).

### 6-3. 빌드 선택

**빌드** 섹션 → **빌드 추가** → TestFlight 에 업로드된 빌드 중 선택.

### 6-4. 메타데이터 작성

- 새로운 기능 (해당 버전의 변경사항 4000자)
- 마케팅 URL / 지원 URL
- 카피라이트
- 등급 (만 4세 이상 등)

### 6-5. 출시 옵션

- **수동 출시** (검토 통과 후 본인이 클릭) — 권장
- **승인 즉시 출시**
- **단계별 출시** (점진적 배포, 7일간)

### 6-6. 검토 위해 제출

**저장** → **검토 위해 제출**.

검토 시간:
- 보통 1~3일
- 거절 시 메시지 확인 → 수정 → 재제출

---

## 7단계: 검토 통과 후 출시

### 7-1. 승인 알림 받음

이메일 / App Store Connect 알림.

### 7-2. 수동 출시 (수동 모드 선택 시)

App Store 탭 → **출시** 버튼 클릭.

전 세계 App Store 에 즉시 반영 (1~2시간).

---

## 8단계: 자동 업로드 (Codemagic)

수동 Archive 대신 GitHub push → 자동 빌드 + TestFlight 업로드.

설정은 [`6.iOS빌드및Codemagic.md`](../capacitor/_doc/정책서-Capacitor/6.iOS빌드및Codemagic.md) 참조.

`codemagic.yaml` 의 `fo-ios-release` 워크플로우 가동:

```bash
git tag fo-v1.0.1
git push origin fo-v1.0.1
```

→ Codemagic 이 자동으로 macOS runner 에서 빌드 + TestFlight 업로드.

## 9단계: 자동 업로드 (Fastlane — 선택)

```bash
sudo gem install fastlane
cd _apps/ios/ShopjoyFoApp/ShopjoyFoApp
fastlane init
```

`fastlane/Fastfile`:

```ruby
default_platform(:ios)

platform :ios do
  desc "Beta build and upload to TestFlight"
  lane :beta do
    increment_build_number(xcodeproj: "ShopjoyFoApp.xcodeproj")
    build_app(
      workspace: "ShopjoyFoApp.xcworkspace",
      scheme: "ShopjoyFoApp-Prod"
    )
    upload_to_testflight(skip_waiting_for_build_processing: true)
  end
end
```

```bash
fastlane beta
```

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `Code signing required` | Step 5-2 에서 자동 서명 선택 / Team 재설정 |
| `Invalid Bundle. The provided icon ...` | 아이콘 1024×1024 PNG 알파 채널 / 투명 배경 제거 |
| `Missing Push Notification Entitlement` | Capability 에서 Push Notifications 추가 |
| Archive 후 `Generic Xcode Archive` 표시 | 타겟 설정 → **Skip Install** = NO 확인 |
| `ITMS-90809: Deprecated API Usage` | Xcode / SDK 업그레이드 |
| TestFlight `처리 중` 30분 이상 | 6-1 의 컴플라이언스 입력 누락 |
