## A. Android 핸드폰

### A-1. USB 디버깅 활성화

핸드폰: 설정 → 휴대전화 정보 → 빌드번호 7번 탭 → 개발자 옵션 → **USB 디버깅** ON.

PC 연결 + 신뢰 다이얼로그 허용.

### A-2. Flutter 인식 확인

```bash
flutter devices
```

다음과 같이 핸드폰이 표시되어야 함:

```
2 connected devices:
Pixel 7 (mobile) • ABCD123456 • android-arm64 • Android 14 (API 34)
Chrome (web)     • chrome     • web-javascript • Google Chrome 120
```

### A-3. local 환경 빌드 + 설치

```bash
cd _apps/flutter/ShopjoyFoApp
flutter run --dart-define=APP_ENV=local
```

자동 진행:
1. APP_ENV=local → `.env.local` 로드
2. Gradle 빌드
3. 핸드폰 자동 설치 + 실행

### A-4. PC LAN IP 변경 (실기기 전용)

`.env.local` 의 BASE_URL 은 `http://10.0.2.2:5501` (에뮬레이터용).

실기기는 PC LAN IP:

```bash
# PC IP 확인
ipconfig | Select-String "IPv4"
# 예: 192.168.0.100
```

`.env.local` 수정:
```
BASE_URL=http://192.168.0.100:5501
```

다시 `flutter run --dart-define=APP_ENV=local`.

PC 방화벽 → 5501 포트 허용.

## C. WebView 디버깅

### C-1. Android (Chrome DevTools)

`chrome://inspect` → ShopjoyFoApp → **inspect**.

### C-2. iOS (Safari Web Inspector)

iPhone 설정 → Safari → 고급 → 웹 속성 ON.
Mac Safari → 개발자용 메뉴 → [iPhone] → ShopjoyFoApp.

### C-3. Flutter DevTools

`flutter run` 출력에 표시되는 URL:

```
A Dart VM Service is available at: http://127.0.0.1:xxxxx/
The Flutter DevTools debugger is available at: http://127.0.0.1:9100?uri=...
```

브라우저로 접속 → Widget Inspector / Network / Performance / Memory 모두 확인.

또는:

```bash
flutter pub global activate devtools
flutter pub global run devtools
```

## D. Hot Reload / Hot Restart

`flutter run` 실행 중인 터미널에서:

| 키 | 동작 |
|---|---|
| `r` | Hot Reload (state 유지, UI 만 갱신) |
| `R` | Hot Restart (state 초기화) |
| `q` | 종료 |
| `o` | 플랫폼 전환 (Android ↔ iOS) |
| `h` | 도움말 |

웹 코드 (WebView 내부 콘텐츠) 변경은 Live Server 가 처리.

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `flutter devices` 빈 목록 | adb 인식 / USB 디버깅 / 케이블 |
| Android 빌드 OOM | `android/gradle.properties` 의 `org.gradle.jvmargs=-Xmx4096m` |
| `Pod install failed` | `cd ios && pod deintegrate && pod install` |
| Hot Reload 안 됨 | `R` (대문자) 로 Hot Restart |
| `flutter pub get` 실패 | 네트워크 / `flutter pub cache repair` |
