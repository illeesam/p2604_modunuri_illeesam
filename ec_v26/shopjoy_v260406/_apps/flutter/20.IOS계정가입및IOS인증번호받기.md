# 20. Apple ID 가입 + iOS 인증번호 받기 — 따라하기 (Flutter)

> Flutter iOS 빌드도 결국 Xcode 프로젝트 (`ios/Runner.xcworkspace`) 라서 절차가 순수 iOS 와 동일합니다.
> 자세한 절차는 → [`_apps/ios/20.IOS계정가입및IOS인증번호받기.md`](../ios/20.IOS계정가입및IOS인증번호받기.md)

## iOS 인증번호 — 한눈에

| 용도 | iOS 식별자 |
|---|---|
| 앱 식별 | **Bundle ID** (Flutter `Runner` 의 PRODUCT_BUNDLE_IDENTIFIER) |
| 팀 식별 | **Team ID** (10자) |
| 푸시 발송 | **APNs 인증 키 (.p8)** + Key ID |
| 코드 서명 | **Distribution Certificate** |
| 빌드 권한 | **Provisioning Profile** |

## 사전 준비

- [ ] **macOS 14+** + Xcode 15.4+
- [ ] CocoaPods 설치 (`sudo gem install cocoapods`)
- [ ] Flutter 3.22+ (`flutter doctor` 모든 항목 통과)
- [ ] 신용카드 (Apple Developer Program $99/년)

---

## 1단계: Apple ID 가입

https://appleid.apple.com → **Apple ID 생성** → 이메일 / 비밀번호 / 전화 인증 → 가입.

**2단계 인증 필수** — https://appleid.apple.com → **로그인 및 보안 → 2단계 인증 → 켜기**.

운영 / 개인 분리 권장.

---

## 2단계: Apple Developer Program 가입

https://developer.apple.com/programs/enroll → 시작하기 → Apple ID 로그인 → **Individual** / **Organization** → 결제 → 1~2일 승인.

---

## 3단계: Team ID 확인

https://developer.apple.com/account → 좌측 **Membership** → **Team ID** 메모 (10자, 예: `ABCDE12345`).

---

## 4단계: Bundle ID 등록

Flutter 의 기본 Bundle ID 는 `flutter create` 시 입력한 `--org` 값에서 결정됩니다. 변경하려면:

### 4-0. Flutter Bundle ID 변경

`ios/Runner.xcodeproj/project.pbxproj` 열거나 Xcode 에서:

```bash
cd _apps/flutter/ShopjoyFoApp
open ios/Runner.xcworkspace
```

Xcode → Runner 타겟 → **General → Identity → Bundle Identifier** → `com.shopjoy.fo` 로 변경.

또는 `change_app_package_name` 패키지로 한 번에:

```bash
flutter pub global activate change_app_package_name
flutter pub global run change_app_package_name:main com.shopjoy.fo
```

### 4-1. Apple Developer Portal 등록

1. https://developer.apple.com/account → **Identifiers** → **+**
2. **App IDs** → **App** → **Continue**
3. Bundle ID: `com.shopjoy.fo` (Explicit)
4. Capabilities ✅ **Push Notifications**
5. **Continue → Register**

### 4-2. Flutter Flavor 분리 시 6개 등록

Android 와 iOS 양쪽 flavor 분리 시:

| 환경 | Bundle ID |
|---|---|
| FO Local | `com.shopjoy.fo.local` |
| FO Dev | `com.shopjoy.fo.dev` |
| FO Prod | `com.shopjoy.fo` |
| BO Local/Dev/Prod | `com.shopjoy.bo.*` |

각 Bundle ID 별로 4-1 반복.

> Flutter flavor 분리 → [`31.배포설정하기.md`](31.배포설정하기.md) 의 **7단계** 참조.

## 5단계: APNs 인증 키 (.p8) 발급

`firebase_messaging` 패키지가 사용.

1. https://developer.apple.com/account → **Keys** → **+**
2. Key Name: `ShopJoy APNs Key`
3. ✅ **Apple Push Notifications service (APNs)** → **Configure** → **All Apps** → **Save**
4. **Continue → Register → Download** → `AuthKey_XXXX.p8` (한 번만)
5. **Key ID** + **Team ID** 메모

---

## 6단계: Distribution Certificate + Provisioning Profile

Mac 키체인 → CSR 생성 → Apple Developer Portal **Certificates → +** → **Apple Distribution** → CSR 업로드 → 다운로드 → 더블클릭 → 키체인 등록.

**Profiles → +** → **App Store** → Bundle ID + Distribution Certificate → 다운로드 → 더블클릭 → Xcode 자동 등록.

상세 → [`_apps/ios/20.IOS계정가입및IOS인증번호받기.md`](../ios/20.IOS계정가입및IOS인증번호받기.md) 의 **6/7단계**.

## 7단계: Xcode 자동 서명 (Flutter `Runner`)

```bash
cd _apps/flutter/ShopjoyFoApp
flutter pub get
cd ios
pod install                    # Pod 의존성 설치
cd ..
open ios/Runner.xcworkspace    # ⚠️ .xcodeproj 가 아닌 .xcworkspace
```

Xcode → **Runner** 타겟 → **Signing & Capabilities**:
- ✅ Automatically manage signing
- Team 선택
- Bundle Identifier: `com.shopjoy.fo`

**+ Capability**:
- ✅ Push Notifications
- ✅ Background Modes → Remote notifications

---

## 8단계: FlutterFire CLI 로 Firebase 등록

Flutter 는 `flutterfire configure` 가 자동으로 iOS / Android 양쪽 Firebase 등록을 처리:

```bash
dart pub global activate flutterfire_cli
cd _apps/flutter/ShopjoyFoApp
flutterfire configure
```

대화형:
1. Firebase 프로젝트 선택
2. 플랫폼: `android`, `ios`
3. iOS Bundle ID 자동 인식 (`com.shopjoy.fo`)

자동 생성:
- `ios/Runner/GoogleService-Info.plist`
- `android/app/google-services.json`
- `lib/firebase_options.dart`

---

## 9단계: Firebase 에 APNs 키 업로드

1. https://console.firebase.google.com → 프로젝트 → ⚙ → 프로젝트 설정
2. **클라우드 메시징** 탭 → **Apple 앱 구성** → **APNs 인증 키 업로드**
3. 5단계의 `.p8` + Key ID + Team ID 입력 → **업로드**

상세 → [`21.구글설정하기.md`](21.구글설정하기.md) 의 **6단계**.

## 10단계: pubspec.yaml + AppDelegate 설정

`pubspec.yaml`:

```yaml
dependencies:
  firebase_core: ^3.6.0
  firebase_messaging: ^15.1.3
```

```bash
flutter pub get
cd ios && pod install && cd ..
```

`ios/Runner/AppDelegate.swift` (자동 생성됨, Firebase 활성화 시):

```swift
import UIKit
import Flutter
import FirebaseCore

@main
@objc class AppDelegate: FlutterAppDelegate {
  override func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
  ) -> Bool {
    FirebaseApp.configure()
    GeneratedPluginRegistrant.register(with: self)
    return super.application(application, didFinishLaunchingWithOptions: launchOptions)
  }
}
```

## 12단계: 카카오 SDK iOS (사용 시)

`kakao_flutter_sdk` 패키지 사용 시:

1. https://developers.kakao.com → 내 애플리케이션 → **플랫폼 → iOS 플랫폼 등록**
2. **Bundle ID** 입력 (각 환경별)
3. iOS 는 키 해시 등록 없음
4. `ios/Runner/Info.plist` 의 `LSApplicationQueriesSchemes`, `CFBundleURLTypes` 설정 필요 (kakao_flutter_sdk README 참조)

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `pod install` arm64 오류 (M1/M2) | `arch -x86_64 pod install` 또는 CocoaPods 1.15+ |
| `module 'Firebase' not found` | `cd ios && pod deintegrate && pod install` |
| `flutterfire configure` 에러 | `dart pub global activate flutterfire_cli` 재실행 |
| `Code signing required` | Step 7 의 Team / Bundle ID 재확인 |
| 시뮬레이터에서 푸시 토큰 nil | iOS 시뮬레이터는 푸시 미지원 — 실기기 필수 |
| `firebase_options.dart` 갱신 안됨 | `flutterfire configure` 재실행 |
