## A. Android (Play Store)

### A-1. 운영 환경 적용 + AAB 빌드

`npm run sync:prod` 가 다음 작업을 한 번에 수행합니다:
1. `.env.prod` (+ `.env.prod.local`) 로드
2. `capacitor.config.ts` 의 `server.url` 을 운영 URL 로 갱신
3. `www/env.js` 자동 생성 (`window.__APP_ENV_NAME__ = 'prod'`)
4. `npx cap sync` 실행 (네이티브 프로젝트 반영)

```bash
cd _apps/capacitor/ShopjoyFoApp

# 1) 운영 환경 적용
npm run sync:prod

# 2) Android Release AAB 빌드
cd android
./gradlew bundleRelease
```

결과: `app/build/outputs/bundle/release/app-release.aab`

### A-1-1. 빌드 전 검증

```bash
# capacitor.config.ts 가 운영 URL + cleartext=false 인지 확인
grep -A3 "server:" capacitor.config.ts
```

기대:
```
server: {
    url: 'https://illeesam.netlify.app/mainframe#sj_index',
    cleartext: false,
    androidScheme: 'https',
```

> ⚠️ `cleartext: true` 인 채로 운영 빌드를 만들면 Play Store 가 경고할 수 있고, `WEB_CONTENTS_DEBUGGING=true` 인 채로 빌드되면 Apple 심사에서 거절될 수 있습니다.

### A-2. Play Console 접속

https://play.google.com/console → ShopJoy FO 앱 선택.

### A-3. 새 버전 만들기

좌측 **출시 → 테스트 → Internal testing → 새 버전 만들기**.

### A-4. AAB 업로드

**App Bundle 추가** → `app-release.aab` 선택.

업로드 검증 (1~2분) → 자동 통과.

### A-5. 출시 노트 작성

한국어 + 영어 작성:

```
- 첫 출시
- 상품 검색 / 주문 / 결제 / 마이페이지 지원
- 카카오페이 / 토스페이 결제
```

### A-6. 출시

**저장 → 검토 → 내부 테스트 출시**.

검토 즉시 통과 (Internal track).

### A-7. 트랙 승격 (Internal → Closed → Production)

검증 완료 후:
- Internal testing → Closed testing → 새 버전 만들기 → 이전 트랙 빌드 사용
- Closed testing → Production → 이전 트랙 빌드 사용 → **단계적 출시 5%**

## B. iOS (App Store / TestFlight)

### B-1. 운영 환경 적용 + Archive

```bash
cd _apps/capacitor/ShopjoyFoApp

# 운영 환경 적용 (config:prod + cap sync ios)
npm run sync:prod

# Xcode 열기
npm run open:ios
```

Xcode 에서:
1. 좌상단 디바이스 → **Any iOS Device (arm64)**
2. **Product → Archive**

> `npm run ios:prod` 한 번에 실행하면 sync:prod + Xcode 자동 빌드까지 진행되지만, App Store 제출용 Archive 는 Xcode UI 에서 수동 실행 권장.

### B-2. Distribute App

Organizer 자동 오픈 → 방금 만든 Archive → **Distribute App** → **App Store Connect** → **Upload**.

5~15분 업로드.

### B-3. App Store Connect 에서 검증

https://appstoreconnect.apple.com → 앱 → **TestFlight** 탭:

- 업로드 후 5~30분 처리
- ❗ 표시 → **암호화 사용 여부** 입력

### B-4. 내부 테스터 추가

좌측 **앱 내부 테스트 → 내부 테스터** → **+** → 본인 / 동료 Apple ID.

### B-5. iPhone TestFlight 앱에서 설치

iPhone → App Store → `TestFlight` 설치 → 초대 메일에서 **재구매**.

### B-6. App Store 제출

검증 완료 후:
- App Store 탭 → **+ 새 버전 만들기**
- 빌드 선택 → 메타데이터 작성 → **검토 위해 제출**
- 1~3일 검토 → 승인 → 자동 출시

## C. CI 자동 배포 (Codemagic)

저장소 루트 `codemagic.yaml` 이 이미 작성됨.

### C-1. 환경변수 그룹 등록

Codemagic 콘솔 → **Environment variables** → **Add group**:

#### `shopjoy_keystore`
- `CM_KEYSTORE` — `base64 -i shopjoy-fo-release.keystore` 결과
- `CM_KEYSTORE_PASSWORD`
- `CM_KEY_PASSWORD`
- `CM_KEY_ALIAS` = `shopjoy-fo`

#### `google_play`
- `GCLOUD_SERVICE_ACCOUNT_CREDENTIALS` — Play Console 서비스 계정 JSON 전체

#### `shopjoy_secrets` (운영 비밀 — `.env.prod.local` 주입용)
- `KAKAO_PAY_CID_PROD` (secure)
- `TOSS_CLIENT_KEY_PROD` (secure)
- `FIREBASE_KEY_PATH` (secure) — 백엔드 푸시 발송용 서비스계정 JSON (필요시)

CI 빌드 단계에서 이 환경변수를 `.env.prod.local` 로 변환하는 스크립트는 [`31.배포설정하기.md`](31.배포설정하기.md) 의 4-3단계 참조.

### C-2. App Store Connect 통합

Codemagic → **Teams → Integrations → Apple Developer Portal** → API Key 등록.

### C-3. 자동 빌드 트리거

```bash
git tag fo-v1.0.0
git push origin fo-v1.0.0
```

→ Codemagic 자동 빌드 + Play Store Internal + TestFlight 업로드.

## D. OTA 업데이트 (Capgo Live Update)

앱스토어 심사 없이 JS/CSS/HTML 핫 업데이트.

### D-1. 설치

```bash
npm install @capgo/capacitor-updater
npx cap sync
```

### D-2. 코드 추가

```javascript
import { CapacitorUpdater } from '@capgo/capacitor-updater';
import { App } from '@capacitor/app';

CapacitorUpdater.notifyAppReady();

App.addListener('appStateChange', async ({ isActive }) => {
  if (isActive) {
    const latest = await CapacitorUpdater.getLatest();
    if (latest.version !== '<current>') {
      const bundle = await CapacitorUpdater.download({
        url: latest.url,
        version: latest.version
      });
      await CapacitorUpdater.set({ id: bundle.id });
    }
  }
});
```

> ⚠️ 결제 / UI / 기능 변경은 OTA 금지 (앱스토어 정책 위반). 버그픽스 / 문구 수정 위주.
