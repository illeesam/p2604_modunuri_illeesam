## A. Android Studio + AVD 에뮬레이터

### A-1. 의존성 동기화

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run sync:local
```

### A-2. Android Studio 에서 열기

```bash
npm run open:android
```

Android Studio 가 실행되며 `_apps/capacitor/ShopjoyFoApp/android/` 프로젝트 자동 import.

### A-3. AVD 생성

Android Studio → **Tools → Device Manager** → **Create Device**:

- 디바이스: **Pixel 7**
- System Image: **Android 14 (API 34) Google Play x86_64**
- AVD Name: `Pixel_7_API_34`
- **Finish**

### A-4. ▶ 실행

좌상단 디바이스 → AVD 선택 → ▶ 클릭 (Shift+F10).

ShopJoy 앱이 에뮬레이터에 표시되면 성공.

### A-5. 라이브 리로드 워크플로우

1. VS Code Live Server 실행 (포트 5501)
2. Android Studio ▶
3. 웹 코드 (`pages/Home01.js`) 수정 → 저장
4. 에뮬레이터 ShopJoy 앱 → 당겨서 새로고침 또는 메뉴

## C. 환경 전환

### C-1. 환경 파일 위치

```
_apps/capacitor/ShopjoyFoApp/
├── .env.local      # http://10.0.2.2:5501  (안드로이드 에뮬레이터)
├── .env.dev        # https://illeesam.netlify.app/mainframe#sj_index
└── .env.prod       # (운영 URL)
```

### C-2. 환경 전환 명령

| 명령 | 동작 |
|---|---|
| `npm run config:local` | `.env.local` 읽어서 `capacitor.config.ts` + `www/env.js` 갱신 |
| `npm run config:dev` | `.env.dev` 적용 |
| `npm run config:prod` | `.env.prod` 적용 |
| `npm run sync:dev` | config:dev + `npx cap sync` (네이티브 반영) |
| `npm run android:dev` | sync:dev + 안드로이드 빌드/실행 (한 번에) |
| `npm run ios:prod` | sync:prod + iOS 빌드/실행 (macOS) |

### C-3. 전환 검증

```bash
npm run config:dev
```

기대 출력:

```
[set-env] DEV 적용 완료
         APP_NAME       = ShopJoy FO Dev
         BASE_URL       = https://illeesam.netlify.app/mainframe#sj_index
         CLEARTEXT      = false
         ANDROID_SCHEME = https
         (loaded from   .env.dev)
         (www/env.js    10 keys)
```

`capacitor.config.ts` 의 `server.url` 도 자동 변경됨.

### C-4. 에디터 우상단에 환경 표시 (사용자 확인용)

`www/env.js` 가 자동으로 `window.__APP_ENV_NAME__` 을 설정하므로, 웹 코드에서:

```javascript
console.log('현재 환경:', window.__APP_ENV_NAME__);
// sjLog 가 자동으로 [local], [dev], [prod] prefix 출력
sjLog.d('현재 환경 확인'); // → [dev][파일:라인:함수] 현재 환경 확인
```

## D. WebView 디버깅

### D-1. Android (Chrome DevTools)

1. 에뮬레이터에서 ShopJoy 앱 실행 중
2. 데스크톱 Chrome → `chrome://inspect/#devices`
3. ShopJoy 항목 → **inspect** 클릭
4. Console / Network / Sources / Vue DevTools 모두 사용 가능

### D-2. iOS (Safari Web Inspector)

1. 시뮬레이터에서 ShopJoy 앱 실행 중
2. Mac Safari → 개발자용 메뉴 → Simulator → ShopJoy

## E. 자주 쓰는 명령

| 명령 | 동작 |
|---|---|
| `npm run config:local/dev/prod` | server.url 만 변경 |
| `npm run sync:local/dev/prod` | config + cap sync (네이티브 반영) |
| `npm run android:local/dev/prod` | sync + 안드로이드 빌드/실행 |
| `npm run ios:local/dev/prod` | sync + iOS 빌드/실행 (macOS) |
| `npm run open:android` | Android Studio 열기 |
| `npm run open:ios` | Xcode 열기 |
| `npx cap doctor` | 환경 진단 |
| `npx cap sync` | 플러그인 추가 후 네이티브 반영 |

## F. 트러블슈팅

| 증상 | 해결 |
|---|---|
| Gradle Sync 무한 대기 | Android Studio → File → Invalidate Caches / Restart |
| AVD 매우 느림 | x86_64 이미지, 8GB+ RAM 할당 |
| `npx cap sync` 후 변경 미반영 | Build → Clean Project |
| iOS `pod install` 실패 (M1/M2) | `arch -x86_64 pod install` |
| 라이브 리로드 안 됨 | 5501 포트 방화벽, IP 확인 |
