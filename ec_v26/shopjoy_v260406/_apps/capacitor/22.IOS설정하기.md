# 22. Apple Developer (iOS) 설정하기 (Capacitor) — 따라하기

ShopJoy Capacitor iOS 앱의 Apple Developer 인증서 / 프로파일 설정.

> Capacitor iOS 는 결국 Xcode 프로젝트 (`ios/App/App.xcworkspace`) 라서 순수 iOS 와 절차 동일.
> 상세 절차는 [`_apps/ios/22.IOS설정하기.md`](../ios/22.IOS설정하기.md) 참조.

## 사전 준비

- [ ] [`11.개발환경.md`](11.개발환경.md) 완료
- [ ] macOS + Xcode
- [ ] Apple Developer Program 유료 가입 ($99/년)

---

## 1단계: App Identifier 등록

https://developer.apple.com/account → **Certificates, Identifiers & Profiles** → **Identifiers** → **+**:

| 환경 | Bundle ID |
|---|---|
| FO | `com.shopjoy.fo` |
| BO | `com.shopjoy.bo` |

**Capabilities** 에서 **Push Notifications** ✅ 체크 → **Register**.

---

## 2단계: APNs 인증 키 발급

좌측 **Keys** → **+** → 이름 `ShopJoy APNs Key` → **Apple Push Notifications service (APNs)** ✅ → **Continue** → **Register** → **Download**.

`.p8` 파일 + Key ID + Team ID 메모.

---

## 3단계: Distribution Certificate

좌측 **Certificates** → **+** → **Apple Distribution**:
1. CSR 생성 (키체인 접근 → 인증 기관에서 인증서 요청)
2. CSR 업로드 → 인증서 다운로드 → 키체인에 추가

---

## 4단계: Provisioning Profile

좌측 **Profiles** → **+** → **App Store** → 본인 Bundle ID + Distribution Certificate → 이름 입력 → **Generate** → 다운로드 → 더블클릭하여 Xcode 자동 등록.

---

## 5단계: Capacitor 프로젝트에 적용

### 5-1. Xcode 열기

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run open:ios
```

### 5-2. 자동 서명 설정

Xcode → 좌측 Project Navigator → **App** → **Signing & Capabilities**:
- ✅ Automatically manage signing
- Team 선택

### 5-3. Bundle ID 확인

`com.shopjoy.fo` 가 정확히 표시되는지 확인. 필요시 `capacitor.config.ts` 의 `appId` 와 동기화:

```bash
# 운영 빌드 시
cd _apps/capacitor/ShopjoyFoApp
npm run sync:prod          # = config:prod (.env.prod 적용) + npx cap sync
```

> `npm run sync:{env}` 가 `.env.{env}` 를 읽고 `capacitor.config.ts` 와 `www/env.js` 를 갱신한 뒤 `cap sync` 를 호출합니다.

### 5-4. 환경별 Bundle ID 분리 (선택)

기본 ShopJoy Capacitor 는 `appId: com.shopjoy.fo` 단일이지만, 환경별 분리하려면 `capacitor.config.ts` 의 `appId` 를 `com.shopjoy.fo.local`, `com.shopjoy.fo.dev` 등으로 변경하고 Apple Developer 에서 각 Bundle ID 별 App Identifier + Provisioning Profile 발급 필요.

> Capacitor 의 `appId` 는 빌드 시점에 결정되므로, 환경별 분리는 `set-env.js` 가 `.env.{env}` 의 별도 키 (예: `BUNDLE_ID_SUFFIX`) 를 읽어 `capacitor.config.ts` 의 `appId` 도 갱신하도록 확장 가능. (현재는 미적용)

## 6단계: Push Notifications Capability

Xcode → Signing & Capabilities 탭 → **+ Capability**:
- **Push Notifications** 추가
- **Background Modes** 추가 → ✅ Remote notifications
