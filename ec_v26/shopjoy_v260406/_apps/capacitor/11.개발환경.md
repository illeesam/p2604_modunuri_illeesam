# 11. Capacitor 개발환경 구축 — 따라하기

이 문서를 따라하면 Capacitor 개발 환경이 완성됩니다.

## 사전 준비 체크리스트

- [ ] Windows 10/11, macOS 14+, 또는 Ubuntu 22+
- [ ] 디스크 여유공간 30 GB 이상
- [ ] 인터넷 연결됨

---

## 1단계: Node.js 20 LTS 설치

### 1-1. 설치

**Windows**:
```powershell
winget install OpenJS.NodeJS.LTS
```

**macOS**:
```bash
brew install node@20
brew link node@20
```

### 1-2. 확인

```bash
node --version    # v20.x.x
npm --version     # 10.x.x
```

---

## 2단계: JDK 17 설치 (Android 빌드용)

**Windows**:
```powershell
winget install EclipseAdoptium.Temurin.17.JDK
```

**macOS**:
```bash
brew install --cask temurin@17
```

확인:
```bash
java -version    # openjdk 17.x.x
```

---

## 3단계: Android Studio 설치

https://developer.android.com/studio 다운로드 → 설치 → 첫 실행 시 SDK 자동 다운로드.

### 3-1. SDK 환경변수 설정

**Windows (PowerShell)**:
```powershell
[System.Environment]::SetEnvironmentVariable('ANDROID_HOME', "$env:LOCALAPPDATA\Android\Sdk", 'User')
$path = [System.Environment]::GetEnvironmentVariable('Path', 'User')
$path += ";$env:LOCALAPPDATA\Android\Sdk\platform-tools;$env:LOCALAPPDATA\Android\Sdk\emulator"
[System.Environment]::SetEnvironmentVariable('Path', $path, 'User')
```

**macOS** (`~/.zprofile`):
```bash
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator
```

새 터미널 열고 확인:
```bash
adb --version
```

## 4단계: Xcode 설치 (macOS, iOS 빌드 시만)

App Store → Xcode 설치 → `xcode-select --install` → `sudo gem install cocoapods`.

> Windows 에서는 iOS 빌드 불가. 자세한 내용은 [`_apps/ios/11.개발환경.md`](../ios/11.개발환경.md) 참조.

## 6단계: 환경 파일 구조 이해 + 동작 확인

### 6-1. 환경 파일 (3개 + 선택 1개)

각 앱(`ShopjoyFoApp` / `ShopjoyBoApp`) 루트에 다음 6개 파일이 이미 준비되어 있습니다:

```
_apps/capacitor/ShopjoyFoApp/
├── .env.local                ← 로컬 개발 (호스트 PC Live Server)
├── .env.dev                  ← Netlify 개발 배포
├── .env.prod                 ← Netlify 운영
├── .env.prod.local           ← (선택) 운영 비밀 오버라이드 — git 무시
├── scripts/set-env.js        ← .env 읽어서 capacitor.config.ts + www/env.js 생성
└── capacitor.config.ts       ← server.url 자동 갱신됨
```

| 파일 | 커밋 여부 | 용도 |
|---|---|---|
| `.env.local` / `.env.dev` / `.env.prod` | ✅ 커밋 | URL / 공개 키만 (운영 비밀 X) |
| `.env.{env}.local` | ❌ git 무시 | 운영 가맹점 키 등 비밀 (CI 환경변수로도 가능) |

### 6-2. 환경 키 항목 (10개 — FO 기준)

```env
APP_ENV=local
APP_NAME=ShopJoy FO Local
BASE_URL=http://10.0.2.2:5501
CLEARTEXT=true
ANDROID_SCHEME=http
DEEP_LINK_SCHEME=shopjoy-fo
WEB_CONTENTS_DEBUGGING=true
ALLOW_MIXED_CONTENT=true
KAKAO_PAY_CID=TC0ONETIME
TOSS_CLIENT_KEY=test_ck_...
```

BO 앱은 `BASE_URL=http://10.0.2.2:5501/bo.html` + 9개 키 (`ENABLE_BIOMETRIC=true` 추가).

### 6-3. set-env.js 의 동작

```bash
cd _apps/capacitor/ShopjoyFoApp
npm run config:local
```

set-env.js 가 다음 작업을 수행:

1. **`.env.local`** 파일 로드
2. **`.env.local.local`** 이 있으면 머지 (운영 비밀 오버라이드)
3. **`capacitor.config.ts`** 의 `server.url`, `cleartext`, `androidScheme` 갱신
4. **`www/env.js`** 자동 생성 — 웹 코드에서 `window.__APP_ENV__` / `window.__APP_ENV_NAME__` 으로 접근

기대 출력:

```
[set-env] LOCAL 적용 완료
         APP_NAME       = ShopJoy FO Local
         BASE_URL       = http://10.0.2.2:5501
         CLEARTEXT      = true
         ANDROID_SCHEME = http
         (loaded from   .env.local)
         (www/env.js    10 keys)
```

### 6-4. 다른 환경 시도

```bash
npm run config:dev    # → BASE_URL = https://illeesam.netlify.app/mainframe#sj_index
npm run config:prod   # → BASE_URL = (운영 URL)
```

### 6-5. 웹 코드에서 환경 접근

`www/env.js` 가 자동 생성되므로 웹 페이지에서:

```html
<script src="env.js"></script>
<script src="sjLog.js"></script>
<script>
  console.log(window.__APP_ENV_NAME__);          // 'local' | 'dev' | 'prod'
  console.log(window.__APP_ENV__.APP_NAME);      // 'ShopJoy FO Local'
  console.log(window.__APP_ENV__.BASE_URL);
  sjLog.appStart();                              // 모든 빌드에서 콘솔에 빌드 정보 출력
</script>
```

> ⚠️ `www/env.js` 는 `.gitignore` 에 등록되어 자동 생성/무시됩니다.
