# 20. Google 계정 가입 + Android 인증번호(SHA-1/SHA-256) 받기 — 따라하기 (Capacitor)

이 문서를 따라하면 Google 계정 가입과 Firebase / 카카오 / Google Sign-In 등에 등록할 **앱 인증번호(SHA-1, SHA-256 fingerprint)** 를 받을 수 있습니다.

> Capacitor 앱은 Android 빌드 시 일반 Android 키스토어를 사용하므로 절차가 동일합니다.

## 사전 준비

- [ ] 인터넷 연결됨
- [ ] [`11.개발환경.md`](11.개발환경.md) 의 JDK 17 설치 완료 (`keytool` 명령 사용)

---

## 1단계: Google 계정 만들기

### 1-1. Google 계정 페이지 접속

브라우저에서 https://accounts.google.com/signup 접속.

### 1-2. 가입 정보 입력

| 항목 | 입력값 |
|---|---|
| 이름 | 본인 이름 |
| 사용자 이름 | `shopjoy.dev@gmail.com` 같은 식별 가능한 이메일 |
| 비밀번호 | 강력한 비밀번호 (12자+, 특수문자 포함) |

→ **다음**.

### 1-3. 전화번호 인증

휴대전화 번호 입력 → SMS 인증코드 6자리 입력 → **확인**.

### 1-4. 복구 정보 + 약관 동의

복구 이메일 / 생년월일 / 성별 → **다음** → **개인정보 보호 및 약관** → **동의** → 가입 완료.

### 1-5. 2단계 인증 활성화 (필수 권장)

1. https://myaccount.google.com/security 접속
2. **2단계 인증** → **사용** 클릭
3. SMS 또는 Authenticator 앱으로 설정

> ⚠️ **운영 키 / Firebase 관리자 계정은 반드시 2단계 인증 필수**.

## 3단계: 디버그 키스토어 SHA-1 / SHA-256 받기

### 3-1. 키스토어 위치

| OS | 위치 |
|---|---|
| **Windows** | `C:\Users\<USERNAME>\.android\debug.keystore` |
| **macOS / Linux** | `~/.android/debug.keystore` |

확인 (Capacitor 프로젝트에서):

```powershell
# Windows
Test-Path "$env:USERPROFILE\.android\debug.keystore"
```

```bash
# macOS / Linux
ls -la ~/.android/debug.keystore
```

없으면 Capacitor 프로젝트의 Android 를 한 번 빌드:

```powershell
cd _apps\capacitor\ShopjoyFoApp
npm run android:local
# 첫 빌드 시 ~/.android/debug.keystore 자동 생성
```

### 3-2. SHA-1 / SHA-256 추출

#### Windows (PowerShell)

```powershell
keytool -list -v `
  -keystore "$env:USERPROFILE\.android\debug.keystore" `
  -alias androiddebugkey `
  -storepass android `
  -keypass android
```

#### macOS / Linux

```bash
keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android \
  -keypass android
```

### 3-3. 출력에서 인증번호 복사

```
인증서 핑거프린트:
         SHA1: A1:B2:C3:...
         SHA-256: A1:B2:C3:...
```

**SHA1** + **SHA-256** 두 줄 모두 복사.

> 디버그 keystore SHA 는 PC 마다 다름. 팀원 각자 등록 필요.

## 4단계: 운영 keystore SHA 받기

Capacitor 운영 빌드는 [`31.배포설정하기.md`](31.배포설정하기.md) 의 **2단계** 따라 별도 keystore 사용.

### 4-1. 키스토어 생성 (이미 있으면 skip)

```powershell
keytool -genkey -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo `
  -keyalg RSA -keysize 2048 -validity 10000
```

> ⚠️ **분실 시 Play Store 업데이트 영구 불가**. 안전 보관 필수.

### 4-2. SHA 추출

```powershell
keytool -list -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo
```

비밀번호 입력 → SHA-1 / SHA-256 출력.

### 4-3. Play App Signing SHA (Play Console 에서 확인)

Play App Signing 사용 시 Play Console 의 서명 키 SHA 도 별도로 등록:

1. Play Console → 앱 → **설정 → 앱 서명**
2. **앱 서명 키 인증서** 의 SHA-1 / SHA-256 복사

## 5단계: 인증번호 등록 위치

### 5-1. Firebase Console

[`21.구글설정하기.md`](21.구글설정하기.md) 의 **A.5단계** 참조:

1. https://console.firebase.google.com → 프로젝트 → ⚙ → **프로젝트 설정**
2. Android 앱 (`com.shopjoy.fo`) 카드 → **SHA 인증서 지문 추가**
3. SHA-1 + SHA-256 모두 입력
4. 새 `google-services.json` 다운로드 → `_apps/capacitor/ShopjoyFoApp/android/app/` 에 재배치
5. `npx cap sync android` 실행

### 5-2. Kakao Developers (카카오 SDK 사용 시)

1. https://developers.kakao.com → 내 애플리케이션
2. **플랫폼 → Android 플랫폼 등록**
3. 패키지명: `com.shopjoy.fo` (Capacitor `appId`)
4. **키 해시** 등록 (SHA-1 의 base64):

```bash
# 디버그 키 해시 (한 번에)
keytool -exportcert -alias androiddebugkey \
  -keystore ~/.android/debug.keystore \
  -storepass android -keypass android \
  | openssl sha1 -binary \
  | openssl base64

# 운영 키 해시
keytool -exportcert -alias shopjoy-fo \
  -keystore shopjoy-fo-release.keystore \
  | openssl sha1 -binary \
  | openssl base64
```

### 5-3. Google Cloud Console (Google Sign-In 등)

1. https://console.cloud.google.com → 프로젝트
2. **API 및 서비스 → 사용자 인증 정보**
3. **OAuth 클라이언트 ID** → Android → 패키지명 + SHA-1
