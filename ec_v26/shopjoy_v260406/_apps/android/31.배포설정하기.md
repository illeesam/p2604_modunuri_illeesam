# 31. 배포 설정하기 (안드로이드)

ShopJoy 안드로이드 앱을 Play Store 에 배포하기 위한 사전 설정.

## 1. 키스토어 생성 (최초 1회, 분실 주의)

```powershell
keytool -genkey -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo `
  -keyalg RSA -keysize 2048 -validity 10000
```

| 입력 항목 | 예시 |
|---|---|
| 키스토어 비밀번호 | (강력한 비밀번호) |
| 별칭 비밀번호 | (동일 또는 별도) |
| 이름 | ShopJoy |
| 조직 단위 | Mobile |
| 조직 | ShopJoy Inc. |
| 도시 / 도/주 / 국가 | Seoul / Seoul / KR |

> ⚠️ **키스토어 분실 시 앱 업데이트 영구 불가**. 클라우드 안전 보관 + 백업 필수 (1Password 등).

## 2. 키스토어를 Gradle 에 등록

`_apps/android/ShopjoyFoApp/key.properties` (gitignore 처리):

```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=shopjoy-fo
storeFile=../shopjoy-fo-release.keystore
```

`app/build.gradle.kts` 에 signingConfigs 추가:

```kotlin
val keystorePropertiesFile = rootProject.file("key.properties")
val keystoreProperties = java.util.Properties()
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(java.io.FileInputStream(keystorePropertiesFile))
}

android {
    signingConfigs {
        create("release") {
            keyAlias = keystoreProperties.getProperty("keyAlias")
            keyPassword = keystoreProperties.getProperty("keyPassword")
            storeFile = file(keystoreProperties.getProperty("storeFile"))
            storePassword = keystoreProperties.getProperty("storePassword")
        }
    }
    buildTypes {
        release {
            signingConfig = signingConfigs.getByName("release")
            isMinifyEnabled = true
            proguardFiles(getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro")
        }
    }
}
```

```
key.properties

```

## 4. 버전 관리 정책

| 항목 | 형식 | 변경 시점 |
|---|---|---|
| `versionName` | `1.2.3` | 기능 추가/수정 |
| `versionCode` | 정수 단조증가 | 빌드마다 +1 |

권장 산정 공식: `versionCode = MAJOR * 10000000 + MINOR * 100000 + PATCH * 1000 + BUILD`
- `1.2.3` 빌드 5회 → `10203005`

## 5. ProGuard 규칙 검토

`app/proguard-rules.pro`:

```proguard
# WebView JS 인터페이스 보호
-keepattributes *Annotation*
-keepattributes JavascriptInterface

-keep class com.shopjoy.fo.** { *; }

# 카카오 SDK 사용 시
# -keep class com.kakao.** { *; }
# -keepclassmembers class com.kakao.** { *; }

# Firebase 사용 시
# -keep class com.google.firebase.** { *; }
```

## 6. 앱 아이콘 + 스플래시

### 아이콘 생성

Android Studio:
1. `app/src/main/res` 우클릭 → **New → Image Asset**
2. **Launcher Icons (Adaptive and Legacy)** 선택
3. Foreground Layer: 본인 PNG (1024×1024 권장, 안전영역 432×432)
4. Background Layer: 단색 또는 PNG
5. Generate → mipmap-* 자동 생성

### 스플래시

`res/drawable/splash_background.xml` 이미 생성됨 → 본인 로고 PNG로 교체.

## 7. 권한 최소화

`AndroidManifest.xml` 의 권한 검토 — Play Store는 사용 안 하는 권한이 있으면 거절될 수 있음.

현재 등록된 권한:
- `INTERNET` — 필수
- `ACCESS_NETWORK_STATE` — 필수
- `POST_NOTIFICATIONS` — FCM 사용 시
- `CAMERA`, `READ_MEDIA_IMAGES` — 상품 사진 등록 시 (FO만)
- `USE_BIOMETRIC` — 관리자 빠른 로그인 (BO만)
- `VIBRATE` — 햅틱

> 권한 정당화 문구 필요 시 Play Console → 앱 콘텐츠 → 민감 권한 사용 사유 작성.

## 8. minSdkVersion 정책

ShopJoy 기준: **minSdkVersion 24 (Android 7.0)**.

- 24 미만 사용자 비율: 약 5% (2026년 기준)
- 변경 시 `app/build.gradle.kts` 의 `minSdk = 24` 수정
