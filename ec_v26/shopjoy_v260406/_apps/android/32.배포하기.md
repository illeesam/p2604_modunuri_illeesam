# 32. 배포하기 (안드로이드)

ShopJoy 안드로이드 앱을 Google Play Store 에 업로드하는 절차.

## 1. AAB (Android App Bundle) 빌드

Play Store 는 AAB 만 허용 (APK 는 사이드로딩 / 내부 배포용).

```powershell
cd _apps\android\ShopjoyFoApp
.\gradlew bundleProdRelease

# 결과: app\build\outputs\bundle\prodRelease\app-prod-release.aab
```

BO 앱:

```powershell
cd _apps\android\ShopjoyBoApp
.\gradlew bundleProdRelease
```

## 2. AAB 내용 검증

```powershell
# AAB 메타정보 확인
java -jar bundletool.jar dump manifest --bundle=app-prod-release.aab

# APK로 변환 (실기기 테스트용)
java -jar bundletool.jar build-apks `
  --bundle=app-prod-release.aab `
  --output=app-prod-release.apks `
  --connected-device

java -jar bundletool.jar install-apks --apks=app-prod-release.apks
```

> bundletool: https://github.com/google/bundletool/releases

## 3. Play Console 업로드 (수동)

1. https://play.google.com/console → 앱 선택
2. **출시 → 테스트 → Internal testing**
3. **새 버전 만들기**
4. AAB 파일 업로드
5. 출시 노트 작성 (한국어 + 영어)
6. **검토 → 내부 테스트로 출시**

검토는 보통 즉시 (Internal track), 1~3일 (Production).

## 4. 트랙 승격 (Promote)

Internal → Closed → Production 단계적 승격:

1. Internal testing 검증 통과
2. **출시 → 테스트 → Closed testing → 새 버전 만들기 → 이전 트랙 빌드 사용** (재빌드 X)
3. Closed testing 검증 통과
4. **출시 → Production → 새 버전 만들기 → 이전 트랙 빌드 사용**
5. **단계적 출시** (예: 5% → 20% → 50% → 100%)

## 5. 자동 업로드 (Codemagic / GitHub Actions)

수동 업로드 대신 CI 가 자동 업로드.

### Codemagic 예시

Play Console → API 액세스 → Service Account JSON 다운로드.
Codemagic 환경변수 그룹 `google_play` 에 `GCLOUD_SERVICE_ACCOUNT_CREDENTIALS` 등록.

`codemagic.yaml` 의 `fo-android-release` 워크플로우:

```yaml
publishing:
  google_play:
    credentials: $GCLOUD_SERVICE_ACCOUNT_CREDENTIALS
    track: internal              # internal / alpha / beta / production
    submit_as_draft: true        # 자동 출시 X, 검토용 초안만
```

`git tag fo-v1.0.0 && git push origin fo-v1.0.0` → 자동 빌드 + Play Console 업로드.

### GitHub Actions

`.github/workflows/android-release.yml`:

```yaml
- uses: r0adkll/upload-google-play@v1
  with:
    serviceAccountJsonPlainText: ${{ secrets.GOOGLE_PLAY_SA_JSON }}
    packageName: com.shopjoy.fo
    releaseFiles: app/build/outputs/bundle/prodRelease/app-prod-release.aab
    track: internal
    status: draft
```

## 6. 환경별 분리 배포

ShopJoy 는 같은 Play Console 앱에 **3개 트랙**으로 분리 가능:

| 환경 | applicationId | Play Console 트랙 |
|---|---|---|
| local | `com.shopjoy.fo.local` | (배포 안 함, 사내 테스트만) |
| dev | `com.shopjoy.fo.dev` | Internal testing 또는 별도 앱 등록 |
| prod | `com.shopjoy.fo` | Production |

또는 **prod 만 Play Store, dev/local 은 사내 APK 직접 배포** (인스턴트 검증).

## 7. 단계적 출시 권장 흐름

| 일자 | 트랙 | 비율 | 비고 |
|---|---|---|---|
| D-day | Internal | 100% (50명) | 즉시, 사내 검증 |
| D+1 | Closed | 100% (베타 그룹) | 1~2일 검토 |
| D+3 | Production | **5%** | 사용자 일부만 |
| D+5 | Production | **20%** | 크래시율 지표 확인 후 |
| D+7 | Production | **50%** | 안정 시 |
| D+10 | Production | **100%** | 전체 출시 |

## 8. 출시 후 모니터링

- **Play Console → 통계** → 설치 / 제거 / 평점
- **Play Console → 안정성** → ANR (Application Not Responding) / 크래시
- **Firebase Crashlytics** (선택) → 자세한 스택트레이스
