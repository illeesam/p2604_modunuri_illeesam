# 12. 핸드폰 USB로 개발/디버그하기

실제 안드로이드 핸드폰에 ShopJoy 앱을 설치하고 디버깅하는 워크플로우.

## Step 1. 핸드폰 설정

### 1.1 개발자 옵션 활성화

1. 설정 → **휴대전화 정보** → **빌드번호** 7번 연속 탭
2. "개발자가 되었습니다" 메시지 표시
3. 설정 → 시스템 → **개발자 옵션** 진입 가능

### 1.2 USB 디버깅 ON

설정 → 개발자 옵션 → **USB 디버깅** 토글 ON

### 1.3 USB 케이블 연결

- 정품 데이터 케이블 사용 권장 (충전 전용 케이블은 인식 안 됨)
- 핸드폰 화면에서 **MTP / 파일 전송** 모드 선택
- "USB 디버깅을 허용하시겠습니까?" 다이얼로그 → **이 컴퓨터에서 항상 허용** 체크 + **확인**

## Step 2. PC에서 인식 확인

```powershell
adb kill-server
adb start-server
adb devices -l
```

기대 출력:
```
List of devices attached
ABCD123456789       device usb:1-1.2 product:taro model:Pixel_7 device:taro
```

`unauthorized` 가 보이면 → 핸드폰 화면 다이얼로그 다시 확인.
빈 목록이면 → USB 케이블 / 포트 / 디버깅 옵션 재확인.

## Step 3. 빌드 + 실기기 설치

### 3.1 환경별 APK 빌드

```powershell
cd _apps\android\ShopjoyFoApp
.\gradlew assembleLocalDebug          # local 환경
.\gradlew assembleDevDebug            # dev 환경
.\gradlew assembleProdDebug           # prod 환경
```

### 3.2 실기기 설치

```powershell
# 자동 빌드 + 설치 + 실행
.\gradlew installLocalDebug

# 또는 수동 설치
adb install -r app\build\outputs\apk\local\debug\app-local-debug.apk
```

여러 기기 연결 시 특정 기기 지정:

```powershell
adb -s ABCD123456789 install -r ...\app-local-debug.apk
```

## Step 4. 디버깅

### 4.1 Chrome DevTools (WebView 디버깅)

1. 데스크톱 Chrome → `chrome://inspect`
2. 핸드폰에서 ShopJoy 앱 실행 중
3. **Remote Target** 섹션 → ShopJoy → **inspect**
4. WebView 의 Console / Network / Sources / DOM 모두 디버깅 가능

### 4.2 Logcat

```powershell
# 전체 로그
adb logcat

# ShopJoy 관련만 필터링
adb logcat | Select-String "ShopjoyFoApp|ShopjoyBoApp"

# 특정 태그
adb logcat ShopjoyFoApp:D *:S
```

### 4.3 Android Studio 디버거 연결

1. Android Studio 에서 **Run → Attach Debugger to Android Process**
2. 실행 중인 핸드폰 + 앱 프로세스 선택
3. Kotlin 코드에 브레이크포인트 + Variable 검사

## Step 5. 호스트 PC 의 Live Server 접근 (local 환경)

`local` flavor 의 BASE_URL = `http://10.0.2.2:5501` 인데, **이는 에뮬레이터 전용**.
실기기는 PC와 같은 네트워크의 LAN IP 사용 필요.

### 5.1 PC IP 확인

```powershell
ipconfig | Select-String "IPv4"
# 예: 192.168.0.100
```

### 5.2 build.gradle.kts 임시 수정 (실기기 테스트 시)

```kotlin
create("local") {
    buildConfigField("String", "BASE_URL", "\"http://192.168.0.100:5501\"")
    // ...
}
```

또는 `local.properties` 에 환경 변수로 주입.

### 5.3 PC 방화벽 허용

Windows 방화벽 → 인바운드 규칙 → 포트 5501 허용.

## Step 6. 무선 디버깅 (Android 11+)

USB 없이 Wi-Fi 만으로 디버깅:

1. 핸드폰 개발자 옵션 → **무선 디버깅** ON
2. 핸드폰: **페어링 코드로 기기 페어링** → IP/Port + 코드 표시
3. PC:
   ```powershell
   adb pair 192.168.0.50:12345
   # 페어링 코드 입력
   adb connect 192.168.0.50:54321
   adb devices
   ```

## 트러블슈팅

| 증상 | 해결 |
|---|---|
| `adb devices` 빈 목록 | 케이블 / USB 디버깅 옵션 / 다이얼로그 허용 재확인 |
| `unauthorized` | 핸드폰 다이얼로그 미수락 — 케이블 재연결 후 다시 |
| `INSTALL_FAILED_VERSION_DOWNGRADE` | 같은 ID 의 더 높은 버전 이미 설치됨 → `adb uninstall com.shopjoy.fo.local` |
| `INSTALL_FAILED_INSUFFICIENT_STORAGE` | 핸드폰 저장공간 부족 |
| WebView 가 빈 화면 | local 환경인데 PC LAN IP 미수정 + 방화벽 |
