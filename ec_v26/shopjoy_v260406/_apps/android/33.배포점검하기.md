# 33. 배포 점검하기 (안드로이드)

Play Store 출시 전 / 후 점검 체크리스트.

## A. 빌드 전 (Pre-build)

- [ ] `versionName` 증가
- [ ] `versionCode` 증가 (단조증가)
- [ ] 디버그 로그 / `Log.d` / `console.log` 정리 (또는 ProGuard 로 제거)
- [ ] `BuildConfig.BASE_URL` 이 운영 URL 가리키는지 확인 (`prod` flavor)
- [ ] `cleartext: true` 사용 안 하는지 (운영은 HTTPS 강제)
- [ ] FCM `google-services.json` 운영 키로 교체
- [ ] 카카오/토스 결제 키 운영 가맹점 ID로 교체
- [ ] 서명 keystore release 설정 적용
- [ ] minSdkVersion / targetSdkVersion 정책 준수 (현재 24/34)

## B. 빌드 후 (Post-build)

### B.1 APK / AAB 검증

```powershell
# AAB 내용 확인
java -jar bundletool.jar dump manifest --bundle=app-prod-release.aab | Select-String "android:versionName|android:versionCode|package="

# 내장된 권한 확인
java -jar bundletool.jar dump manifest --bundle=app-prod-release.aab | Select-String "uses-permission"

# AAB 크기 확인 (50MB 초과 시 Play Asset Delivery 검토)
ls -lh app-prod-release.aab
```

### B.2 실기기 수동 테스트

핵심 플로우:

- [ ] 앱 설치 → 첫 실행
- [ ] 회원 가입 / 로그인 / 자동 로그인
- [ ] 상품 검색 / 조회 / 상세
- [ ] 장바구니 추가 / 수정 / 삭제
- [ ] 주문 → 결제 (카카오/토스)
- [ ] 결제 성공 시 딥링크 복귀
- [ ] 결제 실패/취소 처리
- [ ] 푸시 알림 수신 (포그라운드/백그라운드)
- [ ] 푸시 탭 시 화면 이동
- [ ] 앱 종료 + 재실행 시 토큰 유지

다양한 조건:

- [ ] 화면 회전 / Split Screen
- [ ] 다크 모드
- [ ] 5.4" / 6.1" / 6.7" / 태블릿
- [ ] Android 7 (minSdk) / Android 14 (latest)
- [ ] 오프라인 → 온라인 복귀
- [ ] 저속 네트워크 (3G 시뮬레이션)
- [ ] 백그라운드 → 포그라운드 복귀 (메모리 회수 시 복원)

### B.3 자동화 테스트 (선택)

```bash
# Espresso UI 테스트
./gradlew connectedProdDebugAndroidTest

# Firebase Test Lab (다양한 디바이스 매트릭스)
gcloud firebase test android run --app app-prod-debug.apk --device model=Pixel7,version=34
```

## C. Play Console 등록 점검

- [ ] 앱 아이콘 512×512 PNG (32-bit + 알파)
- [ ] 피쳐 그래픽 1024×500
- [ ] 폰 스크린샷 ≥ 2장 (16:9 또는 9:16)
- [ ] 짧은 설명 (80자 이내) + 자세한 설명 (4000자 이내)
- [ ] 개인정보처리방침 URL 작동 확인
- [ ] 콘텐츠 등급 설문 완료
- [ ] 데이터 안전 양식 작성 (수집 데이터 정직 명시)
- [ ] 광고 포함 여부 정확히 표시
- [ ] 앱 카테고리 (FO: 쇼핑, BO: 비즈니스)
- [ ] 연락처 이메일 + 웹사이트
- [ ] 출시 노트 한국어 + 영어 작성

## D. 사용자 권한 정당화

각 권한이 **앱 기능에서 실제로 사용되는지** 확인:

| 권한 | 정당화 |
|---|---|
| `INTERNET` | API 통신 (자동 승인) |
| `POST_NOTIFICATIONS` | 주문/배송/이벤트 알림 |
| `CAMERA` | 상품 사진 등록 (FO 마이페이지) |
| `READ_MEDIA_IMAGES` | 갤러리에서 상품 이미지 선택 |
| `USE_BIOMETRIC` | 관리자 빠른 로그인 (BO만) |
| `VIBRATE` | 푸시 햅틱 피드백 |

사용 안 하면 Manifest 에서 제거.

## E. 64-bit 지원 확인

```powershell
# 모든 ABI 가 포함되는지 확인
java -jar bundletool.jar build-apks --bundle=app-prod-release.aab --output=tmp.apks
unzip -l tmp.apks | Select-String "lib/"
```

기대: `lib/arm64-v8a/`, `lib/armeabi-v7a/`, `lib/x86_64/` 모두 포함.

> Play Store 는 2019년 8월부터 64-bit 필수. arm64-v8a 가 포함되지 않으면 거절.

## F. 출시 후 모니터링 (D+1, D+3, D+7)

| 지표 | 위치 | 안전 임계값 |
|---|---|---|
| 크래시율 | Play Console → 안정성 | < 0.5% |
| ANR 율 | Play Console → 안정성 | < 0.4% |
| 평점 | Play Console → 통계 | > 4.0 |
| 1성 리뷰 | Play Console → 리뷰 | 즉시 응답 |
| 설치 → 제거 율 | Play Console → 통계 | < 30% |

지표 악화 시:

1. **단계적 출시 일시정지** (Production 비율 더 안 올림)
2. 핫픽스 빌드 → Internal 트랙 우선 검증
3. 안정 확인 후 다음 비율로 진행

## G. 사고 대응 (출시 후 심각한 버그)

1. **즉시 조치**: Play Console → Production → **출시 중단** (Halt rollout)
2. **이전 버전 복구**: 새 버전 출시 못함 (versionCode 증가만 가능). 핫픽스로 빠르게 새 버전 배포.
3. **핫픽스 빌드**: Internal 검증 → Production 5% → 단계적 확대

## H. 심사 거절 사례 & 대응

| 사례 | 원인 | 대응 |
|---|---|---|
| `Minimum Functionality` | WebView 단순 래퍼 | 푸시 / 카메라 / 결제 등 네이티브 기능 명시 |
| `Privacy Policy 누락` | URL 없음 / 접근 불가 | 자체 호스팅, 한국어/영어 동시 제공 |
| `Data Safety 불일치` | 양식과 실제 수집 데이터 다름 | 정확히 신고 |
| `광고 ID 사용` | 광고 ID 신고 누락 | Play Console 양식에서 정직 신고 |
| `백그라운드 위치` | 권한 정당화 부족 | 사용 안 하면 Manifest 제거 |
| `결제 정책 위반` | 외부 결제 사용 | 한국 외부결제 허용 정책 준수 (방통위) |
