# 20. Google 계정 가입 + Android 인증번호(SHA-1/SHA-256) 받기 — 따라하기 (React Native)

이 문서를 따라하면 Google 계정 가입과 Firebase / 카카오 / Google Sign-In 등에 등록할 **앱 인증번호** 를 받을 수 있습니다.

> RN 의 Android 빌드 (`react-native run-android` / `gradlew bundleRelease`) 도 동일한 키스토어를 사용합니다.

## 2단계: 디버그 keystore SHA 받기

### 2-1. 디버그 keystore 위치

| OS                      | 위치                                            |
| ----------------------- | ----------------------------------------------- |
| **Windows**       | `C:\Users\<USERNAME>\.android\debug.keystore` |
| **macOS / Linux** | `~/.android/debug.keystore`                   |

없으면 RN 안드로이드 빌드를 한 번 실행:

```bash
cd _apps/reactNative/ShopjoyFoApp
npm install
npm run android:local      # 첫 빌드 시 자동 생성
```

### 2-2. SHA 추출 — Gradle signingReport (가장 권장)

```bash
cd _apps/reactNative/ShopjoyFoApp/android
./gradlew signingReport
```

출력에서 `Variant: debug` / `Variant: release` 의 SHA-1 / SHA-256 복사.

### 2-3. 수동 추출 (대안)

```powershell
# Windows
keytool -list -v `
  -keystore "$env:USERPROFILE\.android\debug.keystore" `
  -alias androiddebugkey `
  -storepass android `
  -keypass android
```

```bash
# macOS / Linux
keytool -list -v \
  -keystore ~/.android/debug.keystore \
  -alias androiddebugkey \
  -storepass android \
  -keypass android
```

### 2-4. 디버그 keystore 가 RN 프로젝트 안에 있는 경우 (구버전 RN)

RN 0.60 이전 버전은 `android/app/debug.keystore` 가 프로젝트 안에 있을 수 있음:

```bash
keytool -list -v -keystore android/app/debug.keystore -alias androiddebugkey \
  -storepass android -keypass android
```

> RN 0.71+ 는 표준 `~/.android/debug.keystore` 사용.

## 3단계: 운영 keystore SHA 받기

### 3-1. 운영 keystore 생성 (없으면)

[`31.배포설정하기.md`](31.배포설정하기.md) 의 **2-1단계** 따라:

```powershell
keytool -genkey -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo `
  -keyalg RSA -keysize 2048 -validity 10000
```

> ⚠️ **분실 시 Play Store 업데이트 영구 불가**.

### 3-2. SHA 추출

```bash
cd _apps/reactNative/ShopjoyFoApp/android
./gradlew signingReport
```

`Variant: release` 의 SHA 복사.

또는:

```powershell
keytool -list -v -keystore shopjoy-fo-release.keystore -alias shopjoy-fo
```

### 3-3. Play App Signing SHA

Play Console → 앱 → **설정 → 앱 서명** → **앱 서명 키 인증서** SHA-1 / SHA-256.

## 4단계: 인증번호 등록 위치

### 4-1. Firebase Console

[`21.구글설정하기.md`](21.구글설정하기.md) 의 **1-2단계** 참조:

1. https://console.firebase.google.com → 프로젝트 → ⚙ → **프로젝트 설정**
2. Android 앱 카드 → **SHA 인증서 지문 추가**
3. SHA-1 + SHA-256 입력
4. 새 `google-services.json` 다운로드 → `_apps/reactNative/ShopjoyFoApp/android/app/` 에 재배치

### 4-2. Kakao Developers (`@react-native-seoul/kakao-login` 등 사용 시)

1. https://developers.kakao.com → 내 애플리케이션
2. **플랫폼 → Android 플랫폼 등록**
3. 패키지명: `com.shopjoy.fo` (RN `applicationId`)
4. **키 해시** (SHA-1 의 base64):

```bash
# 디버그 키 해시
keytool -exportcert -alias androiddebugkey \
  -keystore ~/.android/debug.keystore \
  -storepass android -keypass android \
  | openssl sha1 -binary \
  | openssl base64

# 운영
keytool -exportcert -alias shopjoy-fo \
  -keystore shopjoy-fo-release.keystore \
  | openssl sha1 -binary \
  | openssl base64
```

### 4-3. Google Cloud Console (`@react-native-google-signin/google-signin` 사용 시)

1. https://console.cloud.google.com → 프로젝트
2. **API 및 서비스 → 사용자 인증 정보**
3. **OAuth 클라이언트 ID** → Android → 패키지명 + SHA-1
4. iOS 클라이언트 ID 도 같이 발급 (RN 양쪽 사용 시)

`google-signin` 패키지 사용 시 `webClientId` 도 별도 OAuth Web 클라이언트 ID 필요.
