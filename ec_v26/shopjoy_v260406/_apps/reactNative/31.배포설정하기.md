
## 2단계: Android Keystore

### 2-1. 키스토어 생성

```powershell
keytool -genkey -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo `
  -keyalg RSA -keysize 2048 -validity 10000
```

> 분실 시 Play Store 업데이트 영구 불가. 안전 보관!

### 2-2. android/gradle.properties 에 추가

```properties
SHOPJOY_UPLOAD_STORE_FILE=shopjoy-fo-release.keystore
SHOPJOY_UPLOAD_KEY_ALIAS=shopjoy-fo
SHOPJOY_UPLOAD_STORE_PASSWORD=YOUR_STORE_PASSWORD
SHOPJOY_UPLOAD_KEY_PASSWORD=YOUR_KEY_PASSWORD
```

### 2-3. android/app/build.gradle 수정

```gradle
android {
    signingConfigs {
        release {
            if (project.hasProperty('SHOPJOY_UPLOAD_STORE_FILE')) {
                storeFile file(SHOPJOY_UPLOAD_STORE_FILE)
                storePassword SHOPJOY_UPLOAD_STORE_PASSWORD
                keyAlias SHOPJOY_UPLOAD_KEY_ALIAS
                keyPassword SHOPJOY_UPLOAD_KEY_PASSWORD
            }
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

### 2-4. .gitignore

```
android/gradle.properties
*.keystore
android/app/google-services.json
ios/ShopjoyFoApp/GoogleService-Info.plist
```

> 단, `gradle.properties` 의 일반 설정과 비밀번호 분리하려면 별도 `secrets.properties` 권장.

## 3단계: 앱 아이콘

## 5단계: ATS / Cleartext

### Android: android/app/src/main/AndroidManifest.xml

```xml
<application android:usesCleartextTraffic="false" ...>
```

운영은 HTTP 차단. local 환경만 cleartext 허용:

```xml
<application
    android:networkSecurityConfig="@xml/network_security_config" ...>
```

`res/xml/network_security_config.xml`:

```xml
<network-security-config>
    <base-config cleartextTrafficPermitted="false" />
    <domain-config cleartextTrafficPermitted="true">
        <domain includeSubdomains="false">10.0.2.2</domain>
        <domain includeSubdomains="false">localhost</domain>
    </domain-config>
</network-security-config>
```

### iOS: Info.plist

```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <false/>
    <key>NSAllowsLocalNetworking</key>
    <true/>
</dict>
```

## 7단계: 환경별 빌드 분리

### 7-1. Android Flavor

`android/app/build.gradle`:

```gradle
android {
    flavorDimensions "env"
    productFlavors {
        local { dimension "env"; applicationIdSuffix ".local" }
        dev   { dimension "env"; applicationIdSuffix ".dev" }
        prod  { dimension "env" }
    }
}
```

이후:
```bash
./gradlew assembleProdRelease
./gradlew bundleProdRelease
```

### 7-2. iOS Configuration

Xcode → Project → Configuration:
- Debug-Local / Debug-Dev / Debug-Prod
- Release-Local / Release-Dev / Release-Prod

각 Configuration 별로 `PRODUCT_BUNDLE_IDENTIFIER` + `INFOPLIST_KEY_BASE_URL` 설정.
