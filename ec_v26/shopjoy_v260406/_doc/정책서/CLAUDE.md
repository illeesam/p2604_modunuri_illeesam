## 정책서 작성 및 관리

### 1. 신규 정책서 추가 시

1. 적절한 도메인 폴더에 파일 생성
2. 파일명: `{도메인}.{번호}.{설명}.md` 형식
3. 해당 폴더의 `CLAUDE.md`에 파일 항목 추가
4. 루트 `정책서/CLAUDE.md`에 파일 수 업데이트

### 2. 정책서 수정 시

- DDL/테이블 변경 → `sy.52.ddl단어사전규칙.md` 동시 수정
- 새 공통 코드 → `sy.08.공통코드.md` 동시 수정
- 상태 코드 변경 → 해당 도메인의 `*.01.상태표.md` 동시 수정
- 기술 표준 변경 → `sy.54.네이밍규칙.md`, `sy.55.mybatis쿼리테이블별칭정책.md` 참조

### 3. 정책서 구조 표준

각 정책서 파일은 다음 구조를 유지:

```markdown
# 정책명

정책의 목적 및 범위 (1~2문단)

## 주요 규칙

- 규칙 1: 설명
- 규칙 2: 설명

## 상태 코드 / 필드

(해당되는 경우)
- `*_status_cd`: 설명

## 관련 테이블

`table1`, `table2`, ...

## 관련 화면

| pageId | 라벨 |
|--------|------|

## 제약사항

- 제약 1
- 제약 2
```
