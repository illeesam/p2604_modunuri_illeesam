# 313. 주문 관리 정책

## 목적
주문 생성, 처리, 변경에 대한 정책 정의

## 범위
- 주문 상태 관리
- 주문 결제 관리
- 주문 취소/환불
- 주문 배송 추적

## 주문 상태 (ORDER_STATUS)
| 상태 | 코드 | 설명 | 주문상품상태 |
|------|------|------|-----------|
| 대기 | PENDING | 결제 대기 중 | NORMAL |
| 결제완료 | PAID | 결제 완료 | NORMAL |
| 준비 | PREPARING | 상품 준비 중 | NORMAL |
| 출고 | SHIPPED | 배송 중 | NORMAL |
| 완료 | COMPLETED | 배송 완료 | SOLD |
| 취소 | CANCELLED | 주문 취소 | CANCELLED |
| 반품 | RETURNED | 전체 반품 | RETURNED |

## 주요 정책

### 1. 주문 생성
- **필수항목**:
  - 상품 (최소 1개)
  - 수량
  - 수령자 정보 (이름, 연락처, 주소)
  - 배송메모 (선택)
- **초기상태**: PENDING
- **자동만료**: 미결제 상태 24시간 후 자동 취소

### 2. 주문 가격 정책
- **상품합계**: 판매가 × 수량
- **할인**: 쿠폰할인, 카테고리할인 적용
- **배송료**: 
  - 기본 배송료 (건당)
  - 무료배송 기준액 이상 무료
  - 도서산간 추가요금
- **최종결제**: 상품합계 - 할인 + 배송료
- **적립금**: 구매 후 포인트 적립 (최종결제액의 1%)

### 3. 주문 상태 변경
- **PENDING** (결제대기)
  - → PAID (결제완료): 결제 성공 시
  - → CANCELLED (취소): 24시간 경과 또는 고객요청
- **PAID** (결제완료)
  - → PREPARING (준비): 판매자 확인
  - → CANCELLED (취소): 배송 전 취소
- **PREPARING** (준비중)
  - → SHIPPED (배송중): 출고 시
- **SHIPPED** (배송중)
  - → COMPLETED (완료): 배송 완료 시
- **COMPLETED** (완료)
  - → RETURNED (반품): 반품 신청 시

### 4. 부분배송
- **지원여부**: 여러 판매자 상품은 부분배송 가능
- **배송료**: 각 배송별로 독립 계산
- **추적**: order_item별로 배송 상태 추적

### 5. 주문 취소 정책
- **결제전 취소**: 제한 없음 (PENDING → CANCELLED)
- **결제후 취소**: 배송 전에만 가능 (PAID → CANCELLED)
  - 결제 환불 처리
  - 수수료 차감
- **배송중 취소**: 불가능 (고객센터 문의)
- **완료후 취소**: 불가능 (반품/교환으로 처리)

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| order_id | 주문ID | YYMMDDhhmmss+rand4 |
| member_id | 회원ID | 로그인 회원 필수 |
| order_date | 주문일시 | TIMESTAMP 기본: 현재 |
| total_price | 상품합계 | BIGINT 기본값 0 |
| discount_amt | 할인금액 | BIGINT 기본값 0 |
| coupon_discount | 쿠폰할인 | BIGINT 기본값 0 |
| cache_use | 적립금사용 | BIGINT 기본값 0 |
| pay_price | 실결제금액 | BIGINT 기본값 0 |
| order_status_cd | 상태 | ORDER_STATUS 코드 |
| order_status_cd_before | 변경전상태 | 상태변경 추적 |
| recv_nm | 수령자명 | 필수 |
| recv_phone | 수령자연락처 | 필수 |
| recv_addr | 수령자주소 | 필수 |
| dliv_status_cd | 배송상태 | DLIV_STATUS 코드 |
