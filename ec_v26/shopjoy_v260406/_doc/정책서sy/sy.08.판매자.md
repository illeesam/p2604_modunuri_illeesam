# 222. 판매자(업체) 관리 정책

## 목적
멀티벤더 플랫폼에서 판매자(업체) 등록, 관리, 수수료 정산 정책 정의

## 범위
- 판매자 등록 및 승인
- 판매자 상태 관리
- 판매자 통계 및 정산

## 판매자 상태 (VENDOR_STATUS)
| 상태 | 코드 | 설명 |
|------|------|------|
| 대기 | PENDING | 등록 심사 대기 중 |
| 승인 | APPROVED | 승인되어 판매 가능 |
| 정지 | SUSPENDED | 일시 정지 |
| 종료 | CLOSED | 판매자 계약 종료 |

## 주요 정책

### 1. 판매자 등록
- **필수서류**:
  - 사업자등록증 (사본)
  - 통장 사본
  - 대표자 신분증
  - 계약서
- **심사기간**: 3~5 영업일
- **수수료율**: 기본 5% (변경 가능)

### 2. 판매자 상태 관리
- **PENDING**: 서류 심사 중
  - 심사 기간 초과 시 자동 거절
- **APPROVED**: 상품 판매 가능
  - 상품 등록 및 관리 권한 보유
  - 매출 통계 확인 가능
- **SUSPENDED**: 정책 위반 시 정지
  - 상품 판매 불가
  - 기존 주문은 배송 처리
- **CLOSED**: 판매자 계약 종료
  - 상품 판매 중지
  - 정산 완료 후 계정 폐쇄

### 3. 판매자 정산
- **정산주기**: 월 1회 (매월 마지막 영업일)
- **정산대상**: 판매 완료된 상품
- **수수료 차감**: 판매가 × 수수료율
- **정산방식**: 계약한 계좌로 송금

### 4. 판매자 성과
- **추적항목**:
  - 총 판매액
  - 판매 건수
  - 반품/환불율
  - 고객만족도

### 5. 판매자 제약사항
- **중복등록**: 1인 최대 3개 업체 허용
- **수수료변경**: 분기별 1회만 변경 가능
- **제휴금지**: 동일계좌 사용 금지

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| vendor_id | 판매자ID | YYMMDDhhmmss+rand4 |
| vendor_nm | 업체명 | 20자 이내 |
| vendor_status_cd | 상태 | VENDOR_STATUS 코드 |
| ceo_nm | 대표자명 | 필수 |
| business_no | 사업자등록번호 | 중복 금지 |
| phone | 연락처 | 필수 |
| fee_rate | 수수료율 | 소수점 2자리 (예: 5.00) |
| bank_account | 정산계좌 | 필수 |
| total_sales_amt | 누적판매액 | BIGINT |
| total_orders | 누적주문수 | INTEGER |
| approve_date | 승인일 | TIMESTAMP |
