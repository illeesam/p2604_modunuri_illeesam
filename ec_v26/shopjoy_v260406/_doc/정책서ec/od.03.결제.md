# 314. 결제 관리 정책

## 목적
다양한 결제 수단 통합 관리 및 결제 프로세스 정의

## 범위
- 결제 수단 관리
- 결제 상태 관리
- PG 연동 및 승인
- 환불 처리

## 결제 상태 (PAY_STATUS)
| 상태 | 코드 | 설명 |
|------|------|------|
| 대기 | PENDING | 결제 대기 중 |
| 완료 | COMPLT | 결제 완료 |
| 실패 | FAILED | 결제 실패 |
| 취소 | CANCELLED | 결제 취소 |
| 환불 | REFUNDED | 환불 완료 |

## 결제 수단 (PAY_METHOD_CD)
| 수단 | 코드 | PG | 특이사항 |
|------|------|------|---------|
| 무통장입금 | BANK_TRANSFER | - | 수동 확인 |
| 가상계좌 | VBANK | - | 자동 확인 |
| 토스페이먼스 | TOSS | TOSS | 카드/계좌/간편결제 |
| 카카오페이 | KAKAO | KAKAO | 선결제 |
| 네이버페이 | NAVER | NAVER | 선결제 |
| 핸드폰결제 | MOBILE | - | 통신사 연동 |

## 주요 정책

### 1. 결제 수단별 정책

#### 1-1. 무통장입금
- **입금기한**: 1일
- **확인**: 수동 확인 또는 자동 입금감지
- **환불**: 입금자명으로 환불

#### 1-2. 가상계좌
- **은행**: 국민은행, 우리은행 등 5개
- **입금기한**: 1일
- **자동확인**: 24시간 자동감지
- **자동만료**: 입금기한 경과 후 자동취소

#### 1-3. 토스페이먼스
- **채널** (pay_channel_cd):
  - CARD: 신용카드/체크카드
  - ACCOUNT: 계좌이체
  - KAKAO: 카카오페이
  - NAVER: 네이버페이
- **결제**: PG 리다이렉트
- **승인**: 실시간 승인
- **할부**: 최대 12개월

#### 1-4. 간편결제 (카카오, 네이버)
- **사전 등록**: 사용자가 앱에서 등록
- **1클릭 결제**: 저장된 카드 사용
- **환불**: PG 통해 자동 처리

### 2. 결제 프로세스
1. **결제 생성**: 주문 → 결제 레코드 생성 (PENDING)
2. **PG 연동**: PG사 결제창 또는 API 호출
3. **승인**: PG 승인 응답
   - 성공 → COMPLT (pg_response 저장)
   - 실패 → FAILED (failure_code, failure_reason 기록)
4. **완료**: 주문 상태 업데이트

### 3. 환불 정책
- **환불상태** (refund_status_cd):
  - PENDING: 환불 대기
  - COMPLT: 환불 완료
  - FAILED: 환불 실패
- **환불기한**: 결제일로부터 60일
- **환불수수료**: 기본 무료
  - 카드: 환율손실률 적용 가능
  - 계좌: 수수료 1,000원 차감 가능

### 4. 보안 정책
- **카드정보**: 마스킹 저장 (1234-****-****-5678)
- **암호화**: 중요정보는 TLS 전송
- **PCI DSS**: PCI 기준 준수
- **로깅**: 모든 결제 이력 기록

### 5. 결제 실패 처리
- **자동 재시도**: 1회 (동일 수단)
- **고객 알림**: 이메일/SMS 발송
- **유예기간**: 재결제 가능 기간 24시간

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| pay_id | 결제ID | YYMMDDhhmmss+rand4 |
| order_id | 주문ID | 필수 FK |
| pay_method_cd | 결제수단 | PAY_METHOD_CD 코드 |
| pay_channel_cd | 결제채널 | TOSS만 해당 |
| pay_amt | 결제금액 | BIGINT 필수 |
| pay_status_cd | 결제상태 | PAY_STATUS 코드 |
| pay_status_cd_before | 변경전상태 | 상태변경 추적 |
| pg_company_cd | PG사 | TOSS/KAKAO/NAVER |
| pg_transaction_id | PG거래ID | PG 응답 |
| pg_approval_no | 승인번호 | PG 응답 |
| pg_response | PG응답데이터 | JSON 포맷 |
| refund_amt | 환불금액 | BIGINT 기본값 0 |
| refund_status_cd | 환불상태 | REFUND_STATUS 코드 |
| refund_date | 환불완료일시 | TIMESTAMP |
| failure_code | 실패코드 | PG 오류코드 |
| failure_reason | 실패사유 | 500자 이내 |

## 관련 테이블
- od_pay: 결제 기본 정보
- od_pay_chg_hist: 결제 변경 이력
- od_pay_status_hist: 결제 상태 이력

## 제약사항
- 환불은 결제 완료 상태에서만 가능
- 가상계좌는 조회 24시간 이내 확인 필수
- 할부는 50,000원 이상에서만 지원
