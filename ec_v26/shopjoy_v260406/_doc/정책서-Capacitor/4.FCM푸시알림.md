## Step 1. Firebase 프로젝트 생성

### 1.1 콘솔에서 프로젝트 만들기

1. https://console.firebase.google.com → **프로젝트 추가**
2. 프로젝트 이름: `shopjoy-prod` (또는 `shopjoy-dev`)
3. Google Analytics 활성화 여부 — 일단 **OFF** (필요시 추가)

### 1.2 Cloud Messaging API 활성화

- 프로젝트 설정 → **클라우드 메시징** 탭
- "API (V1)" 활성화 — 자동으로 켜져 있음
- **레거시 서버 키 (필요시)** — 일부 백엔드 라이브러리는 V1 OAuth2 대신 레거시 키 사용

### 1.3 앱 등록 (Android / iOS 각각 등록)

#### FO 앱
- Android 패키지명: `com.shopjoy.fo`
- iOS 번들 ID:    `com.shopjoy.fo`

#### BO 앱
- Android 패키지명: `com.shopjoy.bo`
- iOS 번들 ID:    `com.shopjoy.bo`

> **앱마다 별도 등록**. 즉 Firebase 한 프로젝트에 4개 앱 등록 (FO/Android, FO/iOS, BO/Android, BO/iOS).

## Step 3. iOS 네이티브 설정 (macOS)

### 3.1 GoogleService-Info.plist 다운로드 + 배치

```
_apps/ShopjoyFoApp/ios/App/App/GoogleService-Info.plist
_apps/ShopjoyBoApp/ios/App/App/GoogleService-Info.plist
```

Xcode에서 프로젝트 트리에 드래그앤드롭 (반드시 **Copy items if needed** + **Add to target: App** 체크).

### 3.2 APNs 인증 키 등록

Apple Developer Portal에서:
1. **Certificates, Identifiers & Profiles** → **Keys** → ➕
2. 이름: `ShopJoy APNs Key`, **Apple Push Notifications service (APNs)** 체크
3. 다운로드한 `.p8` 파일 + Key ID + Team ID 메모

Firebase 콘솔 → 프로젝트 설정 → **클라우드 메시징** → **iOS 앱 구성** → APNs 인증 키 업로드.

### 3.3 Capabilities 추가

Xcode → 프로젝트 타겟 → **Signing & Capabilities**:
- ➕ **Push Notifications**
- ➕ **Background Modes** → **Remote notifications** 체크

### 3.4 AppDelegate.swift 수정

`_apps/ShopjoyFoApp/ios/App/App/AppDelegate.swift`:

```swift
import UIKit
import Capacitor
import FirebaseCore
import FirebaseMessaging

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {
    func application(_ application: UIApplication,
                     didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        FirebaseApp.configure()
        return true
    }

    func application(_ application: UIApplication,
                     didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
        Messaging.messaging().apnsToken = deviceToken
        NotificationCenter.default.post(name: .capacitorDidRegisterForRemoteNotifications,
                                        object: deviceToken)
    }

    func application(_ application: UIApplication,
                     didFailToRegisterForRemoteNotificationsWithError error: Error) {
        NotificationCenter.default.post(name: .capacitorDidFailToRegisterForRemoteNotifications,
                                        object: error)
    }
}
```

## Step 4. 웹 코드: 토큰 수신 + 백엔드 등록

### 4.1 공통 푸시 초기화 모듈

`utils/pushNoti.js` (FO/BO 양쪽 프로젝트에 배치):

```javascript
window.pushNoti = (function () {
  const isNative = window.Capacitor && window.Capacitor.isNativePlatform();

  async function init({ apiClient, apiPath, uiNm, cmdNm }) {
    if (!isNative) {
      console.log('[pushNoti] 웹 환경 — 푸시 초기화 스킵');
      return;
    }

    const { PushNotifications } = window.Capacitor.Plugins;
    const { Device } = window.Capacitor.Plugins;

    // 1) 권한 요청
    let perm = await PushNotifications.checkPermissions();
    if (perm.receive === 'prompt' || perm.receive === 'prompt-with-rationale') {
      perm = await PushNotifications.requestPermissions();
    }
    if (perm.receive !== 'granted') {
      console.warn('[pushNoti] 권한 거부됨');
      return;
    }

    // 2) FCM/APNs 등록
    await PushNotifications.register();

    // 3) 토큰 수신 → 백엔드 등록
    PushNotifications.addListener('registration', async (token) => {
      const info = await Device.getInfo();
      try {
        await apiClient.post(apiPath, {
          pushToken: token.value,
          deviceType: info.platform,            // 'ios' | 'android'
          deviceModel: info.model,
          deviceOs: `${info.platform} ${info.osVersion}`,
          appVersion: window.__APP_VERSION__ || '1.0.0',
        }, window.coUtil.apiHdr(uiNm, cmdNm));
        console.log('[pushNoti] 토큰 등록 완료');
      } catch (err) {
        console.error('[pushNoti] 토큰 등록 실패', err);
      }
    });

    // 4) 등록 실패
    PushNotifications.addListener('registrationError', (err) => {
      console.error('[pushNoti] registrationError', err);
    });

    // 5) 포그라운드 푸시 수신
    PushNotifications.addListener('pushNotificationReceived', (noti) => {
      console.log('[pushNoti] 포그라운드 수신', noti);
      if (window.showToast) {
        window.showToast(`${noti.title}: ${noti.body}`, 'success');
      }
    });

    // 6) 푸시 탭 시 화면 이동
    PushNotifications.addListener('pushNotificationActionPerformed', (action) => {
      const data = action.notification.data || {};
      console.log('[pushNoti] 탭', data);
      if (data.pageId && window.navigate) {
        window.navigate(data.pageId, { dtlId: data.dtlId });
      }
    });
  }

  return { init };
})();
```

## Step 5. 백엔드: 토큰 저장 / 발송 API

### 5.1 컨트롤러 (FO 회원용)

`MbDeviceTokenController.java` (FO):

```java
@RestController
@RequestMapping("/api/fo/my/device-token")
@RequiredArgsConstructor
public class MbDeviceTokenFoController {

    private final MbDeviceTokenService service;

    @PostMapping
    public ResponseEntity<ApiResponse<Void>> register(@RequestBody Map<String, Object> p) {
        p.put("memberId", SecurityUtil.currentUserId());
        service.upsertToken(p);
        return ResponseEntity.ok(ApiResponse.ok(null));
    }
}
```

### 5.2 컨트롤러 (BO 관리자용)

```java
@RestController
@RequestMapping("/api/bo/sy/user-device-token")
@RequiredArgsConstructor
public class SyUserDeviceTokenController {

    private final SyUserDeviceTokenService service;

    @PostMapping
    public ResponseEntity<ApiResponse<Void>> register(@RequestBody Map<String, Object> p) {
        p.put("userId", SecurityUtil.currentUserId());
        service.upsertToken(p);
```

### 5.3 발송 서비스 (Firebase Admin SDK)

`build.gradle`:

```gradle
dependencies {
    implementation 'com.google.firebase:firebase-admin:9.3.0'
}
```

`PushSendService.java`:

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class PushSendService {

    @Value("${firebase.service-account-key-path}")
    private String credentialPath;

    @PostConstruct
    public void init() throws IOException {
        if (FirebaseApp.getApps().isEmpty()) {
            FirebaseOptions options = FirebaseOptions.builder()
                .setCredentials(GoogleCredentials.fromStream(new FileInputStream(credentialPath)))
                .build();
            FirebaseApp.initializeApp(options);
        }
    }

    public void sendToToken(String token, String title, String body, Map<String, String> data) {
        Message msg = Message.builder()
            .setToken(token)
            .setNotification(Notification.builder()
                .setTitle(title)
                .setBody(body)
                .build())
            .putAllData(data == null ? Map.of() : data)
            .build();
        try {
            String resp = FirebaseMessaging.getInstance().send(msg);
            log.info("[fcm] sent: {}", resp);
        } catch (FirebaseMessagingException e) {
            log.error("[fcm] send failed: {}", e.getMessage());
        }
    }

    public void sendToTokens(List<String> tokens, String title, String body, Map<String, String> data) {
        if (tokens == null || tokens.isEmpty()) return;
        MulticastMessage msg = MulticastMessage.builder()
            .addAllTokens(tokens)
            .setNotification(Notification.builder()
                .setTitle(title).setBody(body).build())
            .putAllData(data == null ? Map.of() : data)
            .build();
        try {
            BatchResponse resp = FirebaseMessaging.getInstance().sendEachForMulticast(msg);
            log.info("[fcm] success={}, fail={}", resp.getSuccessCount(), resp.getFailureCount());
        } catch (FirebaseMessagingException e) {
            log.error("[fcm] batch send failed: {}", e.getMessage());
        }
    }
}
```

## DB 스키마

### mb_device_token (FO 회원)

```sql
CREATE TABLE mb_device_token (
  device_token_id   BIGSERIAL    PRIMARY KEY,
  site_id           VARCHAR(20)  NOT NULL,
  member_id         BIGINT       NOT NULL,
  push_token        TEXT         NOT NULL,
  device_type       VARCHAR(20)  NOT NULL,         -- 'ios' | 'android'
  device_model      VARCHAR(100),
  device_os         VARCHAR(100),
  app_version       VARCHAR(20),
  is_active         BOOLEAN      NOT NULL DEFAULT TRUE,
  reg_date          TIMESTAMP    NOT NULL DEFAULT now(),
  upd_date          TIMESTAMP,
  last_used_date    TIMESTAMP,
  CONSTRAINT mb_device_token_uq UNIQUE (site_id, member_id, push_token)
);
CREATE INDEX idx_mb_device_token_member ON mb_device_token(site_id, member_id, is_active);
```

### sy_user_device_token (BO 관리자)

```sql
CREATE TABLE sy_user_device_token (
  user_device_token_id BIGSERIAL    PRIMARY KEY,
  user_id              VARCHAR(50)  NOT NULL,
  push_token           TEXT         NOT NULL,
  device_type          VARCHAR(20)  NOT NULL,
  device_model         VARCHAR(100),
  device_os            VARCHAR(100),
  app_version          VARCHAR(20),
  is_active            BOOLEAN      NOT NULL DEFAULT TRUE,
  reg_date             TIMESTAMP    NOT NULL DEFAULT now(),
  upd_date             TIMESTAMP,
  last_used_date       TIMESTAMP,
  CONSTRAINT sy_user_device_token_uq UNIQUE (user_id, push_token)
);
CREATE INDEX idx_sy_user_device_token_user ON sy_user_device_token(user_id, is_active);
```

### upsert 패턴 (PostgreSQL)

```xml
<insert id="upsertToken">
  INSERT INTO mb_device_token /* mb :: MbDeviceTokenMapper :: upsertToken */
    (site_id, member_id, push_token, device_type, device_model, device_os, app_version, is_active)
  VALUES
    (#{siteId}, #{memberId}, #{pushToken}, #{deviceType}, #{deviceModel}, #{deviceOs}, #{appVersion}, true)
  ON CONFLICT (site_id, member_id, push_token) DO UPDATE SET
    device_model    = EXCLUDED.device_model,
    device_os       = EXCLUDED.device_os,
    app_version     = EXCLUDED.app_version,
    is_active       = true,
    upd_date        = now(),
    last_used_date  = now()
</insert>
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| Android: 토큰 미수신 | `google-services.json` 누락/오배치 | `_apps/Shopjoy*App/android/app/` 위치 확인 |
| Android: `Default FirebaseApp is not initialized` | google-services 플러그인 미적용 | `app/build.gradle` 마지막에 `apply plugin` 확인 |
| iOS: 시뮬레이터에서 토큰 없음 | 시뮬레이터는 푸시 미지원 | **실기기 필수** (iOS 16+ 시뮬레이터 일부 지원) |
| iOS: APNs 토큰 인식 실패 | AppDelegate.swift 누락 | Step 3.4 코드 추가 + `pod install` |
| 권한 다이얼로그 안 뜸 | iOS Info.plist 미수정 | `NSUserNotificationsUsageDescription` 추가 |
| 발송했는데 도착 안 함 | 백엔드 토큰 만료 | FCM이 `UNREGISTERED` 응답 시 DB에서 `is_active=false` 처리 |
| 다른 앱(같은 패키지명) 토큰과 충돌 | FO/BO 같은 packageId 사용 | `com.shopjoy.fo` / `com.shopjoy.bo` 분리 (현재 설정 ✓) |
