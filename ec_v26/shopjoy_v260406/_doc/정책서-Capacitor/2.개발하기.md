## 개발 사이클 개요

```
┌──────────────────────────────────────────────────────────┐
│  ① 웹 코드 수정 (pages/, components/, assets/)           │
│  ② 라이브 리로드 → 즉시 반영 (브라우저처럼)              │
│  ③ 네이티브 기능 테스트만 npx cap run android/ios        │
│  ④ 플러그인 추가/제거 시: npx cap sync                  │
│  ⑤ 네이티브 코드 변경 시: Android Studio / Xcode 직접   │
└──────────────────────────────────────────────────────────┘
```

| 변경 종류 | 명령 | 설명 |
|---|---|---|
| HTML/JS/CSS 수정 | (라이브 리로드 시) 자동 | 브라우저처럼 즉시 반영 |
| 새 플러그인 추가 | `npm install ...` → `npx cap sync` | 네이티브 프로젝트에 통합 |
| `capacitor.config.ts` 수정 | `npx cap sync` | 설정 반영 |
| 네이티브 코드 (Java/Kotlin/Swift) 수정 | `npx cap open android` / `ios` | IDE에서 빌드 |
| 앱 아이콘/스플래시 변경 | 리소스 교체 후 `npx cap sync` | — |

## 라이브 리로드 (Hot Reload)

개발 PC에서 Live Server로 띄운 ShopJoy 웹을 모바일 앱이 직접 로드하는 방식. **코드 수정 → 앱 자동 새로고침**.

### 1. 개발 PC IP 확인

```powershell
ipconfig | Select-String "IPv4"
# 예: 192.168.0.100
```

### 2. Live Server 외부 접속 허용

VS Code Live Server 설정 (`.vscode/settings.json`):
```json
{
  "liveServer.settings.host": "0.0.0.0",
  "liveServer.settings.port": 5501
}
```

방화벽에서 5501 포트 허용 (Windows 방화벽 → 인바운드 규칙).

### 3. capacitor.config.ts 수정 (개발 전용)

`_mobile/shopjoy-fo-app/capacitor.config.ts`:
```typescript
import { CapacitorConfig } from '@capacitor/cli';

const isDev = process.env.NODE_ENV !== 'production';

const config: CapacitorConfig = {
  appId: 'com.shopjoy.fo',
  appName: 'ShopJoy',
  webDir: '../../',

  server: isDev ? {
    url: 'http://192.168.0.100:5501',   // ← 본인 PC IP로 변경
    cleartext: true,                    // HTTP 허용 (개발만)
    androidScheme: 'http',
  } : {
    androidScheme: 'https',
  },
};

export default config;
```

> ⚠️ **배포 빌드 시 `server.url` 반드시 제거**. 외부 서버를 가리키면 앱스토어 정책 위반 + 보안 위험.

### 4. 동기화 후 실행

```powershell
npx cap sync
npx cap run android        # 실기기 또는 에뮬레이터 자동 선택
```

이제 `pages/Home01.js` 한 줄 수정 → **에뮬레이터/실기기에 즉시 반영**.

### 5. 라이브 리로드 끄기 (배포 빌드)

```powershell
$env:NODE_ENV="production"
npx cap sync
npx cap build android
```

## 네이티브 플러그인 사용 패턴

### 1. 푸시 알림 (FCM)

```javascript
// utils/pushNoti.js
import { PushNotifications } from '@capacitor/push-notifications';
import { Capacitor } from '@capacitor/core';

export async function initPushNoti() {
  if (!Capacitor.isNativePlatform()) return;       // 웹에서는 스킵

  // 권한 요청
  const perm = await PushNotifications.requestPermissions();
  if (perm.receive !== 'granted') return;

  // FCM/APNs 등록
  await PushNotifications.register();

  // 토큰 수신 → 백엔드 등록 (mb_device_token 테이블 활용)
  PushNotifications.addListener('registration', async (token) => {
    await window.foApi.post('/fo/my/device-token', {
      pushToken: token.value,
      deviceType: Capacitor.getPlatform(),         // 'ios' | 'android'
    }, coUtil.apiHdr('푸시', '토큰등록'));
  });

  // 포그라운드 푸시 수신
  PushNotifications.addListener('pushNotificationReceived', (noti) => {
    showToast(`${noti.title}: ${noti.body}`, 'success');
  });

  // 푸시 탭 시 화면 이동
  PushNotifications.addListener('pushNotificationActionPerformed', (action) => {
    const data = action.notification.data;
    if (data?.pageId) navigate(data.pageId, { dtlId: data.dtlId });
  });
}
```

> **사전 작업**: Firebase 프로젝트 생성 + `google-services.json` (Android) / `GoogleService-Info.plist` (iOS) 다운로드 후 네이티브 프로젝트에 배치. 상세 → 정책서 별도 문서.

### 2. 카메라 / QR 스캔

```javascript
import { Camera, CameraResultType, CameraSource } from '@capacitor/camera';
import { BarcodeScanner } from '@capacitor-mlkit/barcode-scanning';

// 사진 촬영 (상품 이미지 등록)
async function takePhoto() {
  const photo = await Camera.getPhoto({
    quality: 80,
    resultType: CameraResultType.DataUrl,
    source: CameraSource.Camera,
  });
  return photo.dataUrl;            // base64 dataURL
}

// QR 스캔 (운송장 송장 등)
async function scanQr() {
  const { camera } = await BarcodeScanner.requestPermissions();
  if (camera !== 'granted') return null;

  const { barcodes } = await BarcodeScanner.scan();
  return barcodes[0]?.rawValue;
}
```

### 3. 보안 스토리지 (Preferences)

`localStorage` 대신 **네이티브 키체인/Keystore 기반** 저장.

```javascript
// utils/secureStorage.js
import { Preferences } from '@capacitor/preferences';
import { Capacitor } from '@capacitor/core';

export const secureStorage = {
  async set(key, value) {
    if (Capacitor.isNativePlatform()) {
      await Preferences.set({ key, value: JSON.stringify(value) });
    } else {
      localStorage.setItem(key, JSON.stringify(value));
    }
  },
  async get(key) {
    if (Capacitor.isNativePlatform()) {
      const { value } = await Preferences.get({ key });
      return value ? JSON.parse(value) : null;
    } else {
      const v = localStorage.getItem(key);
      return v ? JSON.parse(v) : null;
    }
  },
  async remove(key) {
    if (Capacitor.isNativePlatform()) {
      await Preferences.remove({ key });
    } else {
      localStorage.removeItem(key);
    }
  },
};
```

기존 코드의 `localStorage.setItem('authToken', token)` →
```javascript
await secureStorage.set('authToken', token);
```

### 4. 생체인증 (BO 관리자 앱)

```javascript
import { BiometricAuth } from '@capacitor-community/biometric-auth';

async function loginWithBiometric() {
  const { isAvailable } = await BiometricAuth.isAvailable();
  if (!isAvailable) {
    showToast('생체인증을 사용할 수 없습니다.', 'error');
    return;
  }

  try {
    await BiometricAuth.verify({
      reason: '관리자 로그인',
      title: 'ShopJoy BO',
      cancelTitle: '취소',
    });
    // 인증 성공 → 저장된 토큰으로 자동 로그인
    const token = await secureStorage.get('boAuthToken');
    if (token) await frontAuth.loginWithToken(token);
  } catch (err) {
    showToast('인증 실패', 'error');
  }
}
```

### 5. 결제창 호출 (카카오/토스 — InAppBrowser)

```javascript
import { Browser } from '@capacitor/browser';

async function payWithKakao(orderId) {
  const { data } = await window.foApi.post('/fo/od/pay/kakao/ready', { orderId });
  // data.next_redirect_mobile_url ← 카카오페이 결제 URL

  await Browser.open({
    url: data.next_redirect_mobile_url,
    presentationStyle: 'popover',
  });

  // 결제 완료 후 딥링크로 앱 복귀 → App.addListener('appUrlOpen', ...) 처리
}
```

딥링크 핸들링:
```javascript
import { App } from '@capacitor/app';

App.addListener('appUrlOpen', async (event) => {
  // event.url 예: shopjoy-fo://pay/success?orderId=...
  const u = new URL(event.url);
  if (u.hostname === 'pay' && u.pathname === '/success') {
    await Browser.close();
    navigate('order', { dtlId: u.searchParams.get('orderId') });
  }
});
```

> URL Scheme 등록 (Android: `AndroidManifest.xml`, iOS: `Info.plist`) — 상세는 정책서 별도 문서.

## API 연동 (백엔드 호출)

### 환경별 API Base URL

기존 `utils/foApiAxios.js` / `utils/boApiAxios.js`가 `api/` 상대경로를 사용 중. 모바일에서는 절대 URL이 필요.

`base/config.js`에 환경 분기 추가:
```javascript
import { Capacitor } from '@capacitor/core';

const API_BASE = (() => {
  if (!Capacitor.isNativePlatform()) {
    return '';                                          // 웹: 상대경로 그대로
  }
  // 네이티브: 환경별 절대 URL
  const isProd = !Capacitor.DEBUG;
  return isProd
    ? 'https://api.shopjoy.com'
    : 'http://10.0.2.2:8080';                           // Android 에뮬레이터 → 호스트 PC
})();

window.SITE_CONFIG = {
  ...,
  apiBase: API_BASE,
};
```

`utils/foApiAxios.js` 수정:
```javascript
const foApi = axios.create({
  baseURL: window.SITE_CONFIG.apiBase + '/api',
  // ...
});
```

### 에뮬레이터 → 호스트 PC IP 매핑

| 환경 | 호스트 PC localhost 접근 |
|---|---|
| **Android 에뮬레이터** | `http://10.0.2.2:8080` |
| **iOS 시뮬레이터** | `http://localhost:8080` 또는 `http://127.0.0.1:8080` |
| **실기기 (USB)** | PC LAN IP (예: `http://192.168.0.100:8080`) |

### CORS 처리

Spring Boot `application.yml`에 모바일 origin 허용:
```yaml
cors:
  allowed-origins:
    - "http://10.0.2.2:8080"
    - "capacitor://localhost"        # iOS Capacitor 내부 origin
    - "http://localhost"             # Android Capacitor 내부 origin
    - "https://api.shopjoy.com"
```

## 네이티브 권한 처리

| 권한 | Android (manifest) | iOS (Info.plist) | Capacitor 호출 |
|---|---|---|---|
| 푸시 알림 | (자동) | `NSUserNotificationsUsageDescription` | `PushNotifications.requestPermissions()` |
| 카메라 | `CAMERA` | `NSCameraUsageDescription` | `Camera.requestPermissions()` |
| 갤러리 | `READ_MEDIA_IMAGES` | `NSPhotoLibraryUsageDescription` | `Camera.requestPermissions({ permissions: ['photos'] })` |
| 위치 | `ACCESS_FINE_LOCATION` | `NSLocationWhenInUseUsageDescription` | `Geolocation.requestPermissions()` |
| 생체인증 | `USE_BIOMETRIC` | `NSFaceIDUsageDescription` | `BiometricAuth.verify()` |

iOS의 경우 **`Info.plist`에 사용 사유 누락 시 앱 크래시**. 반드시 사용자 친화적 문구 작성.

`_mobile/shopjoy-fo-app/ios/App/App/Info.plist` 예시:
```xml
<key>NSCameraUsageDescription</key>
<string>상품 사진 촬영을 위해 카메라 접근 권한이 필요합니다.</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>갤러리에서 상품 이미지를 선택하기 위해 권한이 필요합니다.</string>
<key>NSFaceIDUsageDescription</key>
<string>관리자 빠른 로그인을 위해 Face ID를 사용합니다.</string>
```

## 공통 함정

| 증상 | 원인 | 해결 |
|---|---|---|
| `npx cap sync` 후 변경 미반영 | 빌드 캐시 | Android Studio → Build → Clean Project |
| 라이브 리로드가 안 됨 | 방화벽/IP 미일치 | `ipconfig`로 IP 재확인, 5501 포트 허용 |
| `localStorage`가 비어있음 | 앱 재설치 시 초기화 | `Preferences` 플러그인으로 마이그레이션 |
| HTTP 호출 차단 (Android 9+) | Cleartext 미허용 | `capacitor.config.ts`에 `cleartext: true` (개발 only) |
| iOS 시뮬레이터에서 카메라 미동작 | 시뮬레이터는 카메라 없음 | 실기기 테스트 필수 |
| Vue 3 컴포넌트가 모바일에서만 렌더 안 됨 | 스크립트 로드 순서 | `bo.html` 스크립트 순서 그대로 유지 (CLAUDE.md 참조) |
| 푸시 알림 토큰 미수신 (Android) | `google-services.json` 누락 | `_mobile/shopjoy-*/android/app/`에 배치 |
| 앱 시작 시 흰 화면 | webDir 경로 오류 | `capacitor.config.ts`의 webDir 절대경로 검증 |
| `axios` 요청 실패 (네이티브) | baseURL 상대경로 | `Capacitor.isNativePlatform()`으로 절대 URL 분기 |
