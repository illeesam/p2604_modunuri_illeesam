# 6. iOS 빌드 및 Codemagic CI/CD

Windows 환경에서 iOS 앱을 빌드/배포하기 위한 옵션. **결론: 클라우드 빌드 (Codemagic) 가 가장 현실적**.

## 옵션 비교

| 옵션 | 비용 | 장점 | 단점 |
|---|---|---|---|
| **A. 본인 Mac 구매** | Mac mini M4 약 90만원 (1회) | 빠름, 완전 통제 | 초기 비용, 관리 부담 |
| **B. macOS in Cloud** | $30/월 | 1대 통째 임대, 다양한 작업 | Apple T&C 회색지대 |
| **C. Codemagic** | 월 500분 무료 + $0.038/min | 공식 지원, 자동화 좋음 | 시간당 과금 |
| **D. GitHub Actions (macOS runner)** | Public 무료, Private $0.08/min | GitHub 통합 | macOS runner 가격 비쌈 |
| **E. Bitrise** | 월 200분 무료 + $0.05/min | UI 직관적 | Codemagic보다 빌드 약간 느림 |
| **F. Xcode Cloud** | Apple Developer 가입자 월 25시간 무료 | Apple 공식 | 설정이 Xcode-centric |

**ShopJoy 권장**: **C (Codemagic)** + 운영 안정화 후 **F (Xcode Cloud)** 병행.

## Codemagic 세팅

### Step 1. 계정 생성 + 저장소 연결

1. https://codemagic.io → GitHub 계정으로 가입
2. **Add application** → ShopJoy GitHub 레포 선택
3. 빌드 대상: **Capacitor**

### Step 2. codemagic.yaml 작성

저장소 루트에 `codemagic.yaml` 배치:

```yaml
workflows:
  shopjoy-fo-android-debug:
    name: ShopJoy FO — Android Debug
    instance_type: mac_mini_m2
    max_build_duration: 60
    environment:
      vars:
        APP_DIR: "_apps/ShopjoyFoApp"
      node: 20.18.0
      java: 17
    triggering:
      events:
        - push
      branch_patterns:
        - pattern: 'main'
          include: true
    scripts:
      - name: Install dependencies
        script: |
          cd $APP_DIR
          npm ci
      - name: Set Capacitor env (prod)
        script: |
          cd $APP_DIR
          npm run config:prod
          npx cap sync android
      - name: Build Android APK
        script: |
          cd $APP_DIR/android
          ./gradlew assembleDebug
    artifacts:
      - $APP_DIR/android/app/build/outputs/apk/**/*.apk
    publishing:
      email:
        recipients: [illeesam@gmail.com]

  shopjoy-fo-android-release:
    name: ShopJoy FO — Android Release (Play Store)
    instance_type: mac_mini_m2
    max_build_duration: 60
    environment:
      vars:
        APP_DIR: "_apps/ShopjoyFoApp"
        PACKAGE_NAME: "com.shopjoy.fo"
      groups:
        - shopjoy_keystore
        - google_play
      node: 20.18.0
      java: 17
    triggering:
      events:
        - tag
      tag_patterns:
        - pattern: 'fo-v*'
          include: true
    scripts:
      - name: Install dependencies
        script: |
          cd $APP_DIR
          npm ci
      - name: Set Capacitor env (prod)
        script: |
          cd $APP_DIR
          npm run config:prod
          npx cap sync android
      - name: Setup keystore
        script: |
          echo $CM_KEYSTORE | base64 --decode > $CM_BUILD_DIR/$APP_DIR/android/app/release.keystore
          cat > $CM_BUILD_DIR/$APP_DIR/android/key.properties <<EOF
          storePassword=$CM_KEYSTORE_PASSWORD
          keyPassword=$CM_KEY_PASSWORD
          keyAlias=$CM_KEY_ALIAS
          storeFile=app/release.keystore
          EOF
      - name: Build Android AAB
        script: |
          cd $APP_DIR/android
          ./gradlew bundleRelease
    artifacts:
      - $APP_DIR/android/app/build/outputs/**/*.aab
    publishing:
      google_play:
        credentials: $GCLOUD_SERVICE_ACCOUNT_CREDENTIALS
        track: internal
        submit_as_draft: true
      email:
        recipients: [illeesam@gmail.com]

  shopjoy-fo-ios-release:
    name: ShopJoy FO — iOS Release (TestFlight)
    instance_type: mac_mini_m2
    max_build_duration: 90
    integrations:
      app_store_connect: shopjoy_app_store
    environment:
      vars:
        APP_DIR: "_apps/ShopjoyFoApp"
        BUNDLE_ID: "com.shopjoy.fo"
        XCODE_WORKSPACE: "App.xcworkspace"
        XCODE_SCHEME: "App"
        APP_STORE_APPLE_ID: 1234567890   # ← App Store Connect 의 앱 ID
      node: 20.18.0
      xcode: 15.4
      cocoapods: default
    triggering:
      events:
        - tag
      tag_patterns:
        - pattern: 'fo-v*'
          include: true
    scripts:
      - name: Install dependencies
        script: |
          cd $APP_DIR
          npm ci
      - name: Set Capacitor env (prod)
        script: |
          cd $APP_DIR
          npm run config:prod
          npx cap sync ios
      - name: Set up provisioning profiles
        script: |
          keychain initialize
          app-store-connect fetch-signing-files $BUNDLE_ID \
            --type IOS_APP_STORE \
            --create
          keychain add-certificates
          xcode-project use-profiles
      - name: Increment build number
        script: |
          cd $APP_DIR/ios/App
          BUILD_NUMBER=$(($(app-store-connect get-latest-app-store-build-number "$APP_STORE_APPLE_ID") + 1))
          agvtool new-version -all $BUILD_NUMBER
      - name: Build IPA
        script: |
          cd $APP_DIR/ios/App
          xcode-project build-ipa \
            --workspace "$XCODE_WORKSPACE" \
            --scheme "$XCODE_SCHEME"
    artifacts:
      - $APP_DIR/ios/App/build/ios/ipa/*.ipa
      - /tmp/xcodebuild_logs/*.log
    publishing:
      app_store_connect:
        auth: integration
        submit_to_testflight: true
        beta_groups:
          - Internal Testers
      email:
        recipients: [illeesam@gmail.com]

  shopjoy-bo-android-debug:
    name: ShopJoy BO — Android Debug
    instance_type: mac_mini_m2
    max_build_duration: 60
    environment:
      vars:
        APP_DIR: "_apps/ShopjoyBoApp"
      node: 20.18.0
      java: 17
    triggering:
      events:
        - push
      branch_patterns:
        - pattern: 'main'
          include: true
    scripts:
      - name: Install dependencies
        script: |
          cd $APP_DIR
          npm ci
      - name: Build Android APK
        script: |
          cd $APP_DIR
          npm run config:prod
          npx cap sync android
          cd android
          ./gradlew assembleDebug
    artifacts:
      - $APP_DIR/android/app/build/outputs/apk/**/*.apk
    publishing:
      email:
        recipients: [illeesam@gmail.com]
```

> BO iOS 워크플로우는 FO와 동일 패턴이라 생략 (필요 시 `shopjoy-bo-ios-release` 추가).

### Step 3. 환경변수 그룹 (Codemagic 콘솔)

좌측 메뉴 → **Environment variables** → **Add group**:

#### `shopjoy_keystore` 그룹

| 변수명 | 값 | secure |
|---|---|---|
| `CM_KEYSTORE` | `base64 -i shopjoy-fo-release.keystore` 결과 | ✅ |
| `CM_KEYSTORE_PASSWORD` | 키스토어 비밀번호 | ✅ |
| `CM_KEY_PASSWORD` | 별칭 비밀번호 | ✅ |
| `CM_KEY_ALIAS` | `shopjoy-fo` | |

#### `google_play` 그룹

| 변수명 | 값 |
|---|---|
| `GCLOUD_SERVICE_ACCOUNT_CREDENTIALS` | Google Play Console 서비스계정 JSON 전체 (secure) |

발급 방법:
1. Google Play Console → Setup → API Access → Google Cloud Platform 연결
2. Service Account 생성 → JSON 키 다운로드
3. Play Console 에서 해당 SA에 **Release Manager** 권한 부여

### Step 4. App Store Connect 통합 (iOS)

Codemagic → **Teams → Integrations → Apple Developer Portal**:

1. **App Store Connect API Key** 등록
   - https://appstoreconnect.apple.com → Users and Access → Keys → ➕
   - Role: **Admin** (또는 App Manager)
   - 다운로드한 `.p8` 파일 + Issuer ID + Key ID 입력
2. Integration 이름: `shopjoy_app_store` (codemagic.yaml의 `integrations:` 와 매칭)

### Step 5. 빌드 트리거

```bash
# Android+iOS 동시 빌드
git tag fo-v1.0.0
git push origin fo-v1.0.0

# BO 앱
git tag bo-v1.0.0
git push origin bo-v1.0.0
```

또는 Codemagic 콘솔에서 **Start new build** 수동 실행.

### 빌드 명령

```bash
cd _apps/ShopjoyFoApp
npm install
npm run config:prod
npx cap sync ios

# Xcode 열기
npx cap open ios

# Xcode 에서:
# 1. 디바이스 → "Any iOS Device (arm64)"
# 2. Product → Scheme → App
# 3. Product → Archive
# 4. Organizer → Distribute App → App Store Connect
```

### Fastlane 자동화 (선택)

```bash
sudo gem install fastlane
cd _apps/ShopjoyFoApp/ios/App
fastlane init
```

`fastlane/Fastfile`:

```ruby
default_platform(:ios)

platform :ios do
  desc "Build and upload to TestFlight"
  lane :beta do
    setup_ci(provider: "travis")
    match(type: "appstore", readonly: true)

    increment_build_number(xcodeproj: "App.xcodeproj")

    build_app(
      workspace: "App.xcworkspace",
      scheme: "App",
      export_method: "app-store"
    )

    upload_to_testflight(skip_waiting_for_build_processing: true)
  end
end
```

```bash
fastlane beta
```

## GitHub Actions (옵션 D)

`.github/workflows/ios-fo.yml`:

```yaml
name: ShopJoy FO iOS

on:
  push:
    tags: ['fo-v*']

jobs:
  build:
    runs-on: macos-14
    defaults:
      run:
        working-directory: _apps/ShopjoyFoApp

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with: { node-version: '20' }

      - uses: maxim-lobanov/setup-xcode@v1
        with: { xcode-version: '15.4' }

      - name: Install dependencies
        run: npm ci

      - name: Configure prod
        run: |
          npm run config:prod
          npx cap sync ios

      - name: Install Pods
        run: |
          cd ios/App
          pod install

      - name: Import signing certificate
        env:
          DIST_CERT_BASE64: ${{ secrets.IOS_DIST_CERT_P12_BASE64 }}
          DIST_CERT_PASSWORD: ${{ secrets.IOS_DIST_CERT_PASSWORD }}
        run: |
          echo "$DIST_CERT_BASE64" | base64 -d > /tmp/cert.p12
          security create-keychain -p "" build.keychain
          security import /tmp/cert.p12 -k build.keychain -P "$DIST_CERT_PASSWORD" -T /usr/bin/codesign
          security list-keychains -d user -s build.keychain
          security set-key-partition-list -S apple-tool:,apple: -s -k "" build.keychain

      - name: Install provisioning profile
        env:
          PROFILE_BASE64: ${{ secrets.IOS_PROVISIONING_PROFILE_BASE64 }}
        run: |
          mkdir -p ~/Library/MobileDevice/Provisioning\ Profiles
          echo "$PROFILE_BASE64" | base64 -d > ~/Library/MobileDevice/Provisioning\ Profiles/profile.mobileprovision

      - name: Build IPA
        run: |
          cd ios/App
          xcodebuild -workspace App.xcworkspace \
            -scheme App \
            -configuration Release \
            -archivePath $RUNNER_TEMP/App.xcarchive \
            archive
          xcodebuild -exportArchive \
            -archivePath $RUNNER_TEMP/App.xcarchive \
            -exportPath $RUNNER_TEMP/ipa \
            -exportOptionsPlist exportOptions.plist

      - name: Upload to TestFlight
        env:
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APP_SPECIFIC_PASSWORD: ${{ secrets.APP_SPECIFIC_PASSWORD }}
        run: |
          xcrun altool --upload-app -f $RUNNER_TEMP/ipa/App.ipa \
            -t ios \
            -u "$APPLE_ID" \
            -p "$APP_SPECIFIC_PASSWORD"
```

> **GitHub macOS runner는 Codemagic보다 비쌉니다** ($0.08/min vs $0.038/min). 빌드 빈도가 높으면 Codemagic 우선.

## 권장 운영 흐름

1. **개발 중**: 본인 PC에서 Android 직접 빌드 + 디바이스 테스트 (실시간 피드백)
2. **PR 머지 시**: Codemagic이 자동으로 main 브랜치 Android Debug APK 생성 → 이메일 첨부 → QA 즉시 검증
3. **릴리즈 태그**: `git tag fo-v1.0.0` push → Codemagic이 Android AAB + iOS IPA 생성 → Play Store Internal Testing + TestFlight 자동 업로드
4. **승격 (Promote)**: 검증 통과 시 Play Console / App Store Connect 콘솔에서 Internal → Production 트랙 승격 (수동)

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| Codemagic: `pod install` 실패 | Capacitor 7+ 사용 | Xcode 15.4+ 사용 (codemagic.yaml `xcode: 15.4`) |
| `code signing required` | 인증서/프로파일 미설정 | App Store Connect Integration 활성화 + `xcode-project use-profiles` |
| `app-store-connect fetch-signing-files` 실패 | Bundle ID가 Apple Dev에 없음 | 사전에 https://developer.apple.com → Identifiers → 등록 |
| `BUILD_NUMBER` 충돌 | TestFlight 동일 build 재업로드 시도 | `app-store-connect get-latest-app-store-build-number + 1` 패턴 사용 (yaml 예시) |
| Google Play API 401 | Service Account 권한 부족 | Play Console에서 Release Manager 권한 부여 + 24h 대기 |
| Gradle 빌드 OOM | Codemagic 무료 인스턴스 메모리 한계 | `mac_mini_m2` 인스턴스 사용 (default `mac_pro` 보다 빠르고 저렴) |
