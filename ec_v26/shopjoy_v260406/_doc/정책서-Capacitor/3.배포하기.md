## 배포 준비 체크리스트

### 사전 등록 필요

| 항목 | FO 앱 | BO 앱 | 비용 |
|---|---|---|---|
| Google Play Console 계정 | ✅ | ✅ | USD $25 (1회) |
| Apple Developer Program | ✅ | ✅ | USD $99/년 |
| Firebase 프로젝트 (FCM) | ✅ | ✅ | 무료 (Spark) |
| 사업자 정보 | ✅ | (선택) | — |
| 개인정보처리방침 URL | ✅ | ✅ | 자체 호스팅 |
| 서비스 이용약관 URL | ✅ | ✅ | 자체 호스팅 |

### 정책서 사전 확인

- 카카오/토스 결제 SDK 연동 시 PG사 심사 자료 (FO만)
- 푸시 알림용 FCM/APNs 인증서
- BO 앱: 내부 사용 vs 일반 공개 — 내부 사용이면 Play Store **내부 테스트** 트랙 활용

## 버전 관리 정책

### 시맨틱 버저닝

| 항목 | 형식 | 변경 시점 |
|---|---|---|
| **versionName** (사용자 표시) | `MAJOR.MINOR.PATCH` (예: `1.2.3`) | 기능 추가/수정 |
| **versionCode** (Android, 정수) | 단조증가 (예: `100200300`) | 빌드마다 +1 또는 빌드번호 |
| **CFBundleVersion** (iOS, 문자열) | `1.2.3.45` (빌드번호 포함) | 빌드마다 +1 |

### 권장 versionCode 산정 공식

```
versionCode = MAJOR * 10000000 + MINOR * 100000 + PATCH * 1000 + BUILD
예: 1.2.3 빌드 5번 → 10203005
```

### 변경 위치

| 파일 | 항목 |
|---|---|
| `_mobile/shopjoy-fo-app/android/app/build.gradle` | `versionCode`, `versionName` |
| `_mobile/shopjoy-fo-app/ios/App/App.xcodeproj` | `MARKETING_VERSION`, `CURRENT_PROJECT_VERSION` |
| `_mobile/shopjoy-fo-app/package.json` | `version` (양 플랫폼 동기화 기준) |

### 자동 동기화 스크립트 (권장)

`_mobile/shopjoy-fo-app/scripts/bump-version.js`:
```javascript
const fs = require('fs');
const path = require('path');
const { version } = require('../package.json');
const [maj, min, pat] = version.split('.').map(Number);
const build = Number(process.env.BUILD_NO || 1);
const versionCode = maj * 10000000 + min * 100000 + pat * 1000 + build;

// Android build.gradle 수정
const gradlePath = path.join(__dirname, '../android/app/build.gradle');
let g = fs.readFileSync(gradlePath, 'utf8');
g = g.replace(/versionCode \d+/, `versionCode ${versionCode}`);
g = g.replace(/versionName "[^"]+"/, `versionName "${version}"`);
fs.writeFileSync(gradlePath, g);

// iOS plist 수정 (Info.plist)
// ...

console.log(`Bumped to ${version} (code ${versionCode})`);
```

실행:
```powershell
$env:BUILD_NO=42
node scripts/bump-version.js
```

## 앱 아이콘 / 스플래시 / 로고

### 자동 생성 도구

```powershell
npm install --save-dev @capacitor/assets
```

원본 이미지 준비 (`_mobile/shopjoy-fo-app/assets/`):
- `icon.png` — **1024×1024 PNG** (투명 배경 X, 모서리 둥글게 X — 시스템이 처리)
- `icon-foreground.png` — 1024×1024 (Android Adaptive Icon 전경, 안전영역 432×432 중앙)
- `icon-background.png` — 1024×1024 (Android Adaptive Icon 배경, 단색 권장)
- `splash.png` — **2732×2732 PNG** (정사각, 중앙 로고 1200×1200 영역)
- `splash-dark.png` — 다크모드 스플래시

생성:
```powershell
cd _mobile/shopjoy-fo-app
npx capacitor-assets generate --iconBackgroundColor '#fff0f4' --iconBackgroundColorDark '#1a1a1a'
```

→ Android `mipmap-*` / iOS `Assets.xcassets` 자동 갱신.

### 스플래시 표시 시간 / 페이드

`capacitor.config.ts`:
```typescript
plugins: {
  SplashScreen: {
    launchShowDuration: 2000,
    backgroundColor: '#fff0f4',
    androidSplashResourceName: 'splash',
    splashFullScreen: true,
    splashImmersive: true,
  },
},
```

## Android 배포

### 1. 키스토어 생성 (최초 1회, 분실 주의)

```powershell
keytool -genkey -v `
  -keystore shopjoy-fo-release.keystore `
  -alias shopjoy-fo `
  -keyalg RSA -keysize 2048 -validity 10000
```

| 입력 항목 | 예시 |
|---|---|
| 키스토어 비밀번호 | (강력한 비밀번호 — 반드시 비밀번호 관리자에 저장) |
| 별칭 비밀번호 | (동일/별도) |
| 이름 | ShopJoy |
| 조직 단위 | Mobile |
| 조직 | ShopJoy Inc. |
| 도시 / 도/주 / 국가 | Seoul / Seoul / KR |

> ⚠️ **키스토어 분실 시 앱 업데이트 영구 불가**. 클라우드 안전 보관 + 백업 필수.

### 2. 키스토어를 Gradle에 등록

`_mobile/shopjoy-fo-app/android/key.properties` (gitignore 처리):
```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=shopjoy-fo
storeFile=../../shopjoy-fo-release.keystore
```

`_mobile/shopjoy-fo-app/android/app/build.gradle`:
```gradle
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

### 3. Release AAB 빌드

```powershell
cd _mobile/shopjoy-fo-app

# 환경변수 production으로 설정 (server.url 제거)
$env:NODE_ENV="production"

# 웹 자산 빌드
npm run build              # copy-web.js fo + cap sync

# Android Bundle (AAB) 생성
cd android
./gradlew bundleRelease
```

결과: `_mobile/shopjoy-fo-app/android/app/build/outputs/bundle/release/app-release.aab`

> Google Play는 **AAB만 허용** (APK는 사이드로딩/내부 배포용).

### 4. APK 빌드 (사내 배포용, 선택)

```powershell
./gradlew assembleRelease
# 결과: app/build/outputs/apk/release/app-release.apk
```

### 5. Google Play Console 업로드

1. https://play.google.com/console 접속
2. **앱 만들기** → 기본 정보 입력 (앱 이름, 기본 언어, 무료/유료)
3. **앱 콘텐츠** 섹션 모두 작성:
   - 개인정보처리방침 URL
   - 광고 포함 여부
   - 콘텐츠 등급 설문
   - 타겟 사용자층
   - 데이터 안전 (수집 데이터 명시)
4. **출시** → **테스트** 트랙 (내부 → 비공개 → 공개) 또는 **프로덕션** 트랙
5. AAB 업로드
6. 출시 노트 작성 (한국어 + 영어 권장)
7. **검토를 위해 출시** → 보통 1~3일 심사

### 6. 첫 배포 시 추가 항목

- **앱 아이콘** 512×512 PNG
- **그래픽 자료** 1024×500 (피처 그래픽)
- **스크린샷** 폰 2~8장, 태블릿 2~8장
- **짧은 설명** (80자), **긴 설명** (4000자)
- **연락처 이메일**, 웹사이트, 개인정보 URL

## iOS 배포

### 1. App Store Connect 앱 등록

1. https://appstoreconnect.apple.com → 마이 앱 → ➕
2. Bundle ID: `com.shopjoy.fo` (Capacitor에서 사용한 appId와 동일)
3. SKU: `SHOPJOY-FO-001`
4. 앱 이름, 기본 언어 입력

### 2. Xcode에서 Archive

```bash
cd _mobile/shopjoy-fo-app
NODE_ENV=production npm run build
npx cap sync ios
npx cap open ios
```

Xcode에서:
1. 상단 디바이스 선택 → **Any iOS Device (arm64)**
2. Product → **Archive**
3. Organizer 창 자동 오픈 → **Distribute App** → **App Store Connect** → **Upload**
4. 자동 서명 (Automatic) 권장

### 3. TestFlight 베타 배포

App Store Connect:
1. **TestFlight** 탭 → 빌드 선택
2. 베타 정보 입력 (테스트 정보, 수출 규정 준수)
3. **내부 테스터** (Apple ID 25명까지, 즉시 배포) 또는 **외부 테스터** (10000명, Apple 검토 24h)

### 4. App Store 출시

1. **앱 정보** 작성:
   - 카테고리 (FO: 쇼핑, BO: 비즈니스)
   - 콘텐츠 권한 (4+/9+/12+/17+)
   - 개인정보처리방침 URL
2. **가격 및 사용 가능 여부** — 무료 / 유료
3. **버전 정보**:
   - 스크린샷 6.7" (1290×2796), 5.5" (1242×2208) 필수
   - 앱 미리보기 비디오 (선택)
   - 설명 (4000자), 키워드 (100자)
   - 새로운 기능 (4000자) — 업데이트 시
4. **빌드 선택** (TestFlight 빌드 중 하나)
5. **앱 심사 정보**:
   - 연락처 (전화/이메일)
   - 데모 계정 (BO 앱 필수)
   - 메모 (특수 기능 설명)
6. **심사를 위해 제출** → 1~3일 (드물게 1주)

## OTA 업데이트 (Capgo Live Update)

앱스토어 심사를 거치지 않고 **JS/CSS/HTML만 핫 업데이트**. 네이티브 코드 변경은 불가.

### Capgo 설치

```powershell
npm install @capgo/capacitor-updater
npx cap sync
```

### 초기화

```javascript
import { CapacitorUpdater } from '@capgo/capacitor-updater';
import { App } from '@capacitor/app';

CapacitorUpdater.notifyAppReady();      // 부트 완료 알림 (필수)

App.addListener('appStateChange', async ({ isActive }) => {
  if (isActive) {
    const latest = await CapacitorUpdater.getLatest();
    if (latest.version !== '<current>') {
      const bundle = await CapacitorUpdater.download({ url: latest.url, version: latest.version });
      await CapacitorUpdater.set({ id: bundle.id });   // 다음 부트부터 적용
    }
  }
});
```

> 앱스토어 정책: **결제/UI/기능 변경**은 OTA 금지. **버그 픽스/문구 수정** 위주로 활용.

## CI/CD (선택)

### GitHub Actions — Android 자동 빌드

`.github/workflows/android-release.yml`:
```yaml
name: Android Release

on:
  push:
    tags: ['v*']

jobs:
  build:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: _mobile/shopjoy-fo-app
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - uses: actions/setup-java@v4
        with: { distribution: 'temurin', java-version: '17' }

      - run: npm ci
      - run: NODE_ENV=production npm run build

      - name: Decode keystore
        run: |
          echo "${{ secrets.ANDROID_KEYSTORE_BASE64 }}" | base64 -d > shopjoy-fo-release.keystore

      - name: Build AAB
        run: cd android && ./gradlew bundleRelease
        env:
          KEYSTORE_PASSWORD: ${{ secrets.KEYSTORE_PASSWORD }}
          KEY_ALIAS: ${{ secrets.KEY_ALIAS }}
          KEY_PASSWORD: ${{ secrets.KEY_PASSWORD }}

      - uses: actions/upload-artifact@v4
        with:
          name: app-release
          path: _mobile/shopjoy-fo-app/android/app/build/outputs/bundle/release/*.aab
```

## 심사 거절 사례 & 대응

| 사례 | 원인 | 대응 |
|---|---|---|
| **iOS 4.0 — Minimum Functionality** | WebView 단순 래핑 | 푸시/카메라/생체인증 등 네이티브 기능 명시. 앱스토어 설명에 차별화 강조 |
| **iOS 5.1.1 — Privacy** | 권한 사용 사유 미작성 | `Info.plist`의 모든 `NSXxxUsageDescription` 사용자 친화 문구 |
| **iOS 4.2.3 — Web Wrapper** | 모든 컨텐츠가 웹페이지와 동일 | 모바일 전용 UX (하단 탭바, 네이티브 입력, 햅틱 등) 추가 |
| **Android — Foreground Service 거절** | 권한 정당화 부족 | 사용 사유 정책 페이지 작성 |
| **Android — 광고 ID 미신고** | 데이터 안전 섹션 누락 | Play Console에서 정확히 신고 |
| **결제 정책 위반** | 외부 결제 사용 | 앱 내 결제(IAP) 사용 또는 외부결제 정책 준수 (한국 한정 외부결제 허용) |
