## 사전 준비

| 항목 | 최소 버전 | 권장 버전 | 용도 |
|---|---|---|---|
| **Node.js** | 18.x | 20.x LTS | Capacitor CLI 실행 |
| **npm** | 9.x | 10.x | 패키지 관리 |
| **Android Studio** | Hedgehog (2023.1) | Koala (2024.1) 이상 | Android SDK / 에뮬레이터 |
| **JDK** | 17 | 17 LTS | Android 빌드 (Gradle) |
| **Xcode** (iOS만) | 15.0 | 15.4 이상 | iOS 빌드 — **macOS 필수** |
| **CocoaPods** (iOS만) | 1.12 | 1.15 이상 | iOS 의존성 관리 |
| **Git** | 2.40 | 최신 | 소스 관리 |

> **Windows 환경에서는 iOS 빌드 불가**. iOS 빌드는 macOS + Xcode 필수.
> Windows에서 안드로이드 빌드 + iOS는 별도 macOS 머신/CI 또는 클라우드 빌드(EAS, Codemagic, Bitrise 등) 활용.

## Android 개발환경

### 1. Android Studio 설치
- https://developer.android.com/studio 에서 다운로드
- 설치 시 **Android SDK / Android SDK Platform / Android Virtual Device** 모두 선택

### 2. SDK Platform 설치 (Android Studio → SDK Manager)
- **Android 14 (API 34)** ✅ Capacitor 6.x 권장 타겟
- **Android 13 (API 33)** ✅
- **Android 12L (API 32)** ✅ 호환성 테스트용
- **최소 지원**: API 22 (Android 5.1) — Capacitor 6 기준

### 3. SDK Tools 설치
- Android SDK Build-Tools 34.0.0
- Android SDK Command-line Tools (latest)
- Android Emulator
- Android SDK Platform-Tools

### 4. 환경 변수 설정 (Windows PowerShell)

```powershell
# 시스템 환경 변수 추가 (PowerShell 관리자)
[System.Environment]::SetEnvironmentVariable('ANDROID_HOME', "$env:LOCALAPPDATA\Android\Sdk", 'User')
[System.Environment]::SetEnvironmentVariable('JAVA_HOME', 'C:\Program Files\Android\Android Studio\jbr', 'User')

# PATH에 추가
$path = [System.Environment]::GetEnvironmentVariable('Path', 'User')
$path += ";$env:LOCALAPPDATA\Android\Sdk\platform-tools"
$path += ";$env:LOCALAPPDATA\Android\Sdk\emulator"
$path += ";$env:LOCALAPPDATA\Android\Sdk\cmdline-tools\latest\bin"
[System.Environment]::SetEnvironmentVariable('Path', $path, 'User')
```

### 5. 환경 확인 (새 PowerShell 창)

```powershell
adb --version           # Android Debug Bridge version 1.0.41
emulator -list-avds     # 생성된 AVD 목록
java --version          # openjdk 17.x.x
```

### 6. AVD (Android Virtual Device) 생성
- Android Studio → Device Manager → Create Device
- **Pixel 7** (해상도 1080×2400) 권장
- System Image: **Android 14 (API 34) Google Play x86_64**
- AVD Name: `Pixel_7_API_34`

```
shopjoy_v260406/                      ← 기존 웹 프로젝트 (그대로 유지)
├── index.html                        ← FO 진입점
├── bo.html                           ← BO 진입점
├── assets/
├── pages/
├── api/
├── _apps/
│   └── EcAdminApi/                   ← Spring Boot 백엔드 (기존)
└── _mobile/                          ← 신규 추가 (모바일 앱 루트)
    ├── shopjoy-fo-app/               ← Capacitor 프로젝트 #1 (고객용)
    │   ├── android/                  ← Android Studio 프로젝트
    │   ├── ios/                      ← Xcode 프로젝트
    │   ├── www/                      ← 웹 자산 빌드 결과 (FO)
    │   ├── capacitor.config.ts
    │   └── package.json
    └── shopjoy-bo-app/               ← Capacitor 프로젝트 #2 (관리자용)
        ├── android/
        ├── ios/
        ├── www/

```

## 기존 ShopJoy 웹 연결

Capacitor의 `webDir` 설정으로 기존 정적 웹 자산을 앱에 포함시킨다.

### 옵션 A. 심볼릭 링크 (개발 권장)

`_mobile/shopjoy-fo-app/capacitor.config.ts`:
```typescript
import { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.shopjoy.fo',
  appName: 'ShopJoy',
  webDir: '../../',                    // 프로젝트 루트 (index.html 위치)
  server: {
    androidScheme: 'https',
    // 개발 중 라이브 리로드: PC IP로 변경
    // url: 'http://192.168.0.100:5501',
    // cleartext: true
  },
  android: {
    allowMixedContent: false,
  },
  ios: {
    contentInset: 'always',
  },
};

export default config;
```

`_mobile/shopjoy-bo-app/capacitor.config.ts`:
```typescript
import { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.shopjoy.bo',
  appName: 'ShopJoy BO',
  webDir: '../../',
  server: {
    androidScheme: 'https',
  },
  // BO 앱은 진입점이 bo.html — npm 빌드 스크립트로 분기
};

export default config;
```

### 옵션 B. 빌드 시 복사 (배포 권장)

`_mobile/shopjoy-fo-app/package.json`에 스크립트 추가:
```json
{
  "scripts": {
    "copy:web": "node scripts/copy-web.js fo",
    "build": "npm run copy:web && npx cap sync"
  }
}
```

`_mobile/shopjoy-fo-app/scripts/copy-web.js`:
```javascript
const fs = require('fs-extra');
const path = require('path');

const target = process.argv[2] || 'fo';
const ROOT = path.resolve(__dirname, '../../..');
const WWW  = path.resolve(__dirname, '../www');

(async () => {
  await fs.emptyDir(WWW);

  // 진입점 복사 (FO는 index.html, BO는 bo.html → index.html로 리네임)
  if (target === 'fo') {
    await fs.copy(path.join(ROOT, 'index.html'), path.join(WWW, 'index.html'));
  } else {
    await fs.copy(path.join(ROOT, 'bo.html'), path.join(WWW, 'index.html'));
  }

  // 공통 자산 복사
  for (const dir of ['assets', 'pages', 'api', 'components', 'utils', 'services', 'base', 'layout']) {
    const src = path.join(ROOT, dir);
    if (await fs.pathExists(src)) {
      await fs.copy(src, path.join(WWW, dir));
    }
  }
  console.log(`[copy-web] ${target} 자산 복사 완료 → ${WWW}`);
})();
```

`fs-extra` 설치:
```powershell
npm install --save-dev fs-extra
```

> **권장**: 개발 단계는 옵션 A(라이브 리로드), 배포는 옵션 B(빌드 복사).

## 필수 플러그인 설치

ShopJoy 요구사항: **푸시 / 결제 / 카메라&QR / 생체인증** 모두 포함.

### FO 앱 + BO 앱 공통

```powershell
# 푸시 알림 (FCM/APNs)
npm install @capacitor/push-notifications

# 카메라 + QR 스캔
npm install @capacitor/camera
npm install @capacitor-mlkit/barcode-scanning

# 보안 스토리지 (토큰 저장 — localStorage 대체)
npm install @capacitor/preferences

# 앱 상태 / 딥링크
npm install @capacitor/app

# 네트워크 상태
npm install @capacitor/network

# 디바이스 정보 (FCM 토큰 등록 시 활용)
npm install @capacitor/device

# 스플래시/상태바
npm install @capacitor/splash-screen
npm install @capacitor/status-bar

# 햅틱(진동)
npm install @capacitor/haptics
```

### BO 앱 추가 (생체인증)

```powershell
npm install @capacitor-community/biometric-auth
```

### FO 앱 추가 (결제)

카카오페이/토스페이는 공식 Capacitor 플러그인이 없으므로 두 가지 방식:

1. **WebView 결제창 호출** (MVP 권장) — 기존 웹 결제 플로우 재사용
2. **커스텀 네이티브 플러그인 작성** — 안정화 단계에서 도입

```powershell
# 외부 브라우저(InAppBrowser)로 결제창 띄우기
npm install @capacitor/browser
```

### 플러그인 동기화

```powershell
npx cap sync
```

> `cap sync` = `npm run copy:web` + 네이티브 프로젝트에 플러그인 반영. 플러그인 추가/제거 후 **반드시** 실행.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| `JAVA_HOME is not set` | JDK 미설치/미설정 | Android Studio 내장 JBR 경로로 설정 |
| `SDK location not found` | ANDROID_HOME 누락 | 환경변수 추가 후 PowerShell 재시작 |
| `Could not resolve com.android.tools.build:gradle` | Gradle 버전 호환성 | `_mobile/shopjoy-*/android/build.gradle`의 Gradle 버전 확인 |
| Android 빌드 시 `Minimum supported Gradle version is 8.x` | Gradle Wrapper 버전 부족 | `./gradlew wrapper --gradle-version 8.7` |
| `pod install` 실패 (M1/M2 Mac) | arm64 호환성 | `arch -x86_64 pod install` |
| 에뮬레이터에서 `localhost` API 호출 실패 | 에뮬레이터 → 호스트 PC 연결 | `10.0.2.2`로 호출 (Android 에뮬레이터 → 호스트 PC) |
| iOS 시뮬레이터 → 호스트 PC | localhost는 시뮬레이터 자신 | `127.0.0.1` 그대로 OK 또는 PC LAN IP |
